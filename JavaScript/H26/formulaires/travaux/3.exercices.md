# Exercices pratiques

# Validation HTML, validation JavaScript, messages d’erreur, événements et formulaires dynamiques

## Objectif général

Ces exercices ont pour objectif de vous aider à rendre vos formulaires plus interactifs, plus fiables et plus agréables à utiliser.

À la fin de cette série, vous devriez être capables de :

- utiliser la validation HTML avec `required`, `type`, `minlength`, `maxlength`, `min` et `max` ;
- créer une validation personnalisée avec JavaScript ;
- afficher des messages d’erreur clairs ;
- ajouter ou retirer des classes CSS selon l’état d’un champ ;
- utiliser les événements `input`, `change`, `focus` et `blur` ;
- créer des formulaires dynamiques qui changent selon les choix de l’utilisateur ;
- activer ou désactiver un bouton selon l’état du formulaire.

---

# Consignes générales

Pour chaque exercice :

1. Créez un fichier HTML.
2. Ajoutez le CSS dans une balise `<style>` ou dans un fichier `.css`.
3. Ajoutez le JavaScript dans une balise `<script>` ou dans un fichier `.js`.
4. Testez le formulaire dans le navigateur.
5. Ouvrez la console du navigateur pour vérifier vos résultats.
6. Utilisez `textContent` pour afficher les messages.
7. Évitez d’utiliser `innerHTML` avec des données écrites par l’utilisateur.
8. Utilisez `event.preventDefault()` lorsque JavaScript contrôle la soumission du formulaire.

# Ce qu’il faut retenir après ces exercices

La validation permet de vérifier que les données entrées par l’utilisateur sont utilisables.

La validation HTML est rapide et pratique pour les règles simples.

La validation JavaScript permet de personnaliser les règles, les messages et les réactions du formulaire.

Les messages d’erreur doivent aider l’utilisateur, pas simplement le bloquer.

Les événements importants sont :

```txt
input  → pendant la saisie
change → lorsqu’un choix est modifié
focus  → quand un champ devient actif
blur   → quand un champ est quitté
submit → quand le formulaire est soumis
```

Les formulaires dynamiques permettent d’adapter l’interface selon les choix de l’utilisateur.

Un bon formulaire ne se contente pas de demander des informations. Il guide l’utilisateur, réagit à ses actions et l’aide à corriger ses erreurs.
