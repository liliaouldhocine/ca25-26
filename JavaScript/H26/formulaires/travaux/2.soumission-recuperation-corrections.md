# Corrigés

# Soumission d’un formulaire, récupération des données avec JavaScript, GET et POST

## Objectif des corrigés

Ces corrigés permettent de vérifier la compréhension des notions suivantes :

- l’événement `submit` ;
- le comportement par défaut du navigateur ;
- `event.preventDefault()` ;
- la récupération des données avec `.value`, `.checked`, `FormData`, `get()` et `getAll()` ;
- l’affichage des données dans la page ;
- les méthodes `GET` et `POST` ;
- la récupération de paramètres dans l’URL avec `URLSearchParams` ;
- la préparation de données pour un envoi avec `fetch`.

---

```html
<form>
  <label for="nom">Nom</label>

   Par exemple, si l’utilisateur écrit Sara, l’URL peut devenir :
```

Même sans JavaScript, un formulaire HTML fonctionne déjà. Le navigateur sait qu’un bouton `type="submit"` sert à envoyer le formulaire.

# Corrigé de l’exercice 2 — Écouter l’événement `submit`

## Code corrigé

```html
<form id="contactForm">
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <button type="submit">Envoyer</button>
</form>

<script>
  const form = document.querySelector("#contactForm");

  form.addEventListener("submit", function (event) {
    console.log("Le formulaire a été soumis.");
  });
</script>
```

## Réponse à la question

Il est préférable d’écouter l’événement `submit` du formulaire plutôt que le clic sur le bouton, car un formulaire peut être soumis de plusieurs façons : avec un clic, avec la touche Entrée ou par un autre mécanisme du navigateur.

Cependant, dans cet exercice, la page peut encore se recharger, car `event.preventDefault()` n’a pas encore été utilisé.


# Corrigé de l’exercice 4 — Récupérer une valeur avec `.value`

# Corrigé de l’exercice 5 — Utiliser `trim()`

## Code corrigé

```html
<form id="prenomForm">
  <label for="prenom">Prénom</label>
  <input type="text" id="prenom" name="prenom" />

  <button type="submit">Envoyer</button>
</form>

<script>
  const form = document.querySelector("#prenomForm");
  const prenomInput = document.querySelector("#prenom");

  form.addEventListener("submit", function (event) {
    event.preventDefault();

    const prenom = prenomInput.value.trim();

    console.log(prenom);
  });
</script>
```

## Réponse à la question

`trim()` est utile parce qu’il enlève les espaces inutiles au début et à la fin d’une chaîne de caractères.

Par exemple :

```javascript
"   Sara   ".trim();
```

devient :

```txt
Sara
```

Cela évite de traiter une valeur avec des espaces comme une vraie valeur différente.

# Corrigé de l’exercice 8 — Comprendre `.checked` avec une checkbox

## Code corrigé

```html
<form id="conditionsForm">
  <label>
    <input type="checkbox" id="conditions" name="conditions" />
    J’accepte les conditions
  </label>

  <button type="submit">Envoyer</button>
</form>

<script>
  const form = document.querySelector("#conditionsForm");
  const conditionsInput = document.querySelector("#conditions");

  form.addEventListener("submit", function (event) {
    event.preventDefault();

    console.log(conditionsInput.checked);
  });
</script>
```

## Réponse à la question

`.value` donne la valeur associée à la checkbox, mais ne dit pas si elle est cochée.

Pour savoir si une checkbox est cochée, on utilise :

```javascript
conditionsInput.checked;
```

Cela retourne :

```txt
true
```

ou :

```txt
false
```

# Corrigé de l’exercice 9 — Récupérer un bouton radio sélectionné

## Code corrigé

```html

  <label>

  </label>

  </label>

  <label>
```

```javascript
document.querySelector("input[name='mode']:checked");
```

Si aucun bouton radio n’est sélectionné, le résultat est `null`.

# Corrigé de l’exercice 10 — Récupérer plusieurs checkbox cochées

## Code corrigé

```html
<form id="technologiesForm">
  <p>Technologies connues</p>

  <label>
    <input type="checkbox" name="technologies" value="html" />
    HTML
  </label>

  <label>
    <input type="checkbox" name="technologies" value="css" />
    CSS
  </label>

  <label>
    <input type="checkbox" name="technologies" value="javascript" />
    JavaScript
  </label>

  <label>
    <input type="checkbox" name="technologies" value="git" />
    Git
  </label>

  <button type="submit">Envoyer</button>
</form>

<script>
  const form = document.querySelector("#technologiesForm");

  form.addEventListener("submit", function (event) {
    event.preventDefault();

    const technologiesSelectionnees = document.querySelectorAll(
      "input[name='technologies']:checked",
    );

    const technologies = Array.from(technologiesSelectionnees).map(
      function (checkbox) {
        return checkbox.value;
      },
    );

    console.log(technologies);
  });
</script>
```

## Variante avec fonction fléchée

```javascript
const technologies = Array.from(technologiesSelectionnees).map(
  (checkbox) => checkbox.value,
);
```

## Explication

`querySelectorAll()` retourne une liste d’éléments.
`Array.from()` transforme cette liste en tableau.
`.map()` permet de créer un tableau contenant seulement les valeurs.

# Corrigé de l’exercice 11 — Découvrir `FormData`

## Code corrigé

```html
<form id="inscriptionForm">
  <label for="prenom">Prénom</label>
  <input type="text" id="prenom" name="prenom" />

  <label for="email">Adresse courriel</label>
  <input type="email" id="email" name="email" />

  <label for="niveau">Niveau</label>
  <select id="niveau" name="niveau">
    <option value="">Choisir un niveau</option>
    <option value="debutant">Débutant</option>
    <option value="intermediaire">Intermédiaire</option>
    <option value="avance">Avancé</option>
  </select>

  <button type="submit">Envoyer</button>
</form>

<script>
  const form = document.querySelector("#inscriptionForm");

  form.addEventListener("submit", function (event) {
    event.preventDefault();

    const formData = new FormData(form);

    const prenom = formData.get("prenom");
    const email = formData.get("email");
    const niveau = formData.get("niveau");

    console.log("Prénom :", prenom);
    console.log("Email :", email);
    console.log("Niveau :", niveau);
  });
</script>
```

## Réponse à la question

`FormData` utilise les attributs `name` parce que ce sont les `name` qui représentent les noms des données du formulaire.

Exemple :

```html
<input id="courriel" name="email" />
```

Avec `FormData`, on récupère :

```javascript
formData.get("email");
```

et non :

```javascript
formData.get("courriel");
```

# Corrigé de l’exercice 12 — Transformer `FormData` en objet JavaScript

## Code corrigé

```html
<form id="inscriptionForm">
  <label for="prenom">Prénom</label>
  <input type="text" id="prenom" name="prenom" />

  <label for="email">Adresse courriel</label>
  <input type="email" id="email" name="email" />

  <label for="niveau">Niveau</label>
  <select id="niveau" name="niveau">
    <option value="">Choisir un niveau</option>
    <option value="debutant">Débutant</option>
    <option value="intermediaire">Intermédiaire</option>
    <option value="avance">Avancé</option>
  </select>

  <button type="submit">Envoyer</button>
</form>

<script>
  const form = document.querySelector("#inscriptionForm");

  form.addEventListener("submit", function (event) {
    event.preventDefault();

    const formData = new FormData(form);
    const donnees = Object.fromEntries(formData.entries());

    console.log(donnees);
  });
</script>
```

## Résultat possible

```javascript
{
  prenom: "Sara",
  email: "sara@email.com",
  niveau: "debutant"
}
```

## Explication

`Object.fromEntries()` transforme les paires clé-valeur de `FormData` en objet JavaScript.

# Corrigé de l’exercice 16 — Comprendre GET avec un formulaire de recherche

## Code attendu dans `index.html`

```html
<form action="resultats.html" method="GET">
  <label for="mot">Mot-clé</label>
  <input type="search" id="mot" name="mot" />

  <label for="categorie">Catégorie</label>
  <select id="categorie" name="categorie">
    <option value="">Toutes</option>
    <option value="html">HTML</option>
    <option value="css">CSS</option>
    <option value="javascript">JavaScript</option>
  </select>

  <button type="submit">Rechercher</button>
</form>
```

## Réponses aux questions

1. Avec GET, les données apparaissent dans l’URL.

Exemple :

```txt
resultats.html?mot=formulaire&categorie=javascript
```

2. GET est adapté à une recherche parce qu’une recherche peut être partagée, copiée dans un lien ou retrouvée dans l’historique du navigateur.

3. GET n’est pas adapté à un mot de passe parce que les données apparaissent dans l’URL. Un mot de passe ne doit jamais être visible dans la barre d’adresse.

# Corrigé de l’exercice 21 — Simuler un envoi avec `fetch`

## Code corrigé

```html
<form id="messageForm">
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <label for="message">Message</label>
  <textarea id="message" name="message"></textarea>

  <button type="submit">Envoyer</button>
</form>

<p id="resultat"></p>

<script>
  const form = document.querySelector("#messageForm");
  const resultat = document.querySelector("#resultat");

  form.addEventListener("submit", async function (event) {
    event.preventDefault();

    const formData = new FormData(form);
    const donnees = Object.fromEntries(formData.entries());

    resultat.textContent = "Tentative d’envoi du formulaire...";
    console.log("Données envoyées :", donnees);

    try {
      const response = await fetch("/message", {
        method: "POST",
        headers: {
          "Content-Type": "application/json",
        },
        body: JSON.stringify(donnees),
      });

      console.log("Réponse du serveur :", response);
    } catch (error) {
      console.error("Erreur pendant l’envoi :", error);
    }
  });
</script>
```

## Explication

Si aucun serveur n’existe à l’adresse `/message`, la requête échouera. C’est normal dans cet exercice. L’objectif est de comprendre la structure générale d’un envoi avec `fetch`.

# Corrigé de l’exercice 22 — Identifier les erreurs dans un code JavaScript

## Code corrigé

```html
<form id="userForm">
  <label for="courriel">Adresse courriel</label>
  <input type="email" id="courriel" name="email" />

  <label>
    <input type="checkbox" id="conditions" name="conditions" value="oui" />
    J’accepte les conditions
  </label>

  <button type="submit">Envoyer</button>
</form>

<script>
  const form = document.querySelector("#userForm");

  form.addEventListener("submit", function (event) {
    event.preventDefault();

    const formData = new FormData(form);

    const email = formData.get("email");
    const conditions = document.querySelector("#conditions").checked;

    console.log(email);
    console.log(conditions);
  });
</script>
```

## Réponses aux questions

### 1. Pourquoi `formData.get("courriel")` ne fonctionne-t-il pas ici ?

Parce que `courriel` est le `id` du champ, pas son `name`.

Le champ est :

```html
<input type="email" id="courriel" name="email" />
```

Avec `FormData`, on doit utiliser le `name` :

```javascript
formData.get("email");
```

### 2. Pourquoi `.checked` est-il plus approprié que `.value` pour la checkbox ?

Parce que `.checked` indique si la case est cochée ou non.

```javascript
true;
```

ou :

```javascript
false;
```

`.value` donne seulement la valeur associée à la case, ici `"oui"`, mais ne dit pas si l’utilisateur l’a cochée.

# Corrigé de l’exercice 25 — Synthèse : choisir GET ou POST

| Situation                            | GET ou POST ? | Justification                                                                                             |
| ------------------------------------ | ------------- | --------------------------------------------------------------------------------------------------------- |
| Rechercher un article dans un blogue | GET           | On demande de l’information. Les paramètres de recherche peuvent apparaître dans l’URL.                   |
| Envoyer un formulaire de contact     | POST          | L’utilisateur transmet un message. Les données ne devraient pas être placées dans l’URL.                  |
| Se connecter à un compte             | POST          | Les informations de connexion ne doivent pas être visibles dans l’URL.                                    |
| Filtrer une liste de produits        | GET           | Le filtre sert à consulter des résultats. L’URL peut être partagée.                                       |
| Créer un nouveau commentaire         | POST          | On crée une nouvelle donnée.                                                                              |
| Rechercher un étudiant par nom       | GET           | On demande ou filtre de l’information.                                                                    |
| Modifier un profil utilisateur       | POST          | On envoie des données pour modifier une information existante.                                            |
| S’inscrire à une activité            | POST          | On transmet des informations d’inscription.                                                               |
| Trier une liste par prix             | GET           | Le tri est une demande d’affichage ou de consultation.                                                    |
| Envoyer un mot de passe              | POST          | Un mot de passe ne doit jamais apparaître dans l’URL. En vraie application, il faut aussi utiliser HTTPS. |

# Synthèse des erreurs à surveiller

## 1. Oublier `event.preventDefault()`

Erreur :

```javascript
form.addEventListener("submit", function (event) {
  console.log("Formulaire soumis");
});
```

Correction :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();
  console.log("Formulaire soumis");
});
```

---

## 2. Utiliser le mauvais nom avec `FormData`

HTML :

```html
<input id="courriel" name="email" />
```

Erreur :

```javascript
formData.get("courriel");
```

Correction :

```javascript
formData.get("email");
```

---

## 3. Utiliser `.value` au lieu de `.checked`

Erreur :

```javascript
const conditions = conditionsInput.value;
```

Correction :

```javascript
const conditions = conditionsInput.checked;
```

---

## 4. Oublier `getAll()` pour les checkbox multiples

Erreur :

```javascript
const technologies = formData.get("technologies");
```

Correction :

```javascript
const technologies = formData.getAll("technologies");
```

---

## 5. Réinitialiser le formulaire trop tôt

Erreur :

```javascript
form.reset();

const formData = new FormData(form);
```

Correction :

```javascript
const formData = new FormData(form);
const donnees = Object.fromEntries(formData.entries());

form.reset();
```

# Ce qu’il faut retenir

La structure de base à maîtriser est celle-ci :

```javascript
const form = document.querySelector("#monFormulaire");

form.addEventListener("submit", function (event) {
  event.preventDefault();

  const formData = new FormData(form);
  const donnees = Object.fromEntries(formData.entries());

  console.log(donnees);
});
```

Pour plusieurs checkbox :

```javascript
donnees.technologies = formData.getAll("technologies");
```

Pour lire des paramètres GET :

```javascript
const params = new URLSearchParams(window.location.search);
const mot = params.get("mot");
```

Pour préparer un envoi JSON :

```javascript
const donneesJSON = JSON.stringify(donnees);
```

À ce stade, vous devez être prêts à aborder la suite : **la validation des formulaires, les messages d’erreur et l’expérience utilisateur**.
