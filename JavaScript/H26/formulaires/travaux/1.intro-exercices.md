# Exercices pratiques

# Introduction aux formulaires HTML

## Objectif général

Ces exercices ont pour objectif de vous aider à comprendre comment construire correctement un formulaire HTML.

À la fin de cette série, vous devriez être capables de :

- créer un formulaire avec la balise `<form>` ;
- utiliser correctement les champs HTML ;
- associer un `<label>` à un champ ;
- distinguer `id`, `name`, `value` et `placeholder` ;
- utiliser les champs `input`, `textarea`, `select`, `checkbox` et `radio` ;
- choisir le bon type de champ selon le contexte ;
- structurer un formulaire de manière claire et lisible ;
- éviter les erreurs fréquentes dans les formulaires HTML.

---

# Consignes générales

Pour chaque exercice :

1. Créez un fichier HTML.
2. Écrivez un code propre et bien indenté.
3. Utilisez des labels pour tous les champs importants.
4. Donnez un `id` et un `name` pertinents aux champs.
5. Testez le formulaire dans le navigateur.
6. Vérifiez que les champs obligatoires se comportent correctement.
7. Ne faites pas encore de JavaScript, sauf si cela est explicitement demandé dans un autre cours.

# Exercice 1 — Observer la structure d’un formulaire

## Objectif

Comprendre les éléments de base d’un formulaire HTML.

## Code de départ

Copiez le code suivant dans un fichier HTML :

```html
<form>
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <button type="submit">Envoyer</button>
</form>
```

## Questions

Répondez aux questions suivantes dans un commentaire HTML sous le formulaire.

```html
<!-- Vos réponses ici -->
```

1. À quoi sert la balise `<form>` ?
2. À quoi sert la balise `<label>` ?
3. Pourquoi le `for="nom"` du label correspond-il au `id="nom"` du champ ?
4. À quoi sert l’attribut `name="nom"` ?
5. Quel est le rôle du bouton `type="submit"` ?

# Exercice 3 — Corriger un formulaire mal construit

## Objectif

Repérer les erreurs fréquentes dans un formulaire HTML.

## Code de départ

Voici un formulaire incomplet ou mal construit :

```html
<form>
  <input type="text" placeholder="Prénom" />

  <label>Nom</label>
  <input type="text" id="nom" />

  <label for="email">Email</label>
  <input type="email" id="courriel" />

  <button>Envoyer</button>
</form>
```

## Travail à faire

Corrigez ce formulaire pour qu’il respecte les bonnes pratiques.

Vous devez :

1. Ajouter les labels manquants.
2. Lier correctement les labels aux champs.
3. Ajouter les attributs `name`.
4. Vérifier que les `id` correspondent aux `for`.
5. Ajouter explicitement le type du bouton.
6. Garder un `placeholder` seulement s’il sert d’exemple, et non de remplacement au label.

# Exercice 4 — Comprendre `id` et `name`

## Objectif

Distinguer le rôle de `id` et de `name`.

## Travail à faire

Créez un formulaire avec les champs suivants :

- adresse courriel ;
- nom d’utilisateur ;
- mot de passe.

Pour chaque champ, utilisez un `id` et un `name`.

Ensuite, ajoutez sous le formulaire un commentaire HTML qui explique la différence entre `id` et `name`.

Exemple attendu :

```html
<!--
id sert à ...
name sert à ...
-->
```

## Contraintes

Le champ courriel doit utiliser :

```html
type="email"
```

Le champ mot de passe doit utiliser :

```html
type="password"
```

# Exercice 6 — Utiliser `required`, `min`, `max`, `minlength` et `maxlength`

# Exercice 9 — Comprendre la différence entre checkbox et radio

## Objectif

Savoir quand utiliser des cases à cocher et quand utiliser des boutons radio.

## Travail à faire

Créez un formulaire qui demande à l’utilisateur :

1. Les technologies qu’il connaît déjà.
2. Son mode de cours préféré.

## Partie 1 — Technologies connues

Utilisez des cases à cocher pour les choix suivants :

- HTML ;
- CSS ;
- JavaScript ;
- Git ;
- React.

L’utilisateur doit pouvoir choisir plusieurs réponses.

## Partie 2 — Mode de cours préféré

Utilisez des boutons radio pour les choix suivants :

- en présentiel ;
- à distance ;
- hybride.

L’utilisateur doit pouvoir choisir une seule réponse.

## Contraintes

Pour les checkbox, utilisez le même `name` pour toutes les technologies.

Exemple :

```html
name="technologies"
```

Pour les boutons radio, utilisez le même `name` pour tous les modes.

Exemple :

```html
name="mode"
```

Chaque choix doit avoir une valeur différente avec `value`.

# Exercice 10 — Structurer un formulaire avec `<fieldset>` et `<legend>`

## Objectif

Organiser un formulaire en sections claires.

## Travail à faire

Créez un formulaire d’inscription à un atelier.

Le formulaire doit être divisé en trois sections :

1. Informations personnelles
2. Informations sur le cours
3. Préférences

Utilisez :

```html

  <legend>...</legend>
</fieldset>
```

## Section 1 — Informations personnelles

Ajoutez :

- prénom ;
- nom ;
- adresse courriel.

## Section 2 — Informations sur le cours

Ajoutez :

- niveau actuel en JavaScript avec un `<select>` ;
- motivation avec un `<textarea>`.

## Section 3 — Préférences

Ajoutez :

- mode de participation avec des boutons radio ;
- technologies connues avec des checkbox ;
- acceptation des conditions avec une checkbox obligatoire.

## Contraintes

Tous les champs importants doivent avoir un label.

Les champs prénom, nom, courriel, niveau et acceptation des conditions doivent être obligatoires.

# Exercice 11 — Compléter un formulaire incomplet

    </div>

      <label>

        J’accepte les conditions
      </label>
    </div>

    <button>Envoyer</button>
  </fieldset>

## Travail à faire

Complétez le formulaire.

Vous devez ajouter :

- les `for` sur les labels ;
- les `id` sur les champs ;
- les `name` sur les champs ;
- les `value` dans les options ;
- l’attribut `required` lorsque nécessaire ;
- le type du bouton.

# Exercice 12 — Analyser les attributs d’un champ

## Objectif

Comprendre précisément le rôle des attributs.

## Code à analyser

```html
<label for="username">Nom d’utilisateur</label>
<input
  type="text"
  id="username"
  name="username"
  placeholder="Ex. sara123"
  minlength="3"
  maxlength="20"
  required
/>
```

## Questions

Répondez aux questions suivantes dans un commentaire HTML.

1. Quel est le rôle de `type="text"` ?
2. Quel est le rôle de `id="username"` ?
3. Quel est le rôle de `name="username"` ?
4. Quel est le rôle de `placeholder="Ex. sara123"` ?
5. Quel est le rôle de `minlength="3"` ?
6. Quel est le rôle de `maxlength="20"` ?
7. Quel est le rôle de `required` ?
8. Pourquoi le `for` du label doit-il correspondre au `id` du champ ?

# Ce qu’il faut retenir après ces exercices

```txt
<label> explique le champ.
id identifie l’élément dans la page.
name nomme la donnée du formulaire.
value représente la valeur envoyée.
placeholder donne un exemple.
required rend un champ obligatoire.
checkbox permet plusieurs choix.
radio permet un seul choix.
select permet de choisir dans une liste.
submit déclenche l’envoi du formulaire.
```
