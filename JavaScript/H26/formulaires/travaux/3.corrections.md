# Corrigés

# Validation HTML, validation JavaScript, messages d’erreur, événements et formulaires dynamiques

## Objectif des corrigés

Ces corrigés permettent de vérifier la compréhension des notions suivantes :

- validation HTML avec `required`, `type`, `minlength`, `maxlength`, `min`, `max` ;
- validation personnalisée avec JavaScript ;
- affichage de messages d’erreur ;
- ajout et retrait de classes CSS ;
- événements `input`, `change`, `focus`, `blur` ;
- formulaires dynamiques ;
- activation et désactivation de boutons ;
- récupération finale des données avec `FormData`.

---

# Corrigé de l’exercice 1 — Validation HTML simple

```html
<form>
  <div>
    <label for="username">Nom d’utilisateur</label>
    <input
      type="text"
      id="username"
      name="username"
      minlength="3"
      maxlength="20"
      required
    />
  </div>

  <div>
    <label for="email">Adresse courriel</label>
    <input type="email" id="email" name="email" required />
  </div>

  <div>
    <label for="password">Mot de passe</label>
    <input
      type="password"
      id="password"
      name="password"
      minlength="8"
      required
    />
  </div>

  <div>
    <label for="age">Âge</label>
    <input type="number" id="age" name="age" min="12" max="120" required />
  </div>

  <button type="submit">Créer le compte</button>
</form>

<!--
Lorsqu’un champ obligatoire est vide, le navigateur peut bloquer la soumission du formulaire.
Il affiche alors un message automatique pour demander à l’utilisateur de remplir le champ.
-->
```

## Explication

Ce formulaire utilise uniquement la validation HTML. Le navigateur vérifie automatiquement les contraintes avant de soumettre le formulaire.

# Corrigé de l’exercice 2 — Observer les limites de la validation HTML

```html
<!--
1. Oui, le navigateur bloque généralement l’envoi du formulaire si une règle HTML n’est pas respectée.

2. Les messages affichés par le navigateur ne sont pas facilement personnalisables.
   Ils dépendent du navigateur utilisé et de la langue de celui-ci.

3. On peut vouloir utiliser JavaScript en plus de la validation HTML pour afficher nos propres messages,
   contrôler leur emplacement, ajouter des classes CSS, ou créer des règles plus personnalisées.

4. Non, la validation HTML ne suffit pas pour sécuriser une vraie application.
   Elle améliore l’expérience utilisateur, mais les données doivent aussi être validées côté serveur.
-->
```

## Explication

La validation HTML est utile, mais elle ne donne pas un contrôle complet au développeur. Pour des formulaires plus professionnels, on utilise souvent JavaScript en complément.

# Corrigé de l’exercice 4 — Première validation JavaScript

```html
<form id="contactForm" novalidate>
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <p id="nomError"></p>

  <button type="submit">Envoyer</button>
</form>

<script>
  const form = document.querySelector("#contactForm");
  const nomInput = document.querySelector("#nom");
  const nomError = document.querySelector("#nomError");

  form.addEventListener("submit", function (event) {
    event.preventDefault();

    const nom = nomInput.value.trim();

    if (nom === "") {
      nomError.textContent = "Le nom est obligatoire.";
      return;
    }

    console.log("Formulaire valide.");
  });
</script>
```

## Explication

Le formulaire est contrôlé par JavaScript. On empêche le comportement automatique avec `event.preventDefault()`, puis on vérifie la valeur du champ.

# Corrigé de l’exercice 6 — Valider plusieurs champs avec JavaScript

```html
<style>
  .error {
    color: red;
    font-size: 0.9rem;
  }
</style>

<form id="contactForm" novalidate>
  <div>
    <label for="nom">Nom</label>
    <input type="text" id="nom" name="nom" />
    <p id="nomError" class="error"></p>
  </div>

  <div>
    <label for="email">Adresse courriel</label>
    <input type="email" id="email" name="email" />
    <p id="emailError" class="error"></p>
  </div>

  <div>
    <label for="message">Message</label>
    <textarea id="message" name="message"></textarea>
    <p id="messageError" class="error"></p>
  </div>

  <button type="submit">Envoyer</button>
</form>

<script>
  const form = document.querySelector("#contactForm");

  const nomInput = document.querySelector("#nom");
  const emailInput = document.querySelector("#email");
  const messageInput = document.querySelector("#message");

  const nomError = document.querySelector("#nomError");
  const emailError = document.querySelector("#emailError");
  const messageError = document.querySelector("#messageError");

  form.addEventListener("submit", function (event) {
    event.preventDefault();

    nomError.textContent = "";
    emailError.textContent = "";
    messageError.textContent = "";

    const nom = nomInput.value.trim();
    const email = emailInput.value.trim();
    const message = messageInput.value.trim();

    let formulaireValide = true;

    if (nom === "") {
      nomError.textContent = "Le nom est obligatoire.";
      formulaireValide = false;
    }

    if (email === "") {
      emailError.textContent = "L’adresse courriel est obligatoire.";
      formulaireValide = false;
    }

    if (message === "") {
      messageError.textContent = "Le message est obligatoire.";
      formulaireValide = false;
    }

    if (!formulaireValide) {
      return;
    }

    console.log("Formulaire valide.");
  });
</script>
```

## Explication

Chaque champ possède son propre message d’erreur. Cela permet de guider précisément l’utilisateur.

# Corrigé de l’exercice 10 — Valider une longueur minimale

```html
<form id="accountForm" novalidate>
  <div>
    <label for="username">Nom d’utilisateur</label>
    <input type="text" id="username" name="username" />
    <p id="usernameError"></p>
  </div>

  <div>
    <label for="password">Mot de passe</label>
    <input type="password" id="password" name="password" />
    <p id="passwordError"></p>
  </div>

  <button type="submit">Créer le compte</button>
</form>

<script>
  const form = document.querySelector("#accountForm");

  const usernameInput = document.querySelector("#username");
  const passwordInput = document.querySelector("#password");

  const usernameError = document.querySelector("#usernameError");
  const passwordError = document.querySelector("#passwordError");

  form.addEventListener("submit", function (event) {
    event.preventDefault();

    usernameError.textContent = "";
    passwordError.textContent = "";

    const username = usernameInput.value.trim();
    const password = passwordInput.value.trim();

    let formulaireValide = true;

    if (username === "") {
      usernameError.textContent = "Le nom d’utilisateur est obligatoire.";
      formulaireValide = false;
    } else if (username.length < 3) {
      usernameError.textContent =
        "Le nom d’utilisateur doit contenir au moins 3 caractères.";
      formulaireValide = false;
    }

    if (password === "") {
      passwordError.textContent = "Le mot de passe est obligatoire.";
      formulaireValide = false;
    } else if (password.length < 8) {
      passwordError.textContent =
        "Le mot de passe doit contenir au moins 8 caractères.";
      formulaireValide = false;
    }

    if (!formulaireValide) {
      return;
    }

    console.log("Formulaire valide.");
  });
</script>
```

## Explication

On vérifie d’abord si le champ est vide, puis seulement ensuite sa longueur. Cela évite d’afficher un message de longueur alors que le champ n’a pas encore été rempli.

# Corrigé de l’exercice 14 — Utiliser l’événement `blur`

```html
<label for="email">Adresse courriel</label>
<input type="email" id="email" name="email" />

<p id="emailHelp"></p>
<p id="emailError"></p>

<script>
  const emailInput = document.querySelector("#email");
  const emailHelp = document.querySelector("#emailHelp");
  const emailError = document.querySelector("#emailError");

  emailInput.addEventListener("focus", function () {
    emailHelp.textContent =
      "Entrez une adresse courriel valide, par exemple nom@exemple.com.";
  });

  emailInput.addEventListener("blur", function () {
    emailHelp.textContent = "";

    const email = emailInput.value.trim();

    if (email === "") {
      emailError.textContent = "L’adresse courriel est obligatoire.";
    } else {
      emailError.textContent = "";
    }
  });
</script>
```

## Explication

L’événement `blur` se déclenche lorsque l’utilisateur quitte le champ. C’est un bon moment pour faire une validation douce.

# Corrigé de l’exercice 15 — Comparer `input`, `change`, `focus` et `blur`

```html
<label for="prenom">Prénom</label>
<input type="text" id="prenom" name="prenom" />

<label for="niveau">Niveau</label>
<select id="niveau" name="niveau">
  <option value="">Choisir un niveau</option>
  <option value="debutant">Débutant</option>
  <option value="intermediaire">Intermédiaire</option>
  <option value="avance">Avancé</option>
</select>

<script>
  const champ = document.querySelector("#prenom");
  const niveauSelect = document.querySelector("#niveau");

  champ.addEventListener("input", function () {
    console.log("input : la valeur change pendant la saisie.");
  });

  champ.addEventListener("focus", function () {
    console.log("focus : le champ est actif.");
  });

  champ.addEventListener("blur", function () {
    console.log("blur : le champ a perdu le focus.");
  });

  niveauSelect.addEventListener("change", function () {
    console.log("change : le choix du select a changé.");
  });

  /*
  1. L’événement input se déclenche pendant la saisie.
  2. L’événement focus se déclenche quand un champ devient actif.
  3. L’événement blur se déclenche quand un champ perd le focus.
  4. L’événement change est particulièrement utile avec un select.
  */
</script>
```

# Corrigé de l’exercice 18 — Rendre un champ obligatoire dynamiquement

```html

    display: none;
  }
</style>

<form id="accountForm">
  <label for="typeCompte">Type de compte</label>
  <select id="typeCompte" name="typeCompte">
```

désactive le bouton.

# Corrigé de l’exercice 21 — Ajouter un champ dynamiquement

```html
<form id="skillsForm">
  <div id="skillsContainer">
    <div>
      <label>Compétence</label>
      <input type="text" name="competences" />
    </div>
  </div>

  <button type="button" id="addSkillButton">Ajouter une compétence</button>
  <button type="submit">Envoyer</button>
</form>

<script>
  const skillsContainer = document.querySelector("#skillsContainer");
  const addSkillButton = document.querySelector("#addSkillButton");

  addSkillButton.addEventListener("click", function () {
    const div = document.createElement("div");

    const label = document.createElement("label");
    label.textContent = "Compétence";

    const input = document.createElement("input");
    input.type = "text";
    input.name = "competences";

    div.appendChild(label);
    div.appendChild(input);

    skillsContainer.appendChild(div);
  });
</script>
```

## Réponse à la question

Le bouton “Ajouter une compétence” doit être `type="button"` parce qu’il ne doit pas soumettre le formulaire. S’il était `type="submit"`, il déclencherait l’envoi du formulaire au lieu d’ajouter un champ.

# Corrigé de l’exercice 22 — Récupérer les champs ajoutés dynamiquement

```html
<form id="skillsForm">
  <div id="skillsContainer">
    <div>
      <label>Compétence</label>
      <input type="text" name="competences" />
    </div>
  </div>

  <button type="button" id="addSkillButton">Ajouter une compétence</button>
  <button type="submit">Envoyer</button>
</form>

<script>
  const form = document.querySelector("#skillsForm");
  const skillsContainer = document.querySelector("#skillsContainer");
  const addSkillButton = document.querySelector("#addSkillButton");

  addSkillButton.addEventListener("click", function () {
    const div = document.createElement("div");

    const label = document.createElement("label");
    label.textContent = "Compétence";

    const input = document.createElement("input");
    input.type = "text";
    input.name = "competences";

    div.appendChild(label);
    div.appendChild(input);

    skillsContainer.appendChild(div);
  });

  form.addEventListener("submit", function (event) {
    event.preventDefault();

    const formData = new FormData(form);
    const competences = formData.getAll("competences");

    console.log(competences);
  });
</script>
```

## Explication

Comme tous les champs de compétences ont le même `name`, on utilise :

```javascript
formData.getAll("competences");
```

# Corrigé de l’exercice 23 — Formulaire dynamique selon le profil

```html
<style>
  .hidden {
    display: none;
  }
</style>

<form id="profilForm">
  <div>
    <label for="prenom">Prénom</label>
    <input type="text" id="prenom" name="prenom" />
  </div>

  <div>
    <label for="typeProfil">Type de profil</label>
    <select id="typeProfil" name="typeProfil">
      <option value="">Choisir un type</option>
      <option value="etudiant">Étudiant</option>
      <option value="professionnel">Professionnel</option>
    </select>
  </div>

  <div id="etudiantSection" class="hidden">
    <label for="ecole">Nom de l’école</label>
    <input type="text" id="ecole" name="ecole" />
  </div>

  <div id="professionnelSection" class="hidden">
    <label for="entreprise">Nom de l’entreprise</label>
    <input type="text" id="entreprise" name="entreprise" />
  </div>

  <button type="submit">Envoyer</button>
</form>

<script>
  const typeProfilSelect = document.querySelector("#typeProfil");

  const etudiantSection = document.querySelector("#etudiantSection");
  const professionnelSection = document.querySelector("#professionnelSection");

  const ecoleInput = document.querySelector("#ecole");
  const entrepriseInput = document.querySelector("#entreprise");

  typeProfilSelect.addEventListener("change", function () {
    const typeProfil = typeProfilSelect.value;

    etudiantSection.classList.add("hidden");
    professionnelSection.classList.add("hidden");

    ecoleInput.required = false;
    entrepriseInput.required = false;

    ecoleInput.value = "";
    entrepriseInput.value = "";

    if (typeProfil === "etudiant") {
      etudiantSection.classList.remove("hidden");
      ecoleInput.required = true;
    }

    if (typeProfil === "professionnel") {
      professionnelSection.classList.remove("hidden");
      entrepriseInput.required = true;
    }
  });
</script>
```

# Corrigé de l’exercice 24 — Validation d’un formulaire dynamique

```html
<style>

    display: none;
  }

  .error {
    color: red;
    font-size: 0.9rem;
  }

  .input-error {
    border: 2px solid red;
  }
</style>

<form id="profilForm" novalidate>
  <div>
    <label for="prenom">Prénom</label>
    <input type="text" id="prenom" name="prenom" />
    <p id="prenomError" class="error"></p>
  </div>

  <div>
    <label for="typeProfil">Type de profil</label>
    <select id="typeProfil" name="typeProfil">
      <option value="">Choisir un type</option>
      <option value="etudiant">Étudiant</option>
      <option value="professionnel">Professionnel</option>
    </select>
    <p id="typeProfilError" class="error"></p>
  </div>

  <div id="etudiantSection" class="hidden">
    <label for="ecole">Nom de l’école</label>
    <input type="text" id="ecole" name="ecole" />
    <p id="ecoleError" class="error"></p>
  </div>

  <div id="professionnelSection" class="hidden">
    <label for="entreprise">Nom de l’entreprise</label>
    <input type="text" id="entreprise" name="entreprise" />
    <p id="entrepriseError" class="error"></p>
  </div>

  <button type="submit">Envoyer</button>
</form>

<script>
  const form = document.querySelector("#profilForm");

  const prenomInput = document.querySelector("#prenom");
  const typeProfilSelect = document.querySelector("#typeProfil");
  const ecoleInput = document.querySelector("#ecole");
  const entrepriseInput = document.querySelector("#entreprise");

  const etudiantSection = document.querySelector("#etudiantSection");
  const professionnelSection = document.querySelector("#professionnelSection");

  const prenomError = document.querySelector("#prenomError");
  const typeProfilError = document.querySelector("#typeProfilError");
  const ecoleError = document.querySelector("#ecoleError");
  const entrepriseError = document.querySelector("#entrepriseError");

  typeProfilSelect.addEventListener("change", function () {
    const typeProfil = typeProfilSelect.value;

    etudiantSection.classList.add("hidden");
    professionnelSection.classList.add("hidden");

    ecoleInput.required = false;
    entrepriseInput.required = false;

    ecoleInput.value = "";
    entrepriseInput.value = "";

    ecoleError.textContent = "";
    entrepriseError.textContent = "";

    if (typeProfil === "etudiant") {
      etudiantSection.classList.remove("hidden");
      ecoleInput.required = true;
    }

    if (typeProfil === "professionnel") {
      professionnelSection.classList.remove("hidden");
      entrepriseInput.required = true;
    }
  });

  form.addEventListener("submit", function (event) {
    event.preventDefault();

    prenomError.textContent = "";
    typeProfilError.textContent = "";
    ecoleError.textContent = "";
    entrepriseError.textContent = "";

    prenomInput.classList.remove("input-error");
    typeProfilSelect.classList.remove("input-error");
    ecoleInput.classList.remove("input-error");
    entrepriseInput.classList.remove("input-error");

    const prenom = prenomInput.value.trim();
    const typeProfil = typeProfilSelect.value;
    const ecole = ecoleInput.value.trim();
    const entreprise = entrepriseInput.value.trim();

    let formulaireValide = true;

    if (prenom === "") {
      prenomError.textContent = "Le prénom est obligatoire.";
      prenomInput.classList.add("input-error");
      formulaireValide = false;
    }

    if (typeProfil === "") {
      typeProfilError.textContent = "Veuillez choisir un type de profil.";
      typeProfilSelect.classList.add("input-error");
      formulaireValide = false;
    }

    if (typeProfil === "etudiant" && ecole === "") {
      ecoleError.textContent = "Le nom de l’école est obligatoire.";
      ecoleInput.classList.add("input-error");
      formulaireValide = false;
    }

    if (typeProfil === "professionnel" && entreprise === "") {
      entrepriseError.textContent = "Le nom de l’entreprise est obligatoire.";
      entrepriseInput.classList.add("input-error");
      formulaireValide = false;
    }

    if (!formulaireValide) {
      return;
    }

    const formData = new FormData(form);
    const donnees = Object.fromEntries(formData.entries());

    console.log(donnees);
  });
</script>
```

# Corrigé du défi supplémentaire — Ajouter un résumé avant l’envoi

Pour ajouter le résumé, on ajoute cette section après le formulaire :

```html
<section>
  <h2>Résumé de l’inscription</h2>
  <pre id="resume"></pre>
</section>
```

Puis on ajoute cette sélection dans le JavaScript :

```javascript
const resume = document.querySelector("#resume");
```

Enfin, dans la partie où le formulaire est valide, on ajoute :

```javascript
resume.textContent = JSON.stringify(donnees, null, 2);
```

# Synthèse pédagogique des erreurs fréquentes

## 1. Oublier `event.preventDefault()`

Erreur :

```javascript
form.addEventListener("submit", function (event) {
  console.log("Formulaire soumis");
});
```

Correction :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();
  console.log("Formulaire soumis");
});
```

---

## 2. Ne pas vider les anciens messages d’erreur

Erreur :

```javascript
if (prenom === "") {
  prenomError.textContent = "Le prénom est obligatoire.";
}
```

Correction :

```javascript
prenomError.textContent = "";

if (prenom === "") {
  prenomError.textContent = "Le prénom est obligatoire.";
}
```

---

## 3. Oublier `trim()`

Erreur :

```javascript
const prenom = prenomInput.value;
```

Correction :

```javascript
const prenom = prenomInput.value.trim();
```

---

## 4. Valider un champ caché inutilement

Erreur :

```javascript
if (ecole === "") {
  ecoleError.textContent = "Le nom de l’école est obligatoire.";
}
```

Correction :

```javascript
if (typeParticipant === "etudiant" && ecole === "") {
  ecoleError.textContent = "Le nom de l’école est obligatoire.";
}
```

---

## 5. Oublier de vider un champ caché

Correction recommandée :

```javascript
ecoleInput.value = "";
entrepriseInput.value = "";
```

---

## 6. Utiliser un bouton dynamique sans `type="button"`

Erreur :

```html
<button id="addSkillButton">Ajouter une compétence</button>
```

Correction :

```html
<button type="button" id="addSkillButton">Ajouter une compétence</button>
```

# Ce qu’il faut retenir

La validation JavaScript suit souvent cette structure :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();

  let formulaireValide = true;

  if (champ.value.trim() === "") {
    messageErreur.textContent = "Ce champ est obligatoire.";
    formulaireValide = false;
  }

  if (!formulaireValide) {
    return;
  }

  console.log("Formulaire valide.");
});
```

Les événements importants sont :

```txt
input  → pendant la saisie
change → lorsqu’un choix change
focus  → quand un champ devient actif
blur   → quand un champ est quitté
submit → quand le formulaire est soumis
```

Les formulaires dynamiques permettent de créer des interfaces plus intelligentes, qui affichent seulement les champs nécessaires selon les choix de l’utilisateur.
