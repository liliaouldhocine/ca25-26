# Exercices pratiques

# Soumission d’un formulaire, récupération des données avec JavaScript, GET et POST

## Objectif général

Ces exercices ont pour objectif de vous aider à comprendre ce qui se passe lorsqu’un formulaire est soumis.

À la fin de cette série, vous devriez être capables de :

- écouter l’événement `submit` d’un formulaire ;
- comprendre le comportement par défaut du navigateur ;
- utiliser `event.preventDefault()` ;
- récupérer les valeurs des champs avec JavaScript ;
- utiliser `.value`, `.checked`, `FormData`, `get()` et `getAll()` ;
- transformer les données d’un formulaire en objet JavaScript ;
- comprendre la différence entre GET et POST ;
- récupérer des paramètres GET dans l’URL ;
- préparer des données pour un envoi avec `fetch`.

---

# Consignes générales

Pour chaque exercice :

1. Créez un fichier HTML.
2. Ajoutez votre JavaScript dans une balise `<script>` ou dans un fichier `.js` séparé.
3. Testez toujours votre formulaire dans le navigateur.
4. Ouvrez la console du navigateur pour observer les résultats.
5. N’ajoutez pas encore de validation complexe, sauf si l’exercice le demande.
6. Concentrez-vous sur la soumission, la récupération des données et la logique GET / POST.

# Ce qu’il faut retenir après ces exercices

La soumission d’un formulaire déclenche un événement important : `submit`.

JavaScript peut intercepter cet événement avec :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();
});
```

La méthode `.value` permet de récupérer la valeur d’un champ texte.

La propriété `.checked` permet de savoir si une checkbox est cochée.

`FormData` permet de récupérer plus facilement les données d’un formulaire complet.

```javascript
const formData = new FormData(form);
```

`Object.fromEntries()` permet de transformer ces données en objet JavaScript.

```javascript
const donnees = Object.fromEntries(formData.entries());
```

Pour plusieurs checkbox avec le même `name`, il faut utiliser :

```javascript
formData.getAll("technologies");
```

GET place les données dans l’URL et convient aux recherches.

POST envoie les données dans le corps de la requête et convient aux formulaires de contact, inscriptions, connexions ou créations de données.

Ces exercices préparent directement la prochaine étape : **la validation des formulaires et l’affichage des messages d’erreur**.
