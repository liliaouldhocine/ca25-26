# Corrigés — Introduction aux formulaires HTML

## Objectif des corrigés

Ces corrigés servent à vérifier la compréhension des bases des formulaires HTML :

- la balise `<form>` ;
- les champs HTML ;
- les labels ;
- les attributs `id`, `name`, `value`, `placeholder`, `required` ;
- les champs `input`, `textarea`, `select`, `checkbox`, `radio` ;
- la structure claire d’un formulaire.

Ces corrigés ne contiennent pas encore de JavaScript. L’objectif est d’abord de maîtriser la construction HTML du formulaire.

---

# Corrigé de l’exercice 1 — Observer la structure d’un formulaire

## Code corrigé avec réponses

```html
<form>
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <button type="submit">Envoyer</button>
</form>

<!--
1. La balise <form> sert à regrouper les champs d’un formulaire.
   Elle indique au navigateur que les éléments à l’intérieur appartiennent au même formulaire.

2. La balise <label> sert à décrire un champ.
   Elle indique à l’utilisateur quelle information il doit saisir.

3. Le for="nom" du label correspond au id="nom" du champ pour créer un lien entre le texte du label et le champ.
   Cela améliore la lisibilité, l’accessibilité et permet de cliquer sur le label pour activer le champ.

4. L’attribut name="nom" sert à nommer la donnée du formulaire.
   Si l’utilisateur écrit "Sara", la donnée pourra être comprise comme : nom = Sara.

5. Le bouton type="submit" sert à soumettre le formulaire.
   Il déclenche l’envoi ou la soumission du formulaire.
-->
```

## Explication pédagogique

Ce premier exercice permet de comprendre que chaque élément a un rôle précis.

Le formulaire n’est pas seulement un ensemble de champs visuels. C’est une structure qui prépare des données à être envoyées ou traitées plus tard.

# Corrigé de l’exercice 2 — Créer un formulaire d’identité simple

## Code corrigé

```html
<form>
  <div>
    <label for="prenom">Prénom</label>
    <input type="text" id="prenom" name="prenom" />
  </div>

  <div>
    <label for="nom">Nom</label>
    <input type="text" id="nom" name="nom" />
  </div>

  <div>
    <label for="ville">Ville</label>
    <input type="text" id="ville" name="ville" />
  </div>

  <div>
    <label for="pays">Pays</label>
    <input type="text" id="pays" name="pays" />
  </div>

  <button type="submit">Envoyer</button>
</form>
```

## Explication pédagogique

Chaque champ possède :

```html
<label for="prenom">Prénom</label>
<input type="text" id="prenom" name="prenom" />
```

Le `for` du label correspond au `id` du champ.

Le `name` permet de nommer la donnée. Par exemple :

```txt
prenom = Sara
nom = Dupont
ville = Montréal
pays = Canada
```

# Corrigé de l’exercice 3 — Corriger un formulaire mal construit

## Code de départ

```html
<form>
  <input type="text" placeholder="Prénom" />

  <label>Nom</label>
  <input type="text" id="nom" />

  <label for="email">Email</label>
  <input type="email" id="courriel" />

  <button>Envoyer</button>
</form>
```

## Code corrigé

```html
<form>
  <div>
    <label for="prenom">Prénom</label>
    <input type="text" id="prenom" name="prenom" placeholder="Ex. Sara" />
  </div>

  <div>
    <label for="nom">Nom</label>
    <input type="text" id="nom" name="nom" placeholder="Ex. Dupont" />
  </div>

  <div>
    <label for="courriel">Adresse courriel</label>
    <input
      type="email"
      id="courriel"
      name="email"
      placeholder="exemple@email.com"
    />
  </div>

  <button type="submit">Envoyer</button>
</form>
```

## Erreurs corrigées

Le premier champ avait un `placeholder`, mais pas de label. Le placeholder ne doit pas remplacer le label.

Le champ nom avait un label, mais le label n’était pas lié au champ avec `for`.

Le label du courriel avait :

```html
<label for="email"></label>
```

mais le champ avait :

```html
id="courriel"
```

Les deux ne correspondaient pas.

Le bouton n’avait pas de type explicite. Il est préférable d’écrire :

```html
<button type="submit">Envoyer</button>
```

# Corrigé de l’exercice 4 — Comprendre `id` et `name`

## Code corrigé

```html
<form>
  <div>
    <label for="email">Adresse courriel</label>
    <input type="email" id="email" name="email" />
  </div>

  <div>

  </div>

  <div>
    <label for="password">Mot de passe</label>
    <input type="password" id="password" name="password" />
  </div>

  <button type="submit">Créer le compte</button>

<!--
id sert à identifier un élément précis dans la page.
Il est utilisé pour lier un label à un champ, pour cibler l’élément en CSS ou pour le récupérer avec JavaScript.

name sert à nommer la donnée du formulaire.
Lorsque le formulaire est soumis ou récupéré, c’est le name qui permet d’identifier chaque valeur.
-->
```

## Explication pédagogique

Dans ce champ :

```html
<input type="email" id="email" name="email" />
```

le `id` identifie l’élément dans la page.
Le `name` identifie la donnée du formulaire.

Ils ont ici la même valeur pour simplifier, mais ils n’ont pas exactement le même rôle.

# Corrigé de l’exercice 6 — Utiliser `required`, `min`, `max`, `minlength` et `maxlength`

## Code corrigé

```html
<form>
  <div>
    <label for="username">Nom d’utilisateur</label>
    <input
      type="text"
      id="username"
      name="username"
      minlength="3"
      maxlength="20"
      required
    />
  </div>

  <div>
    <label for="email">Adresse courriel</label>
    <input type="email" id="email" name="email" required />
  </div>

  <div>
    <label for="password">Mot de passe</label>
    <input
      type="password"
      id="password"
      name="password"
      minlength="8"
      required
    />
  </div>

  <div>
    <label for="age">Âge</label>
    <input type="number" id="age" name="age" min="12" max="120" required />
  </div>

  <button type="submit">Créer le compte</button>
</form>
```

## Explication pédagogique

Le champ suivant est obligatoire et doit contenir entre 3 et 20 caractères :

```html
<input
  type="text"
  id="username"
  name="username"
  minlength="3"
  maxlength="20"
  required
/>
```

Le mot de passe doit contenir au moins 8 caractères :

```html
minlength="8"
```

L’âge doit être compris entre 12 et 120 :

```html
min="12" max="120"
```

# Corrigé de l’exercice 7 — Créer une zone de texte avec `<textarea>`

## Code corrigé

```html
<form>
  <div>
    <label for="nom">Nom</label>
    <input type="text" id="nom" name="nom" required />
  </div>

  <div>
    <label for="email">Adresse courriel</label>
    <input type="email" id="email" name="email" required />
  </div>

  <div>
    <label for="sujet">Sujet</label>
    <input type="text" id="sujet" name="sujet" />
  </div>

  <div>
    <label for="message">Message</label>
    <textarea
      id="message"
      name="message"
      placeholder="Écrivez votre message ici..."
      required
    ></textarea>
  </div>

  <button type="submit">Envoyer le message</button>
</form>
```

## Explication pédagogique

Le champ `message` utilise une balise `<textarea>` parce qu’il peut contenir un texte long.

```html
<textarea id="message" name="message"></textarea>
```

Contrairement à `<input>`, la balise `<textarea>` possède une balise ouvrante et une balise fermante.

Le sujet est facultatif, donc il ne contient pas `required`.

# Corrigé de l’exercice 9 — Comprendre la différence entre checkbox et radio

## Code corrigé

```html
<form>
  <fieldset>
    <legend>Technologies connues</legend>

    <label>
      <input type="checkbox" name="technologies" value="html" />
      HTML
    </label>

    <label>
      <input type="checkbox" name="technologies" value="css" />
      CSS
    </label>

    <label>
      <input type="checkbox" name="technologies" value="javascript" />
      JavaScript
    </label>

    <label>
      <input type="checkbox" name="technologies" value="git" />
      Git
    </label>

    <label>
      <input type="checkbox" name="technologies" value="react" />
      React
    </label>
  </fieldset>

  <fieldset>
    <legend>Mode de cours préféré</legend>

    <label>
      <input type="radio" name="mode" value="presentiel" />
      En présentiel
    </label>

    <label>
      <input type="radio" name="mode" value="distance" />
      À distance
    </label>

    <label>
      <input type="radio" name="mode" value="hybride" />
      Hybride
    </label>
  </fieldset>

  <button type="submit">Envoyer</button>
</form>
```

## Explication pédagogique

Les checkbox permettent plusieurs choix.

C’est pour cela que l’utilisateur peut cocher HTML, CSS et JavaScript en même temps.

Toutes les checkbox ont le même `name` :

```html
name="technologies"
```

mais elles ont des valeurs différentes :

```html
value="html" value="css" value="javascript"
```

Les boutons radio permettent un seul choix dans un groupe.

Tous les boutons radio ont le même `name` :

```html
name="mode"
```

Le navigateur comprend ainsi que l’utilisateur ne peut choisir qu’un seul mode.

# Corrigé de l’exercice 12 — Analyser les attributs d’un champ

## Code analysé

```html
<label for="username">Nom d’utilisateur</label>
<input
  type="text"
  id="username"
  name="username"
  placeholder="Ex. sara123"
  minlength="3"
  maxlength="20"
  required
/>
```

## Réponses corrigées

```html
<!--
1. type="text" indique que le champ attend une courte valeur textuelle.

2. id="username" identifie ce champ précis dans la page.
   Il sert notamment à le lier au label avec for="username".

3. name="username" nomme la donnée du formulaire.
   Si l’utilisateur écrit "sara123", la donnée pourra être représentée ainsi : username = sara123.

4. placeholder="Ex. sara123" donne un exemple de valeur attendue.
   Il ne remplace pas le label.

5. minlength="3" impose un minimum de 3 caractères.

6. maxlength="20" impose un maximum de 20 caractères.

7. required rend le champ obligatoire.
   Le formulaire ne devrait pas être soumis si ce champ est vide.

8. Le for du label doit correspondre au id du champ pour que le navigateur comprenne que ce label décrit ce champ.
   Cela améliore la clarté, l’accessibilité et l’utilisation du formulaire.
-->
```

## Explication pédagogique

Ce champ est bien construit parce qu’il contient :

- un label clair ;
- un lien entre `for` et `id` ;
- un `name` ;
- un exemple avec `placeholder` ;
- des contraintes de longueur ;
- l’attribut `required`.

# Corrigé de l’exercice 14 — Identifier les erreurs dans un formulaire radio

## Code de départ

```html
<p>Choisissez votre niveau</p>

<label>
  <input type="radio" name="debutant" value="debutant" />
  Débutant
</label>

<label>
  <input type="radio" name="intermediaire" value="intermediaire" />
  Intermédiaire
</label>

<label>
  <input type="radio" name="avance" value="avance" />
  Avancé
</label>
```

## Réponses

### 1. Quel est le problème avec les attributs `name` ?

Chaque bouton radio a un `name` différent :

```html
name="debutant" name="intermediaire" name="avance"
```

Le navigateur ne comprend donc pas que ces boutons appartiennent au même groupe.

---

### 2. Que risque-t-il de se passer dans le navigateur ?

L’utilisateur risque de pouvoir sélectionner plusieurs options en même temps, car le navigateur considère que ce sont trois groupes différents.

---

### 3. Quelle valeur de `name` devrait être utilisée pour les trois boutons radio ?

Les trois boutons devraient avoir le même `name`, par exemple :

```html
name="niveau"
```

---

### 4. Code corrigé

```html
<p>Choisissez votre niveau</p>

<label>
  <input type="radio" name="niveau" value="debutant" />
  Débutant
</label>

<label>
  <input type="radio" name="niveau" value="intermediaire" />
  Intermédiaire
</label>

<label>
  <input type="radio" name="niveau" value="avance" />
  Avancé
</label>
```

## Explication pédagogique

Pour les boutons radio, le `name` sert à créer le groupe.

Même `name` signifie :

```txt
Ces choix appartiennent à la même question.
L’utilisateur ne peut choisir qu’une seule réponse.
```

Chaque bouton garde cependant une valeur différente avec `value`.

# Commentaire pédagogique sur le mini-projet

Ce formulaire est une bonne synthèse parce qu’il utilise presque toutes les notions importantes de cette première partie.

On y retrouve :

```html
<form></form>
```

pour créer le formulaire.

```html
<fieldset>
  <legend></legend>
</fieldset>
```

pour organiser les sections.

```html
<label for=""> <input id="" /></label>
```

pour lier les labels aux champs.

```html
name=""
```

pour nommer les données.

```html
required
```

pour rendre certains champs obligatoires.

```html
type="text" type="email" type="tel" type="number" type="file" type="radio"
type="checkbox"
```

pour choisir le bon type de champ.

```html
<select>
  <option></option>
</select>
```

pour créer une liste déroulante.

```html
<textarea>
```

pour les réponses longues.

# Synthèse des erreurs à surveiller dans les copies

Voici les erreurs les plus probables.

## 1. Oublier les labels

Erreur :

```html
<input type="text" name="prenom" />
```

Correction :

```html
<label for="prenom">Prénom</label>
<input type="text" id="prenom" name="prenom" />
```

---

## 2. Mettre un `for` qui ne correspond pas au `id`

Erreur :

```html
<label for="email">Adresse courriel</label>
<input type="email" id="courriel" name="email" />
```

Correction :

```html
<label for="courriel">Adresse courriel</label>
<input type="email" id="courriel" name="email" />
```

ou :

```html
<label for="email">Adresse courriel</label>
<input type="email" id="email" name="email" />
```

---

## 3. Oublier le `name`

Erreur :

```html
<input type="text" id="prenom" />
```

Correction :

```html
<input type="text" id="prenom" name="prenom" />
```

---

## 4. Donner des `id` identiques

Erreur :

```html
<input id="nom" name="prenom" /> <input id="nom" name="nom" />
```

Correction :

```html
<input id="prenom" name="prenom" /> <input id="nom" name="nom" />
```

---

## 5. Mal construire les boutons radio

Erreur :

```html
<input type="radio" name="presentiel" value="presentiel" />
<input type="radio" name="distance" value="distance" />
```

Correction :

```html
<input type="radio" name="mode" value="presentiel" />
<input type="radio" name="mode" value="distance" />
```

---

## 6. Oublier les `value` dans les options

Erreur :

```html
<option>Débutant</option>
<option>Avancé</option>
```

Correction :

```html
<option value="debutant">Débutant</option>
<option value="avance">Avancé</option>
```

---

## 7. Oublier le type du bouton

Erreur :

```html
<button>Envoyer</button>
```

Correction :

```html
<button type="submit">Envoyer</button>
```

---

# Ce que vous devez retenir après les corrigés

Un formulaire bien construit doit être clair pour l’utilisateur et logique pour le programme.

La structure minimale recommandée est souvent :

```html
<form>
  <div>
    <label for="champ">Nom du champ</label>
    <input type="text" id="champ" name="champ" />
  </div>

  <button type="submit">Envoyer</button>
</form>
```
