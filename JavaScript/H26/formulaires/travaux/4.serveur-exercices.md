# Exercices pratiques

# Envoi avec `fetch` et réception côté serveur avec Express

## Objectif général

Ces exercices ont pour objectif de vous aider à comprendre comment un formulaire peut envoyer ses données à un serveur, et comment un serveur Express peut les recevoir.

À la fin de cette série, vous devriez être capables de :

- récupérer les données d’un formulaire avec JavaScript ;
- transformer les données en objet JavaScript ;
- transformer un objet en JSON avec `JSON.stringify()` ;
- envoyer une requête avec `fetch` ;
- utiliser la méthode `POST` ;
- lire une réponse serveur avec `response.json()` ;
- vérifier si une réponse est correcte avec `response.ok` ;
- créer un serveur Express simple ;
- servir une page HTML avec Express ;
- recevoir des données JSON dans `req.body` ;
- répondre au navigateur avec `res.json()` ;
- faire une validation simple côté serveur.

---

# Consignes générales

Pour ces exercices, vous allez travailler avec deux parties :

```txt
Côté client  → HTML + JavaScript dans le navigateur
Côté serveur → Node.js + Express
```

Pour les exercices avec Express, utilisez une structure de projet semblable à celle-ci :

```txt
mon-projet/
│
├── server.js
├── package.json
│
└── public/
    ├── index.html
    └── script.js
```

Avant de commencer les exercices avec Express, installez Express :

```bash
npm init -y
npm install express
```

Si vous utilisez `import`, ajoutez ceci dans `package.json` :

```json
{
  "type": "module"
}
```

Comprendre l’erreur `Cannot POST`.

## Travail à faire

Modifiez volontairement l’URL dans `fetch`.

Par exemple, utilisez :

```javascript
fetch("/api/contacts", {
  method: "POST",
  ...
});
```

au lieu de :

```javascript
fetch("/api/contact", {
  method: "POST",
  ...
});
```

## Questions

1. Que se passe-t-il ?
2. Est-ce que la route Express existe ?
3. Pourquoi l’URL dans `fetch` doit-elle correspondre exactement à la route du serveur ?
4. Comment corriger le problème ?

# Exercice 29 — Lire une réponse qui n’est pas du JSON

## Objectif

Comprendre l’erreur liée à `response.json()`.

## Travail à faire

Dans Express, créez une route qui répond avec du texte simple :

```javascript
app.get("/api/texte", function (req, res) {
  res.send("Réponse texte simple");
});
```

Côté client, essayez volontairement de lire la réponse avec :

```javascript
const data = await response.json();
```

## Questions

1. Que se passe-t-il ?
2. Pourquoi `response.json()` ne fonctionne-t-il pas ici ?
3. Quelle méthode pourrait-on utiliser pour lire du texte simple ?
4. Pourquoi est-il important que le client et le serveur soient d’accord sur le format de réponse ?

# Défi supplémentaire — Application complète avec deux formulaires

## Objectif

Créer une petite application Express avec deux formulaires différents.

## Formulaire 1 — Contact

Champs :

- nom ;
- email ;
- message.

Route :

```txt
POST /api/contact
```

---

## Formulaire 2 — Inscription

Champs :

- prénom ;
- nom ;
- email ;
- niveau ;
- motivation.

Route :

```txt
POST /api/inscription
```

---

## Contraintes

Votre serveur doit :

1. servir les fichiers du dossier `public` ;
2. lire le JSON ;
3. avoir deux routes POST ;
4. valider les champs obligatoires ;
5. retourner des messages d’erreur clairs ;
6. retourner des réponses JSON ;
7. utiliser des statuts HTTP appropriés.

Votre JavaScript côté client doit :

1. gérer les deux formulaires ;
2. envoyer chaque formulaire vers la bonne route ;
3. afficher le message de réponse près du bon formulaire ;
4. utiliser `try...catch` ;
5. vérifier `response.ok`.

# Ce qu’il faut retenir après ces exercices

Pour envoyer un formulaire avec `fetch`, la structure générale côté client est :

```javascript
form.addEventListener("submit", async function (event) {
  event.preventDefault();

  const formData = new FormData(form);
  const donnees = Object.fromEntries(formData.entries());

  const response = await fetch("/api/contact", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
    },
    body: JSON.stringify(donnees),
  });

  const data = await response.json();
});
```

Côté serveur, la structure générale est :

```javascript
app.use(express.json());

app.post("/api/contact", function (req, res) {
  console.log(req.body);

  res.json({
    message: "Données reçues.",
  });
});
```

La logique complète est :

```txt
Le formulaire est soumis.
JavaScript bloque le rechargement.
Les données sont récupérées.
Les données sont transformées en JSON.
fetch envoie une requête POST.
Express reçoit la requête.
express.json() lit les données.
req.body contient les données.
Le serveur valide.
Le serveur répond.
Le navigateur affiche la réponse.
```

Ces exercices préparent la suite : mieux structurer le code, gérer plusieurs routes, connecter une base de données ou construire une API plus complète.
