# Support de cours

# Soumission d’un formulaire, récupération des données avec JavaScript, GET et POST

## Présentation générale

Dans le cours précédent, nous avons étudié la structure d’un formulaire HTML : la balise `<form>`, les champs, les labels et les attributs importants comme `id`, `name`, `value`, `placeholder` et `required`.

Nous allons maintenant passer à l’étape suivante : comprendre ce qui se passe lorsqu’un formulaire est soumis.

Un formulaire n’est pas seulement un ensemble de champs visibles dans une page. Il possède aussi un comportement. Lorsqu’un utilisateur clique sur un bouton d’envoi, appuie sur la touche Entrée ou déclenche une soumission, le navigateur réagit.

Pour bien maîtriser les formulaires, il faut donc comprendre trois notions importantes :

```txt
1. La soumission du formulaire
2. La récupération des données avec JavaScript
3. Les méthodes GET et POST
```

Ces notions sont essentielles pour construire des interfaces interactives, valider des données, préparer un envoi vers une API ou comprendre comment une page communique avec un serveur.

---

# 1. Rappel : le rôle d’un formulaire

Un formulaire permet à l’utilisateur d’entrer ou de choisir des informations.

Exemple :

```html
<form>
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <button type="submit">Envoyer</button>
</form>
```

Ce formulaire contient :

- un champ pour écrire un nom ;
- un bouton pour envoyer le formulaire.

Cependant, tant que l’on ne comprend pas la soumission, on ne comprend pas encore ce qui arrive aux données.

---

# 2. Qu’est-ce que la soumission d’un formulaire ?

La soumission d’un formulaire correspond au moment où l’utilisateur demande au navigateur d’envoyer ou de traiter les informations du formulaire.

Elle peut se produire lorsque :

- l’utilisateur clique sur un bouton `submit` ;
- l’utilisateur appuie sur la touche Entrée dans certains champs ;
- du JavaScript déclenche la soumission du formulaire.

Exemple :

```html
<form>
  <label for="email">Adresse courriel</label>
  <input type="email" id="email" name="email" />

  <button type="submit">Envoyer</button>
</form>
```

Lorsque l’utilisateur clique sur le bouton, le formulaire est soumis.

Le navigateur va alors essayer de faire ce que le formulaire lui demande.

---

# 3. Le bouton `type="submit"`

Le bouton le plus important dans un formulaire est le bouton de soumission.

```html
<button type="submit">Envoyer</button>
```

Ce bouton déclenche l’événement de soumission du formulaire.

Il signifie :

> L’utilisateur a terminé de remplir le formulaire et souhaite l’envoyer.

Exemple complet :

```html
<form id="contactForm">
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <button type="submit">Envoyer</button>
</form>
```

Ici, le bouton soumet le formulaire `contactForm`.

---

# 4. Attention au bouton sans type

Dans un formulaire, un bouton sans attribut `type` peut être interprété comme un bouton de soumission.

Exemple à éviter :

```html
<form>
  <input type="text" name="nom" />

  <button>Annuler</button>
</form>
```

Même si le texte du bouton est “Annuler”, le navigateur peut le traiter comme :

```html
<button type="submit">Annuler</button>
```

Cela peut provoquer une soumission involontaire du formulaire.

Il est donc préférable d’écrire explicitement le type du bouton.

```html
<button type="submit">Envoyer</button> <button type="button">Annuler</button>
```

---

# 5. Les trois types principaux de boutons

Dans un formulaire, on rencontre surtout trois types de boutons.

## 5.1 `type="submit"`

```html
<button type="submit">Envoyer</button>
```

Ce bouton soumet le formulaire.

Il est utilisé pour envoyer ou traiter les données.

---

## 5.2 `type="button"`

```html
<button type="button">Afficher plus d’options</button>
```

Ce bouton ne soumet pas le formulaire automatiquement.

Il est souvent utilisé avec JavaScript pour :

- afficher une section ;
- cacher un élément ;
- ajouter un champ ;
- ouvrir une fenêtre ;
- calculer une valeur ;
- déclencher une action sans soumettre le formulaire.

---

## 5.3 `type="reset"`

```html
<button type="reset">Réinitialiser</button>
```

Ce bouton remet les champs du formulaire à leur valeur initiale.

Il faut l’utiliser avec prudence, car il peut faire perdre les informations saisies par l’utilisateur.

---

# 6. Le comportement par défaut du navigateur

Par défaut, lorsqu’un formulaire est soumis, le navigateur tente de l’envoyer.

Pour savoir où et comment l’envoyer, il regarde deux attributs importants de la balise `<form>` :

```html
<form action="/contact" method="POST"></form>
```

Ces deux attributs sont :

```txt
action  → indique où envoyer les données
method  → indique comment envoyer les données
```

Exemple :

```html
<form action="/contact" method="POST">
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <button type="submit">Envoyer</button>
</form>
```

Ici, le navigateur comprend :

```txt
Je dois envoyer les données vers /contact
Je dois utiliser la méthode POST
```

---

# 7. Que se passe-t-il si on ne précise pas `action` et `method` ?

Si on écrit :

```html
<form>
  <input type="text" name="nom" />
  <button type="submit">Envoyer</button>
</form>
```

le navigateur utilise des comportements par défaut.

Généralement :

- l’action se fait vers la page actuelle ;
- la méthode par défaut est `GET`.

Cela peut entraîner un rechargement de la page ou l’ajout des données dans l’URL.

Pour éviter les comportements inattendus, il est préférable d’être explicite.

Exemple :

```html
<form action="/contact" method="POST"></form>
```

ou, si JavaScript contrôle entièrement le formulaire :

```html
<form id="contactForm"></form>
```

avec `event.preventDefault()` côté JavaScript.

---

# 8. L’événement `submit`

En JavaScript, on peut écouter la soumission du formulaire avec l’événement `submit`.

Exemple :

```html
<form id="contactForm">
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <button type="submit">Envoyer</button>
</form>
```

```javascript
const form = document.querySelector("#contactForm");

form.addEventListener("submit", function (event) {
  console.log("Le formulaire a été soumis.");
});
```

Lorsque l’utilisateur soumet le formulaire, la fonction est exécutée.

---

# 9. Pourquoi écouter `submit` plutôt que `click` ?

Une erreur fréquente consiste à écouter seulement le clic sur le bouton.

Exemple moins recommandé :

```javascript
const button = document.querySelector("button");

button.addEventListener("click", function () {
  console.log("Bouton cliqué");
});
```

Ce code détecte le clic sur le bouton, mais il ne représente pas toute la logique du formulaire.

Un formulaire peut être soumis :

- par un clic sur le bouton ;
- par la touche Entrée ;
- par une action JavaScript ;
- par certains comportements du navigateur.

Il est donc préférable d’écouter le formulaire lui-même :

```javascript
const form = document.querySelector("#contactForm");

form.addEventListener("submit", function (event) {
  console.log("Formulaire soumis");
});
```

L’événement `submit` représente mieux l’action réelle.

---

# 10. Le problème du rechargement de la page

Si on écoute l’événement `submit` sans rien faire d’autre, le navigateur peut quand même exécuter son comportement par défaut.

Exemple :

```javascript
form.addEventListener("submit", function (event) {
  console.log("Formulaire soumis");
});
```

Le message peut apparaître très brièvement dans la console, puis la page peut se recharger.

Pourquoi ?

Parce que le navigateur essaie d’envoyer le formulaire automatiquement.

---

# 11. `event.preventDefault()`

Pour empêcher le comportement par défaut du navigateur, on utilise :

```javascript
event.preventDefault();
```

Exemple :

```javascript
const form = document.querySelector("#contactForm");

form.addEventListener("submit", function (event) {
  event.preventDefault();

  console.log("Le formulaire est contrôlé par JavaScript.");
});
```

Cette ligne signifie :

> Ne fais pas l’action automatique du navigateur. Je vais gérer le formulaire avec JavaScript.

C’est une notion fondamentale.

---

# 12. Exemple complet avec `submit` et `preventDefault()`

## HTML

```html
<form id="contactForm">
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <button type="submit">Envoyer</button>
</form>
```

## JavaScript

```javascript
const form = document.querySelector("#contactForm");

form.addEventListener("submit", function (event) {
  event.preventDefault();

  console.log("Soumission interceptée par JavaScript.");
});
```

Dans cet exemple :

1. L’utilisateur soumet le formulaire.
2. L’événement `submit` se déclenche.
3. JavaScript reçoit l’événement.
4. `event.preventDefault()` empêche le rechargement.
5. JavaScript peut maintenant récupérer les données.

---

# 13. Le rôle de l’objet `event`

Dans cette fonction :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();
});
```

le paramètre `event` représente l’événement qui vient de se produire.

On peut l’appeler `event`, `e` ou autrement, mais `event` est plus clair pour débuter.

Exemple :

```javascript
form.addEventListener("submit", function (e) {
  e.preventDefault();
});
```

Les deux versions fonctionnent, mais celle-ci est plus explicite :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();
});
```

---

# 14. `event.target` et `event.currentTarget`

Dans un événement, deux propriétés peuvent être utiles :

```javascript
event.target;
event.currentTarget;
```

Dans le cas d’un formulaire, on utilise souvent :

```javascript
event.currentTarget;
```

Exemple :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();

  console.log(event.currentTarget);
});
```

`event.currentTarget` représente l’élément sur lequel l’écouteur d’événement a été installé.

Ici, c’est le formulaire.

Cela permet d’écrire :

```javascript
const formData = new FormData(event.currentTarget);
```

Cette approche est pratique, car elle évite de dépendre directement du nom de la variable `form`.

# 15. Validation HTML et soumission

Avant de déclencher réellement la soumission, le navigateur peut vérifier certains attributs HTML comme :

```html
required type="email" minlength maxlength min max
```

Exemple :

```html
<form id="signupForm">
  <label for="email">Adresse courriel</label>
  <input type="email" id="email" name="email" required />

  <button type="submit">Envoyer</button>
</form>
```

Si l’utilisateur laisse le champ vide, le navigateur peut bloquer la soumission.

Cela signifie que le code JavaScript lié à `submit` peut ne pas s’exécuter tant que le formulaire n’est pas valide selon les règles HTML du navigateur.

---

# 16. Désactiver la validation HTML automatique avec `novalidate`

Parfois, on veut gérer toute la validation soi-même avec JavaScript.

Dans ce cas, on peut ajouter l’attribut `novalidate` au formulaire.

```html
<form id="signupForm" novalidate>
  <label for="email">Adresse courriel</label>
  <input type="email" id="email" name="email" required />

  <button type="submit">Envoyer</button>
</form>
```

`novalidate` signifie :

> Ne bloque pas automatiquement la soumission avec la validation HTML du navigateur.

Cela ne veut pas dire que les données sont valides. Cela veut seulement dire que JavaScript devra gérer la validation.

# 17. Récupérer les données avec JavaScript

Une fois que JavaScript intercepte la soumission, il peut récupérer les valeurs des champs.

Il existe plusieurs méthodes :

```txt
1. Récupérer les champs un par un avec .value
2. Utiliser .checked pour les checkbox et radio
3. Utiliser FormData
4. Transformer les données en objet JavaScript
```

Nous allons les voir progressivement.

---

# 18. Récupérer une valeur avec `.value`

La propriété `.value` permet de récupérer la valeur d’un champ.

## HTML

```html
<form id="contactForm">
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <button type="submit">Envoyer</button>
</form>
```

## JavaScript

```javascript
const form = document.querySelector("#contactForm");
const nomInput = document.querySelector("#nom");

form.addEventListener("submit", function (event) {
  event.preventDefault();

  const nom = nomInput.value;

  console.log(nom);
});
```

Si l’utilisateur écrit `Sara`, la console affichera :

```txt
Sara
```

---

# 19. Nettoyer une valeur avec `trim()`

Un utilisateur peut écrire des espaces avant ou après une valeur.

Exemple :

```txt
   Sara
```

Avec JavaScript, on peut utiliser `trim()` pour enlever les espaces inutiles au début et à la fin.

```javascript
const nom = nomInput.value.trim();
```

Exemple :

```javascript
const texte = "   Sara   ";
console.log(texte.trim());
```

Résultat :

```txt
Sara
```

Cette méthode est très utile pour les formulaires.

# 20. Exemple avec plusieurs champs

## HTML

```html
<form id="contactForm">
  <div>
    <label for="nom">Nom</label>
    <input type="text" id="nom" name="nom" />
  </div>

  <div>
    <label for="email">Adresse courriel</label>
    <input type="email" id="email" name="email" />
  </div>

  <div>
    <label for="message">Message</label>
    <textarea id="message" name="message"></textarea>
  </div>

  <button type="submit">Envoyer</button>
</form>
```

## JavaScript

```javascript
const form = document.querySelector("#contactForm");

const nomInput = document.querySelector("#nom");
const emailInput = document.querySelector("#email");
const messageInput = document.querySelector("#message");

form.addEventListener("submit", function (event) {
  event.preventDefault();

  const nom = nomInput.value.trim();
  const email = emailInput.value.trim();
  const message = messageInput.value.trim();

  console.log("Nom :", nom);
  console.log("Email :", email);
  console.log("Message :", message);
});
```

Cette méthode est claire pour débuter, car chaque champ est récupéré explicitement.

# 21. Récupérer une checkbox avec `.checked`

Une case à cocher ne se traite pas exactement comme un champ texte.

Exemple HTML :

```html
<label>
  <input type="checkbox" id="conditions" name="conditions" />
  J’accepte les conditions
</label>
```

JavaScript :

```javascript
const conditionsInput = document.querySelector("#conditions");

console.log(conditionsInput.checked);
```

La propriété `.checked` retourne un booléen :

```txt
true   → la case est cochée
false  → la case n’est pas cochée
```

Exemple dans un formulaire :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();

  const conditionsAcceptees = conditionsInput.checked;

  console.log(conditionsAcceptees);
});
```

# 22. Récupérer une option radio sélectionnée

Les boutons radio permettent un seul choix dans un groupe.

## HTML

```html
<p>Mode de participation</p>

<label>
  <input type="radio" name="mode" value="presentiel" />
  Présentiel
</label>

<label>
  <input type="radio" name="mode" value="distance" />
  À distance
</label>

<label>
  <input type="radio" name="mode" value="hybride" />
  Hybride
</label>
```

Pour récupérer l’option sélectionnée, on peut utiliser :

```javascript
const modeSelectionne = document.querySelector("input[name='mode']:checked");
```

Exemple complet :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();

  const modeSelectionne = document.querySelector("input[name='mode']:checked");

  if (modeSelectionne !== null) {
    console.log(modeSelectionne.value);
  } else {
    console.log("Aucun mode sélectionné.");
  }
});
```

Si l’utilisateur sélectionne “À distance”, la valeur sera :

```txt
distance
```

# 23. Récupérer plusieurs checkbox cochées

Pour plusieurs cases à cocher, il faut récupérer toutes les cases sélectionnées.

## HTML

```html
<p>Technologies connues</p>

<label>
  <input type="checkbox" name="technologies" value="html" />
  HTML
</label>

<label>
  <input type="checkbox" name="technologies" value="css" />
  CSS
</label>

<label>
  <input type="checkbox" name="technologies" value="javascript" />
  JavaScript
</label>
```

## JavaScript

```javascript
const technologiesSelectionnees = document.querySelectorAll(
  "input[name='technologies']:checked",
);

technologiesSelectionnees.forEach(function (checkbox) {
  console.log(checkbox.value);
});
```

Pour obtenir un tableau de valeurs :

```javascript
const technologies = Array.from(technologiesSelectionnees).map(
  function (checkbox) {
    return checkbox.value;
  },
);

console.log(technologies);
```

Résultat possible :

```javascript
["html", "css", "javascript"];
```

# 24. Récupérer les données avec `FormData`

Lorsque le formulaire contient plusieurs champs, récupérer chaque valeur une par une peut devenir long.

JavaScript propose un outil pratique : `FormData`.

Exemple HTML :

```html
<form id="contactForm">
  <input type="text" name="nom" />
  <input type="email" name="email" />
  <textarea name="message"></textarea>

  <button type="submit">Envoyer</button>
</form>
```

JavaScript :

```javascript
const form = document.querySelector("#contactForm");

form.addEventListener("submit", function (event) {
  event.preventDefault();

  const formData = new FormData(form);

  const nom = formData.get("nom");
  const email = formData.get("email");
  const message = formData.get("message");

  console.log(nom);
  console.log(email);
  console.log(message);
});
```

`FormData` utilise les attributs `name`.

Donc ce champ :

```html
<input type="email" name="email" />
```

se récupère avec :

```javascript
formData.get("email");
```

---

# 25. Point important : `FormData` utilise `name`, pas `id`

Cette distinction est essentielle.

Exemple :

```html
<label for="courriel">Adresse courriel</label>
<input type="email" id="courriel" name="email" />
```

Avec JavaScript :

```javascript
const formData = new FormData(form);
const email = formData.get("email");
```

On utilise :

```javascript
formData.get("email");
```

et non :

```javascript
formData.get("courriel");
```

Pourquoi ?

Parce que `email` est le `name`, tandis que `courriel` est le `id`.

# 26. Transformer `FormData` en objet JavaScript

Il est souvent pratique de transformer les données du formulaire en objet.

```javascript
const formData = new FormData(form);
const donnees = Object.fromEntries(formData.entries());

console.log(donnees);
```

Exemple de résultat :

```javascript
{
  nom: "Sara",
  email: "sara@email.com",
  message: "Bonjour"
}
```

Cette forme est très pratique pour :

- afficher un résumé ;
- envoyer les données à une API ;
- valider les champs ;
- manipuler les données dans JavaScript.

# 28. Limite de `Object.fromEntries()` avec plusieurs checkbox

Attention : si plusieurs checkbox partagent le même `name`, `Object.fromEntries()` ne conserve pas toujours toutes les valeurs.

Exemple :

```html
<label>
  <input type="checkbox" name="technologies" value="html" />
  HTML
</label>

<label>
  <input type="checkbox" name="technologies" value="css" />
  CSS
</label>

<label>
  <input type="checkbox" name="technologies" value="javascript" />
  JavaScript
</label>
```

Si plusieurs cases sont cochées, il faut utiliser :

```javascript
formData.getAll("technologies");
```

Exemple :

```javascript
const formData = new FormData(form);
const technologies = formData.getAll("technologies");

console.log(technologies);
```

Résultat possible :

```javascript
["html", "css", "javascript"];
```

Pour construire un objet plus complet :

```javascript
const donnees = Object.fromEntries(formData.entries());

donnees.technologies = formData.getAll("technologies");

console.log(donnees);
```

# 30. Afficher les données dans la page

Récupérer les données ne sert pas seulement à les afficher dans la console.

On peut aussi afficher un résumé dans la page.

## HTML

```html
<form id="contactForm">
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <label for="email">Adresse courriel</label>
  <input type="email" id="email" name="email" />

  <button type="submit">Envoyer</button>
</form>

<div id="resume"></div>
```

## JavaScript

```javascript
const form = document.querySelector("#contactForm");
const resume = document.querySelector("#resume");

form.addEventListener("submit", function (event) {
  event.preventDefault();

  const formData = new FormData(form);
  const donnees = Object.fromEntries(formData.entries());

  resume.textContent = `Nom : ${donnees.nom}, Email : ${donnees.email}`;
});
```

Ici, les données sont affichées dans la page sans rechargement.

# 31. Pourquoi utiliser `textContent` plutôt que `innerHTML` ?

Quand on affiche une donnée écrite par l’utilisateur, il faut être prudent.

Exemple à éviter :

```javascript
resume.innerHTML = donnees.nom;
```

Si on insère directement du contenu utilisateur avec `innerHTML`, on peut créer des risques de sécurité.

Meilleure pratique :

```javascript
resume.textContent = donnees.nom;
```

`textContent` affiche le texte sans l’interpréter comme du HTML.

Il faut retenir :

```txt
Donnée utilisateur simple → textContent
HTML contrôlé par le développeur → innerHTML avec prudence
```

# 32. Réinitialiser un formulaire avec `reset()`

Après une soumission réussie, on peut vider le formulaire.

```javascript
form.reset();
```

Exemple :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();

  const formData = new FormData(form);
  const donnees = Object.fromEntries(formData.entries());

  console.log(donnees);

  form.reset();
});
```

`reset()` remet les champs à leur valeur initiale.

Si un champ avait une valeur définie dans le HTML avec `value`, cette valeur peut revenir après le reset.

# 33. Comprendre GET et POST

Lorsqu’un formulaire est soumis de manière classique, le navigateur utilise une méthode HTTP.

Les deux méthodes les plus importantes pour les formulaires sont :

```txt
GET
POST
```

Elles servent toutes les deux à transmettre des données, mais elles ne le font pas de la même manière.

---

# 34. Le rôle de l’attribut `method`

L’attribut `method` indique comment les données sont envoyées.

Exemple :

```html
<form action="/recherche" method="GET"></form>
```

ou :

```html
<form action="/contact" method="POST"></form>
```

La méthode doit être choisie selon l’intention du formulaire.

---

# 35. Le rôle de l’attribut `action`

L’attribut `action` indique où les données doivent être envoyées.

Exemple :

```html
<form action="/contact" method="POST"></form>
```

Cela signifie :

```txt
Envoyer les données vers /contact
en utilisant la méthode POST
```

Autre exemple :

```html
<form action="/recherche" method="GET"></form>
```

Cela signifie :

```txt
Envoyer les données vers /recherche
en utilisant la méthode GET
```

---

# 36. La méthode GET

La méthode GET sert généralement à demander ou rechercher de l’information.

Elle est souvent utilisée pour :

- une recherche ;
- un filtre ;
- un tri ;
- une consultation ;
- une navigation avec paramètres.

Exemple :

```html
<form action="/recherche" method="GET">
  <label for="mot">Mot-clé</label>
  <input type="search" id="mot" name="mot" />

  <button type="submit">Rechercher</button>
</form>
```

Si l’utilisateur écrit :

```txt
javascript
```

l’URL peut devenir :

```txt
/recherche?mot=javascript
```

---

# 37. Comprendre les paramètres dans une URL

Une URL avec GET peut contenir des paramètres.

Exemple :

```txt
/recherche?mot=javascript&niveau=debutant
```

On peut la lire ainsi :

```txt
/recherche
```

C’est la destination.

```txt
?
```

indique le début des paramètres.

```txt
mot=javascript
```

est un premier paramètre.

```txt
&
```

sépare deux paramètres.

```txt
niveau=debutant
```

est un deuxième paramètre.

## HTML

```html
<form action="/recherche" method="GET">
  <div>
    <label for="mot">Mot-clé</label>
    <input type="search" id="mot" name="mot" />
  </div>

  <div>
    <label for="niveau">Niveau</label>
    <select id="niveau" name="niveau">
      <option value="">Tous les niveaux</option>
      <option value="debutant">Débutant</option>

      <option value="avance">Avancé</option>
    </select>
  </div>
```

Si l’utilisateur entre :

```txt
mot = formulaire

```

l’URL peut devenir :

```txt
/recherche?mot=formulaire&niveau=debutant
```

---

# 39. Avantages de GET

GET est utile lorsque les données représentent une recherche ou une demande d’information.

Il permet :

- de voir les paramètres dans l’URL ;
- de partager un lien ;
- d’enregistrer une recherche en favori ;
- de revenir à une recherche avec l’historique du navigateur ;
- de créer des filtres visibles dans l’adresse de la page.

Exemple :

```txt
/recherche?mot=javascript&niveau=debutant
```

Cette URL peut être copiée et envoyée à quelqu’un.

---

# 40. Limites de GET

GET ne doit pas être utilisé pour des données sensibles.

À éviter :

```txt
/connexion?email=sara@email.com&password=123456
```

Les données apparaissent dans l’URL.

Elles peuvent être visibles :

- dans la barre d’adresse ;
- dans l’historique du navigateur ;
- dans certains journaux de serveur ;
- dans un lien partagé par erreur.

Il ne faut donc pas utiliser GET pour :

- un mot de passe ;
- des informations confidentielles ;
- des données personnelles sensibles ;
- des actions qui modifient réellement des données.

---

# 41. La méthode POST

La méthode POST sert généralement à envoyer des données pour créer, modifier ou transmettre quelque chose.

Elle est souvent utilisée pour :

- un formulaire de contact ;
- une inscription ;
- une connexion ;
- l’ajout d’un commentaire ;
- la création d’un compte ;
- l’envoi d’une commande ;
- la modification d’un profil.

Exemple :

```html
<form action="/contact" method="POST">
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <label for="message">Message</label>
  <textarea id="message" name="message"></textarea>

  <button type="submit">Envoyer</button>
</form>
```

Avec POST, les données ne sont pas ajoutées directement dans l’URL.

Elles sont envoyées dans le corps de la requête.

---

# 42. POST est-il sécurisé ?

POST est plus approprié que GET pour envoyer des données comme un formulaire de contact ou de connexion, mais POST ne rend pas automatiquement les données sécurisées.

Il faut distinguer deux choses :

```txt
POST  → n’affiche pas les données dans l’URL
HTTPS → protège la communication entre le navigateur et le serveur
```

Pour une vraie application, les données sensibles doivent être envoyées avec HTTPS.

À retenir :

```txt
POST cache les données de l’URL.
HTTPS protège les données pendant le transport.
```

---

# 43. Comparaison entre GET et POST

| Élément                         | GET                    | POST                            |
| ------------------------------- | ---------------------- | ------------------------------- |
| Intention principale            | Demander ou rechercher | Envoyer ou transmettre          |
| Emplacement des données         | Dans l’URL             | Dans le corps de la requête     |
| Visible dans la barre d’adresse | Oui                    | Non                             |
| Peut être partagé par lien      | Oui                    | Non directement                 |
| Exemple typique                 | Recherche              | Formulaire de contact           |
| Données sensibles               | À éviter               | Plus approprié, avec HTTPS      |
| Effet attendu                   | Consultation           | Création, envoi ou modification |

## Utiliser POST lorsque l’utilisateur envoie une information

Exemples :

```txt
J’envoie un message.
J’ajoute un commentaire.
Je modifie mon profil.
```

Formulaire typique :


# 47. Soumission classique ou soumission avec JavaScript

Il y a deux grandes façons de gérer un formulaire.

## 47.1 Soumission HTML classique

Exemple :

```html
<form action="/contact" method="POST">
  <input type="text" name="nom" />
  <button type="submit">Envoyer</button>
</form>
```

Dans ce cas :

1. L’utilisateur soumet le formulaire.
2. Le navigateur envoie les données.
3. La page peut changer ou se recharger.
4. Le serveur reçoit les données.
5. Le serveur répond.

C’est le fonctionnement traditionnel.

---

## 47.2 Soumission contrôlée par JavaScript

Exemple :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();

  const formData = new FormData(form);
  const donnees = Object.fromEntries(formData.entries());

  console.log(donnees);
});
```

Dans ce cas :

1. L’utilisateur soumet le formulaire.
2. JavaScript intercepte l’événement.
3. `preventDefault()` bloque l’envoi automatique.
4. JavaScript récupère les données.
5. JavaScript décide quoi faire ensuite.

Cette approche est fréquente dans les applications modernes.

---

# 48. Envoyer des données avec `fetch`

Une fois les données récupérées avec JavaScript, on peut les envoyer à une API avec `fetch`.

Exemple :

```javascript
const response = await fetch("/contact", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
  },
  body: JSON.stringify(donnees),
});
```

Ce code signifie :

```txt
Envoyer une requête vers /contact
avec la méthode POST
en indiquant que les données sont en JSON
et en plaçant les données dans le corps de la requête
```

---

# 49. Pourquoi utiliser `JSON.stringify()` ?

JavaScript manipule des objets.

Exemple :

```javascript
const donnees = {
  nom: "Sara",
  email: "sara@email.com",
};
```

Mais lorsqu’on envoie ces données à une API, il faut souvent les transformer en JSON.

```javascript
JSON.stringify(donnees);
```

Résultat envoyé :

```json
{
  "nom": "Sara",
  "email": "sara@email.com"
}
```

`JSON.stringify()` transforme un objet JavaScript en texte JSON.

# 50. Exemple complet : formulaire + récupération + envoi POST avec `fetch`

## HTML

```html
<form id="contactForm">
  <div>
    <label for="nom">Nom</label>
    <input type="text" id="nom" name="nom" required />
  </div>

  <div>
    <label for="email">Adresse courriel</label>
    <input type="email" id="email" name="email" required />
  </div>

  <div>

  </div>

  <button type="submit">Envoyer</button>
</form>
```

## JavaScript

```javascript

```

# 51. Lire la réponse du serveur

Quand on utilise `fetch`, le serveur peut répondre.

Si le serveur répond avec du JSON, on peut lire la réponse avec :

```javascript

```

Exemple :

```javascript
const data = await response.json();

console.log(data.message);
```

Si le serveur renvoie :

```json

  "message": "Votre message a bien été reçu."
}
```

alors :

```javascript
data.message;
```

contient :

```txt
Votre message a bien été reçu.
```

# 52. Vérifier si la réponse est correcte

Dans une vraie application, il est utile de vérifier si la requête a réussi.

```javascript
if (!response.ok) {
  throw new Error("Erreur lors de l’envoi du formulaire.");
}
```

Exemple plus complet :

```javascript
const response = await fetch("/contact", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
  },
  body: JSON.stringify(donnees),
});

if (!response.ok) {
  throw new Error("Erreur lors de l’envoi du formulaire.");
}

const data = await response.json();
```

Cela permet de mieux gérer les erreurs.

Cette version est plus professionnelle, car elle tient compte des erreurs possibles.

# 54. Récupérer les données côté serveur avec Express

Pour comprendre GET et POST complètement, il faut aussi savoir où arrivent les données.

Dans une application JavaScript avec Node.js et Express, on peut récupérer les données côté serveur.

---

## 54.1 Récupérer des données GET avec Express

Formulaire HTML :

```html
<form action="/recherche" method="GET">
  <input type="search" name="mot" />
  <button type="submit">Rechercher</button>
</form>
```

Si l’utilisateur cherche `javascript`, l’URL devient :

```txt
/recherche?mot=javascript
```

Côté serveur Express :

```javascript
app.get("/recherche", function (req, res) {
  const mot = req.query.mot;

  console.log("Recherche :", mot);

  res.send(`Recherche pour : ${mot}`);
});
```

Avec GET, les données se trouvent dans :

```javascript
req.query;
```

---

## 54.2 Récupérer des données POST d’un formulaire HTML classique

Formulaire HTML :

```html
<form action="/contact" method="POST">
  <input type="text" name="nom" />
  <input type="email" name="email" />
  <button type="submit">Envoyer</button>
</form>
```

Côté serveur Express :

```javascript
app.use(express.urlencoded({ extended: true }));

app.post("/contact", function (req, res) {
  const nom = req.body.nom;
  const email = req.body.email;

  console.log("Nom :", nom);
  console.log("Email :", email);

  res.send("Formulaire reçu.");
});
```

La ligne suivante est importante :

```javascript
app.use(express.urlencoded({ extended: true }));
```

Elle permet à Express de comprendre les données envoyées par un formulaire HTML classique.

Avec POST, les données se trouvent généralement dans :

```javascript
req.body;
```

---

## 54.3 Récupérer des données POST envoyées en JSON avec `fetch`

Si le navigateur envoie les données ainsi :

```javascript
fetch("/contact", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
  },
  body: JSON.stringify(donnees),
});
```

alors Express doit utiliser :

```javascript
app.use(express.json());
```

Exemple :

```javascript
app.use(express.json());

app.post("/contact", function (req, res) {
  const nom = req.body.nom;
  const email = req.body.email;
  const message = req.body.message;

  console.log(nom);
  console.log(email);
  console.log(message);

  res.json({
    message: "Votre message a bien été reçu.",
  });
});
```

---

# 55. Différence entre `express.urlencoded()` et `express.json()`

Ces deux lignes ne servent pas exactement à la même chose.

## `express.urlencoded()`

```javascript
app.use(express.urlencoded({ extended: true }));
```

Sert à lire les données envoyées par un formulaire HTML classique.

Exemple :

```html
<form action="/contact" method="POST"></form>
```

---

## `express.json()`

```javascript
app.use(express.json());
```

Sert à lire les données JSON envoyées avec JavaScript, souvent avec `fetch`.

Exemple :

```javascript
fetch("/contact", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
  },
  body: JSON.stringify(donnees),
});
```

# 56. Résumé technique : où récupérer les données ?

| Situation                              | Où sont les données ?                  |
| -------------------------------------- | -------------------------------------- |
| Formulaire GET côté navigateur         | Dans l’URL                             |
| Formulaire GET côté JavaScript         | `URLSearchParams`                      |
| Formulaire GET côté Express            | `req.query`                            |
| Formulaire POST classique côté Express | `req.body` avec `express.urlencoded()` |
| POST JSON avec `fetch` côté Express    | `req.body` avec `express.json()`       |
| Formulaire intercepté côté navigateur  | `FormData`, `.value`, `.checked`       |

# 58. Erreurs fréquentes à éviter

## Erreur 1 : oublier `event.preventDefault()`

Erreur :

```javascript
form.addEventListener("submit", function (event) {
  console.log("Formulaire soumis");
});
```

Correction :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();

  console.log("Formulaire soumis");
});
```

Sans `preventDefault()`, la page peut se recharger.

---

## Erreur 2 : écouter le bouton au lieu du formulaire

Moins recommandé :

```javascript
button.addEventListener("click", function () {
  console.log("Bouton cliqué");
});
```

Meilleure approche :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();

  console.log("Formulaire soumis");
});
```

Le formulaire peut être soumis autrement que par un clic.

---

## Erreur 3 : oublier le `name`

Erreur :

```html
<input type="email" id="email" />
```

Correction :

```html
<input type="email" id="email" name="email" />
```

Sans `name`, `FormData` ne pourra pas récupérer correctement la donnée.

---

## Erreur 4 : confondre `id` et `name`

HTML :

```html
<input type="email" id="courriel" name="email" />
```

Erreur :

```javascript
formData.get("courriel");
```

Correction :

```javascript
formData.get("email");
```

`FormData` utilise le `name`.

---

## Erreur 5 : utiliser `.value` pour une checkbox

Erreur :

```javascript
const conditions = conditionsInput.value;
```

Cela ne dit pas si la case est cochée.

Correction :

```javascript
const conditions = conditionsInput.checked;
```

`.checked` indique `true` ou `false`.

---

## Erreur 6 : oublier `getAll()` pour plusieurs checkbox

Erreur :

```javascript
const technologies = formData.get("technologies");
```

Cette méthode récupère une seule valeur.

Correction :

```javascript
const technologies = formData.getAll("technologies");
```

`getAll()` récupère toutes les valeurs associées au même `name`.

---

## Erreur 7 : utiliser GET pour des données sensibles

À éviter :

```html
<form action="/connexion" method="GET"></form>
```

Meilleur choix :

```html
<form action="/connexion" method="POST"></form>
```

Et dans une vraie application, il faut utiliser HTTPS.

---

## Erreur 8 : croire que POST suffit pour sécuriser les données

POST n’affiche pas les données dans l’URL, mais il ne chiffre pas automatiquement la communication.

À retenir :

```txt
POST n’est pas une protection complète.
HTTPS est nécessaire pour protéger les données pendant leur transport.
```

---

## Erreur 9 : utiliser `innerHTML` avec une donnée utilisateur

À éviter :

```javascript
resume.innerHTML = donnees.nom;
```

Meilleur choix :

```javascript
resume.textContent = donnees.nom;
```

# 59. Ce qu’il faut absolument retenir

## Sur la soumission

Un formulaire est soumis lorsqu’un utilisateur déclenche son envoi.

La meilleure manière d’écouter cette action en JavaScript est :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();
});
```

Le bouton de soumission est :

```html
<button type="submit">Envoyer</button>
```

Il faut éviter les boutons sans type dans un formulaire.

---

## Sur le comportement par défaut

Par défaut, le navigateur essaie d’envoyer le formulaire.

Cela peut provoquer :

- un rechargement de page ;
- un changement d’URL ;
- un envoi vers l’adresse indiquée dans `action`.

Pour contrôler le formulaire avec JavaScript, on utilise :

```javascript
event.preventDefault();
```

---

## Sur la récupération des données

Pour récupérer une valeur simple :

```javascript
input.value;
```

Pour vérifier une checkbox :

```javascript
checkbox.checked;
```

Pour récupérer un formulaire complet :

```javascript
const formData = new FormData(form);
```

Pour transformer les données en objet :

```javascript
const donnees = Object.fromEntries(formData.entries());
```

Pour récupérer plusieurs checkbox :

```javascript
formData.getAll("technologies");
```

---

## Sur GET

GET sert surtout à demander ou rechercher des informations.

Les données sont visibles dans l’URL.

Exemple :

```txt
/recherche?mot=javascript
```

On peut les lire avec :

```javascript
const params = new URLSearchParams(window.location.search);
```

Côté Express, on les récupère avec :

```javascript
req.query;
```

---

## Sur POST

POST sert surtout à envoyer des informations.

Les données ne sont pas visibles dans l’URL.

Côté Express, on les récupère avec :

```javascript
req.body;
```

avec :

```javascript
app.use(express.urlencoded({ extended: true }));
```

pour un formulaire HTML classique, ou :

```javascript
app.use(express.json());
```

pour des données JSON envoyées avec `fetch`.

# Conclusion

La soumission d’un formulaire est une étape centrale dans le développement web. Elle marque le moment où les données saisies par l’utilisateur deviennent exploitables par la page, par JavaScript ou par un serveur.

Le navigateur possède un comportement par défaut : il essaie d’envoyer le formulaire. JavaScript permet d’intercepter ce comportement avec `event.preventDefault()`, puis de récupérer les données avec `.value`, `.checked` ou `FormData`.

GET et POST permettent ensuite de comprendre comment les données circulent. GET est adapté aux recherches et aux filtres, car les données apparaissent dans l’URL. POST est adapté aux envois de données, aux inscriptions, aux connexions et aux formulaires de contact, car les données sont envoyées dans le corps de la requête.

La logique à retenir est la suivante :

```txt
L’utilisateur remplit le formulaire.
Le formulaire est soumis.
Le navigateur déclenche l’événement submit.
JavaScript peut intercepter la soumission.
Les données sont récupérées.
Les données peuvent être affichées, validées ou envoyées.
GET place les données dans l’URL.
POST place les données dans le corps de la requête.
```
