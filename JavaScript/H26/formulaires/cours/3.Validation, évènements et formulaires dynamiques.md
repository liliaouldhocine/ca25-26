# Support de cours

# Validation des formulaires, messages d’erreur, événements et formulaires dynamiques

## Présentation générale

Dans les cours précédents, nous avons vu comment construire un formulaire HTML, comment le soumettre, comment empêcher le comportement par défaut du navigateur et comment récupérer les données avec JavaScript.

Nous allons maintenant aborder une étape essentielle : **la validation et l’interaction**.

Un formulaire ne doit pas seulement récupérer des données. Il doit aussi aider l’utilisateur à entrer des informations correctes, claires et complètes.

Dans ce cours, nous allons étudier quatre grands sujets :

```txt
1. La validation HTML et la validation JavaScript
2. Les messages d’erreur et l’expérience utilisateur
3. Les événements des formulaires : input, change, focus, blur
4. Les formulaires dynamiques
```

L’objectif est de comprendre comment rendre un formulaire plus fiable, plus clair et plus agréable à utiliser.

---

# 1. Pourquoi valider un formulaire ?

Valider un formulaire signifie vérifier que les données entrées par l’utilisateur respectent certaines règles.

Par exemple :

- le nom ne doit pas être vide ;
- l’adresse courriel doit avoir un format valide ;
- le mot de passe doit être assez long ;
- l’âge doit être dans une plage acceptable ;
- une case obligatoire doit être cochée ;
- une option doit être sélectionnée ;
- un message doit contenir un minimum de caractères.

Sans validation, un formulaire peut recevoir des données incomplètes, incorrectes ou inutilisables.

Exemple :

```txt
Nom :
Email : bonjour
Âge : -5
Message :
```

Ces données posent problème. Le programme ne peut pas les traiter correctement.

La validation sert donc à protéger la qualité des données et à guider l’utilisateur.

---

# 2. Les deux niveaux principaux de validation côté client

Dans une page web, on peut valider un formulaire de deux façons principales :

```txt
Validation HTML
Validation JavaScript
```

La validation HTML utilise directement les attributs des champs.

La validation JavaScript utilise du code pour vérifier les valeurs et afficher des messages personnalisés.

Ces deux approches peuvent être utilisées ensemble.

---

# 3. La validation HTML

La validation HTML est la première forme de validation. Elle est simple, rapide et intégrée au navigateur.

Elle repose sur des attributs comme :

```html
required type="email" minlength maxlength min max pattern
```

Ces attributs donnent des règles au navigateur.

---

## 3.1 L’attribut `required`

L’attribut `required` rend un champ obligatoire.

```html
<label for="nom">Nom</label> <input type="text" id="nom" name="nom" required />
```

Si l’utilisateur essaie de soumettre le formulaire sans remplir ce champ, le navigateur peut bloquer l’envoi.

Exemple complet :

```html
<form>
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" required />

  <button type="submit">Envoyer</button>
</form>
```

Ici, le champ `nom` doit être rempli.

---

## 3.2 Le type `email`

Un champ de type `email` demande une valeur qui ressemble à une adresse courriel.

```html
<label for="email">Adresse courriel</label>
<input type="email" id="email" name="email" required />
```

Si l’utilisateur écrit :

```txt
bonjour
```

le navigateur peut refuser la soumission.

Mais il faut bien comprendre la limite : `type="email"` vérifie seulement la forme générale de l’adresse. Il ne vérifie pas si l’adresse existe réellement.

---

## 3.3 Les attributs `minlength` et `maxlength`

Ces attributs servent à contrôler la longueur d’un texte.

```html
<label for="username">Nom d’utilisateur</label>
<input
  type="text"
  id="username"
  name="username"
  minlength="3"
  maxlength="20"
  required
/>
```

Ici, le nom d’utilisateur doit contenir au moins 3 caractères et au maximum 20 caractères.

---

## 3.4 Les attributs `min` et `max`

Ces attributs servent surtout avec les champs numériques ou les dates.

```html
<label for="age">Âge</label>
<input type="number" id="age" name="age" min="12" max="120" required />
```

Ici, l’âge doit être compris entre 12 et 120.

---

## 3.5 L’attribut `pattern`

L’attribut `pattern` permet d’imposer un format précis à l’aide d’une expression régulière.

Exemple simple pour un code postal canadien :

```html
<label for="codePostal">Code postal</label>
<input
  type="text"
  id="codePostal"
  name="codePostal"
  pattern="[A-Za-z][0-9][A-Za-z][ -]?[0-9][A-Za-z][0-9]"
/>
```

Cet attribut est puissant, mais il peut être difficile pour les débutants. Il faut l’introduire progressivement.

Dans un premier temps, il est souvent préférable d’utiliser des règles simples avec JavaScript avant d’aller trop loin dans les expressions régulières.

---

# 4. Exemple de formulaire avec validation HTML

```html
<form>
  <div>
    <label for="prenom">Prénom</label>
    <input type="text" id="prenom" name="prenom" minlength="2" required />
  </div>

  <div>
    <label for="email">Adresse courriel</label>
    <input type="email" id="email" name="email" required />
  </div>

  <div>
    <label for="age">Âge</label>
    <input type="number" id="age" name="age" min="12" max="120" required />
  </div>

  <button type="submit">Envoyer</button>
</form>
```

Dans cet exemple, le navigateur vérifie déjà plusieurs choses :

- le prénom est obligatoire ;
- le prénom doit contenir au moins 2 caractères ;
- le courriel est obligatoire ;
- le courriel doit avoir un format valide ;
- l’âge est obligatoire ;
- l’âge doit être entre 12 et 120.

---

# 5. Avantages de la validation HTML

La validation HTML est utile parce qu’elle est :

- simple à écrire ;
- directement comprise par le navigateur ;
- rapide à mettre en place ;
- adaptée aux règles simples ;
- pratique pour les débutants.

Exemple :

```html
<input type="email" required />
```

Une seule ligne permet déjà de demander un courriel obligatoire.

---

# 6. Limites de la validation HTML

La validation HTML ne suffit pas toujours.

Elle a plusieurs limites :

- les messages d’erreur dépendent du navigateur ;
- les messages ne sont pas toujours personnalisés ;
- les règles complexes sont plus difficiles à écrire ;
- l’apparence des erreurs est moins contrôlable ;
- un utilisateur peut modifier le HTML dans les outils du navigateur ;
- elle ne remplace jamais une validation côté serveur.

Il faut retenir ceci :

```txt
La validation HTML améliore l’expérience utilisateur.
Elle ne sécurise pas complètement une application.
```

Dans une vraie application, il faut toujours valider les données côté serveur également.

---

# 7. La validation JavaScript

La validation JavaScript permet de contrôler plus précisément les règles et les messages.

Elle permet de :

- récupérer les valeurs du formulaire ;
- vérifier les champs un par un ;
- afficher des messages personnalisés ;
- empêcher l’envoi si les données sont invalides ;
- ajouter ou retirer des classes CSS ;
- réagir pendant la saisie ;
- gérer des règles plus complexes.

Exemple simple :

```javascript
if (nom === "") {
  console.log("Le nom est obligatoire.");
}
```

Avec JavaScript, nous pouvons décider exactement quoi vérifier et quoi afficher.

---

# 8. Structure de base d’une validation JavaScript

Une validation JavaScript suit souvent cette logique :

```txt
1. Écouter la soumission du formulaire
2. Empêcher le comportement par défaut
3. Récupérer les valeurs
4. Nettoyer les valeurs avec trim()
5. Vérifier les règles
6. Afficher les messages d’erreur
7. Si tout est valide, traiter les données
```

Exemple de structure :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();

  // 1. Récupérer les valeurs
  // 2. Vérifier les erreurs
  // 3. Afficher les messages
  // 4. Continuer seulement si le formulaire est valide
});
```

---

# 9. Exemple simple de validation JavaScript

## HTML

```html
<form id="contactForm" novalidate>
  <div>
    <label for="nom">Nom</label>
    <input type="text" id="nom" name="nom" />
    <p id="nomError" class="error"></p>
  </div>

  <div>
    <label for="email">Adresse courriel</label>
    <input type="email" id="email" name="email" />
    <p id="emailError" class="error"></p>
  </div>

  <button type="submit">Envoyer</button>
</form>
```

## JavaScript

```javascript
const form = document.querySelector("#contactForm");

const nomInput = document.querySelector("#nom");
const emailInput = document.querySelector("#email");

const nomError = document.querySelector("#nomError");
const emailError = document.querySelector("#emailError");

form.addEventListener("submit", function (event) {
  event.preventDefault();

  nomError.textContent = "";
  emailError.textContent = "";

  const nom = nomInput.value.trim();
  const email = emailInput.value.trim();

  let formulaireValide = true;

  if (nom === "") {
    nomError.textContent = "Le nom est obligatoire.";
    formulaireValide = false;
  }

  if (email === "") {
    emailError.textContent = "L’adresse courriel est obligatoire.";
    formulaireValide = false;
  }

  if (!formulaireValide) {
    return;
  }

  console.log("Formulaire valide.");
});
```

---

# 10. Le rôle de `novalidate`

Dans l’exemple précédent, le formulaire contient :

```html
<form id="contactForm" novalidate></form>
```

L’attribut `novalidate` désactive la validation HTML automatique du navigateur.

Cela permet de laisser JavaScript gérer les messages d’erreur.

Sans `novalidate`, le navigateur peut bloquer la soumission avant même que notre code JavaScript affiche ses propres messages.

`novalidate` ne signifie pas que le formulaire n’a pas besoin d’être validé. Il signifie simplement :

```txt
Je ne veux pas utiliser les messages automatiques du navigateur.
Je vais gérer la validation moi-même avec JavaScript.
```

---

# 11. Les messages d’erreur

Un message d’erreur sert à expliquer à l’utilisateur ce qui doit être corrigé.

Un bon message d’erreur doit être :

- clair ;
- précis ;
- visible ;
- placé près du champ concerné ;
- formulé de manière respectueuse ;
- utile pour corriger le problème.

---

## 11.1 Exemple de mauvais message

```txt
Erreur.
```

Ce message est trop vague.

L’utilisateur ne sait pas ce qu’il doit corriger.

---

## 11.2 Exemple de meilleur message

```txt
Le nom est obligatoire.
```

Ce message indique clairement le problème.

---

## 11.3 Exemple encore plus utile

```txt
Veuillez entrer votre nom complet.
```

Ce message indique ce que l’utilisateur doit faire.

---

# 12. Placer les messages d’erreur dans le HTML

Il est recommandé de prévoir un emplacement pour chaque message d’erreur.

```html
<div>
  <label for="email">Adresse courriel</label>
  <input type="email" id="email" name="email" />
  <p id="emailError" class="error"></p>
</div>
```

Le paragraphe vide servira à afficher l’erreur si nécessaire.

```javascript
emailError.textContent = "L’adresse courriel est obligatoire.";
```

Si le champ est valide, on peut vider le message :

```javascript
emailError.textContent = "";
```

---

# 13. Styliser les messages d’erreur avec CSS

Exemple :

```css
.error {
  color: red;
  font-size: 0.9rem;
  margin-top: 4px;
}
```

On peut aussi ajouter une bordure au champ invalide :

```css
.input-error {
  border: 2px solid red;
}
```

Puis avec JavaScript :

```javascript
nomInput.classList.add("input-error");
```

Et pour retirer l’erreur :

```javascript
nomInput.classList.remove("input-error");
```

---

# 14. Ajouter et retirer des classes CSS selon l’erreur

## HTML

```html
<input type="text" id="nom" name="nom" />
<p id="nomError" class="error"></p>
```

## CSS

```css
.error {
  color: red;
}

.input-error {
  border: 2px solid red;
}

.input-success {
  border: 2px solid green;
}
```

## JavaScript

```javascript
if (nom === "") {
  nomError.textContent = "Le nom est obligatoire.";
  nomInput.classList.add("input-error");
  nomInput.classList.remove("input-success");
  formulaireValide = false;
} else {
  nomError.textContent = "";
  nomInput.classList.remove("input-error");
  nomInput.classList.add("input-success");
}
```

Ce code rend le formulaire plus visuel.

L’utilisateur voit immédiatement quels champs posent problème.

# 15. Expérience utilisateur et validation

La validation ne doit pas seulement bloquer l’utilisateur. Elle doit l’aider.

Un formulaire bien conçu doit répondre à ces questions :

```txt
L’utilisateur comprend-il ce qu’il doit remplir ?
Sait-il quels champs sont obligatoires ?
Comprend-il ce qui ne va pas ?
Sait-il comment corriger son erreur ?
Les messages apparaissent-ils au bon endroit ?
Le formulaire évite-t-il les frustrations inutiles ?
```

La validation fait donc partie de l’expérience utilisateur.

---

# 16. Bonnes pratiques pour les messages d’erreur

Voici quelques règles simples.

## Être précis

À éviter :

```txt
Champ invalide.
```

Mieux :

```txt
Veuillez entrer une adresse courriel valide.
```

---

## Être proche du champ

Le message doit être affiché près du champ concerné.

Mauvaise pratique :

```txt
Toutes les erreurs sont affichées très loin en bas de page.
```

Meilleure pratique :

```html
<label for="email">Adresse courriel</label>
<input type="email" id="email" name="email" />
<p id="emailError" class="error"></p>
```

---

## Ne pas accuser l’utilisateur

À éviter :

```txt
Vous avez mal rempli le formulaire.
```

Mieux :

```txt
Veuillez remplir les champs obligatoires.
```

---

## Ne pas afficher trop tôt des erreurs agressives

Par exemple, afficher une erreur rouge dès que l’utilisateur commence à taper peut être désagréable.

Il est souvent préférable de valider :

- au moment de la soumission ;
- lorsque l’utilisateur quitte le champ ;
- ou après quelques caractères, selon le contexte.

# 17. Les événements des formulaires

Jusqu’ici, nous avons surtout utilisé l’événement :

```javascript
submit;
```

Mais les formulaires peuvent déclencher plusieurs événements utiles.

Les plus importants sont :

```txt
input
change
focus
blur
submit
```

Dans ce cours, nous allons étudier :

```txt
input
change
focus
blur
```

Ces événements permettent de rendre le formulaire plus interactif.

---

# 18. L’événement `input`

L’événement `input` se déclenche chaque fois que la valeur d’un champ change pendant la saisie.

Exemple :

```html
<input type="text" id="prenom" name="prenom" />
```

```javascript
const prenomInput = document.querySelector("#prenom");

prenomInput.addEventListener("input", function () {
  console.log(prenomInput.value);
});
```

Chaque fois que l’utilisateur tape une lettre, le code s’exécute.

---

## 18.1 Utiliser `input` pour afficher un aperçu en direct

## HTML

```html
<label for="prenom">Prénom</label>
<input type="text" id="prenom" name="prenom" />

<p id="apercu"></p>
```

## JavaScript

```javascript
const prenomInput = document.querySelector("#prenom");
const apercu = document.querySelector("#apercu");

prenomInput.addEventListener("input", function () {
  apercu.textContent = `Bonjour, ${prenomInput.value}`;
});
```

Si l’utilisateur écrit `Sara`, la page affiche :

```txt
Bonjour, Sara
```

Ce type d’interaction est utile pour créer des aperçus dynamiques.

---

# 19. L’événement `change`

L’événement `change` se déclenche lorsque la valeur d’un champ est modifiée et confirmée.

Il est très utilisé avec :

- les listes déroulantes ;
- les checkbox ;
- les boutons radio ;
- les champs de date ;
- les champs fichier.

Exemple avec un `<select>` :

```html
<label for="niveau">Niveau</label>

<select id="niveau" name="niveau">
  <option value="">Choisir un niveau</option>
  <option value="debutant">Débutant</option>
  <option value="intermediaire">Intermédiaire</option>
  <option value="avance">Avancé</option>
</select>

<p id="messageNiveau"></p>
```

```javascript
const niveauSelect = document.querySelector("#niveau");
const messageNiveau = document.querySelector("#messageNiveau");

niveauSelect.addEventListener("change", function () {
  messageNiveau.textContent = `Niveau choisi : ${niveauSelect.value}`;
});
```

Lorsque l’utilisateur choisit un niveau, le message se met à jour.

---

# 20. Différence entre `input` et `change`

Ces deux événements se ressemblent, mais ils ne sont pas identiques.

## `input`

Se déclenche pendant que l’utilisateur modifie la valeur.

Exemple : à chaque lettre tapée dans un champ texte.

```javascript
input.addEventListener("input", function () {
  console.log("La valeur change pendant la saisie.");
});
```

---

## `change`

Se déclenche lorsque le changement est confirmé.

Exemple : après avoir choisi une option dans un `<select>`.

```javascript
select.addEventListener("change", function () {
  console.log("Le choix a changé.");
});
```

---

## Résumé simple

```txt
input  → pendant la saisie
change → après un changement confirmé
```

---

# 21. L’événement `focus`

L’événement `focus` se déclenche lorsqu’un champ devient actif.

Un champ devient actif lorsque l’utilisateur clique dedans ou y accède avec le clavier.

Exemple :

```html
<label for="email">Adresse courriel</label>
<input type="email" id="email" name="email" />

<p id="aideEmail"></p>
```

```javascript
const emailInput = document.querySelector("#email");
const aideEmail = document.querySelector("#aideEmail");

emailInput.addEventListener("focus", function () {
  aideEmail.textContent =
    "Entrez une adresse courriel valide, par exemple nom@exemple.com.";
});
```

Quand l’utilisateur entre dans le champ, un message d’aide apparaît.

---

# 22. L’événement `blur`

L’événement `blur` se déclenche lorsqu’un champ perd le focus.

Autrement dit, lorsque l’utilisateur quitte le champ.

Exemple :

```javascript
emailInput.addEventListener("blur", function () {
  aideEmail.textContent = "";
});
```

On peut aussi utiliser `blur` pour valider un champ lorsque l’utilisateur le quitte.

```javascript
emailInput.addEventListener("blur", function () {
  const email = emailInput.value.trim();

  if (email === "") {
    emailError.textContent = "L’adresse courriel est obligatoire.";
  } else {
    emailError.textContent = "";
  }
});
```

---

# 23. Différence entre `focus` et `blur`

```txt
focus → l’utilisateur entre dans le champ
blur  → l’utilisateur quitte le champ
```

Exemple concret :

```javascript
input.addEventListener("focus", function () {
  console.log("Le champ est actif.");
});

input.addEventListener("blur", function () {
  console.log("Le champ n’est plus actif.");
});
```

Ces événements permettent d’accompagner l’utilisateur pendant sa saisie.

---

# 24. Exemple complet avec `input`, `focus` et `blur`

## HTML

```html
<form id="signupForm" novalidate>
  <div>
    <label for="username">Nom d’utilisateur</label>
    <input type="text" id="username" name="username" />
    <p id="usernameHelp" class="help"></p>
    <p id="usernameError" class="error"></p>
  </div>

  <button type="submit">Créer le compte</button>
</form>
```

## CSS

```css
.error {
  color: red;
}

.help {
  color: #555;
}
```

## JavaScript

```javascript
const usernameInput = document.querySelector("#username");
const usernameHelp = document.querySelector("#usernameHelp");
const usernameError = document.querySelector("#usernameError");

usernameInput.addEventListener("focus", function () {
  usernameHelp.textContent =
    "Le nom d’utilisateur doit contenir au moins 3 caractères.";
});

usernameInput.addEventListener("input", function () {
  const username = usernameInput.value.trim();

  if (username.length > 0 && username.length < 3) {
    usernameError.textContent = "Le nom d’utilisateur est trop court.";
  } else {
    usernameError.textContent = "";
  }
});

usernameInput.addEventListener("blur", function () {
  usernameHelp.textContent = "";

  const username = usernameInput.value.trim();

  if (username === "") {
    usernameError.textContent = "Le nom d’utilisateur est obligatoire.";
  }
});
```

Dans cet exemple :

- `focus` affiche une aide ;
- `input` vérifie pendant la saisie ;
- `blur` vérifie quand l’utilisateur quitte le champ.

# 25. Les formulaires dynamiques

Un formulaire dynamique est un formulaire qui change selon les actions de l’utilisateur.

Par exemple :

- afficher un champ seulement si une option est sélectionnée ;
- cacher une section inutile ;
- ajouter un champ supplémentaire ;
- activer ou désactiver un bouton ;
- afficher un résumé en direct ;
- modifier les options d’un menu selon un autre choix.

Les formulaires dynamiques permettent de rendre l’interface plus claire.

Au lieu d’afficher tous les champs en même temps, on affiche seulement les champs nécessaires.

---

# 26. Exemple : afficher une section selon un choix

Supposons qu’un utilisateur choisisse un type de compte :

- personnel ;
- entreprise.

Si l’utilisateur choisit “entreprise”, on affiche un champ pour le nom de l’entreprise.

## HTML

```html
<form id="accountForm">
  <div>
    <label for="typeCompte">Type de compte</label>
    <select id="typeCompte" name="typeCompte">
      <option value="">Choisir un type</option>
      <option value="personnel">Personnel</option>
      <option value="entreprise">Entreprise</option>
    </select>
  </div>

  <div id="entrepriseSection" class="hidden">
    <label for="nomEntreprise">Nom de l’entreprise</label>
    <input type="text" id="nomEntreprise" name="nomEntreprise" />
  </div>

  <button type="submit">Envoyer</button>
</form>
```

## CSS

```css
.hidden {
  display: none;
}
```

## JavaScript

```javascript
const typeCompteSelect = document.querySelector("#typeCompte");
const entrepriseSection = document.querySelector("#entrepriseSection");

typeCompteSelect.addEventListener("change", function () {
  if (typeCompteSelect.value === "entreprise") {
    entrepriseSection.classList.remove("hidden");
  } else {
    entrepriseSection.classList.add("hidden");
  }
});
```

Dans cet exemple, le formulaire change selon le choix de l’utilisateur.

---

# 27. Rendre un champ obligatoire dynamiquement

Si la section entreprise est affichée, le champ du nom de l’entreprise devrait peut-être devenir obligatoire.

On peut modifier l’attribut `required` avec JavaScript.

```javascript
const nomEntrepriseInput = document.querySelector("#nomEntreprise");

typeCompteSelect.addEventListener("change", function () {
  if (typeCompteSelect.value === "entreprise") {
    entrepriseSection.classList.remove("hidden");
    nomEntrepriseInput.required = true;
  } else {
    entrepriseSection.classList.add("hidden");
    nomEntrepriseInput.required = false;
    nomEntrepriseInput.value = "";
  }
});
```

Ici :

```javascript
nomEntrepriseInput.required = true;
```

rend le champ obligatoire.

Et :

```javascript
nomEntrepriseInput.required = false;
```

retire l’obligation.

La ligne :

```javascript
nomEntrepriseInput.value = "";
```

vide le champ si la section est cachée.

Cela évite de garder une donnée inutile.

# 28. Exemple : activer ou désactiver un bouton d’envoi

On peut désactiver le bouton tant que certaines conditions ne sont pas remplies.

## HTML

```html
<form id="termsForm">
  <label>
    <input type="checkbox" id="conditions" name="conditions" />
    J’accepte les conditions
  </label>

  <button type="submit" id="submitButton" disabled>Envoyer</button>
</form>
```

## JavaScript

```javascript
const conditionsInput = document.querySelector("#conditions");
const submitButton = document.querySelector("#submitButton");

conditionsInput.addEventListener("change", function () {
  submitButton.disabled = !conditionsInput.checked;
});
```

Si la case est cochée, le bouton devient actif.

Si la case n’est pas cochée, le bouton reste désactivé.

# 29. Exemple : afficher un résumé en direct

Un formulaire peut afficher un aperçu des données pendant la saisie.

## HTML

```html
<form id="profilForm">
  <div>
    <label for="prenom">Prénom</label>
    <input type="text" id="prenom" name="prenom" />
  </div>

  <div>
    <label for="ville">Ville</label>
    <input type="text" id="ville" name="ville" />
  </div>
</form>

<section>
  <h2>Aperçu</h2>
  <p id="apercu"></p>
</section>
```

## JavaScript

```javascript
const prenomInput = document.querySelector("#prenom");
const villeInput = document.querySelector("#ville");
const apercu = document.querySelector("#apercu");

function mettreAJourApercu() {
  const prenom = prenomInput.value.trim();
  const ville = villeInput.value.trim();

  if (prenom === "" && ville === "") {
    apercu.textContent = "";
    return;
  }

  apercu.textContent = `${prenom} habite à ${ville}.`;
}

prenomInput.addEventListener("input", mettreAJourApercu);
villeInput.addEventListener("input", mettreAJourApercu);
```

Ici, la fonction `mettreAJourApercu` est appelée dès que l’utilisateur modifie un champ.

# 30. Exemple : ajouter un champ dynamiquement

Un formulaire peut aussi permettre d’ajouter plusieurs informations du même type.

Exemple : ajouter plusieurs compétences.

## HTML

```html
<form id="skillsForm">
  <div id="skillsContainer">
    <div>
      <label>Compétence</label>
      <input type="text" name="competences" />
    </div>
  </div>

  <button type="button" id="addSkillButton">Ajouter une compétence</button>
  <button type="submit">Envoyer</button>
</form>
```

## JavaScript

```javascript
const skillsContainer = document.querySelector("#skillsContainer");
const addSkillButton = document.querySelector("#addSkillButton");

addSkillButton.addEventListener("click", function () {
  const div = document.createElement("div");

  const label = document.createElement("label");
  label.textContent = "Compétence";

  const input = document.createElement("input");
  input.type = "text";
  input.name = "competences";

  div.appendChild(label);
  div.appendChild(input);

  skillsContainer.appendChild(div);
});
```

Ce code crée un nouveau champ chaque fois que l’utilisateur clique sur le bouton.

Le bouton doit être :

```html
<button type="button"></button>
```

et non :

```html
<button type="submit"></button>
```

Sinon, il soumettrait le formulaire au lieu d’ajouter un champ.

---

# 31. Récupérer les champs ajoutés dynamiquement

Comme plusieurs champs ont le même `name`, on peut utiliser `FormData.getAll()`.

```javascript
const form = document.querySelector("#skillsForm");

form.addEventListener("submit", function (event) {
  event.preventDefault();

  const formData = new FormData(form);
  const competences = formData.getAll("competences");

  console.log(competences);
});
```

Résultat possible :

```javascript
["HTML", "CSS", "JavaScript"];
```

# 32. Exemple complet : formulaire dynamique avec validation

## HTML

```html
<form id="formationForm" novalidate>
  <div>
    <label for="prenom">Prénom</label>
    <input type="text" id="prenom" name="prenom" />
    <p id="prenomError" class="error"></p>
  </div>

  <div>
    <label for="typeProfil">Type de profil</label>
    <select id="typeProfil" name="typeProfil">
      <option value="">Choisir un type</option>
      <option value="etudiant">Étudiant</option>
      <option value="professionnel">Professionnel</option>
    </select>
    <p id="typeProfilError" class="error"></p>
  </div>

  <div id="ecoleSection" class="hidden">
    <label for="ecole">Nom de l’école</label>
    <input type="text" id="ecole" name="ecole" />
    <p id="ecoleError" class="error"></p>
  </div>

  <div id="entrepriseSection" class="hidden">
    <label for="entreprise">Nom de l’entreprise</label>
    <input type="text" id="entreprise" name="entreprise" />
    <p id="entrepriseError" class="error"></p>
  </div>

  <button type="submit">Envoyer</button>
</form>
```

## CSS

```css
.hidden {
  display: none;
}

.error {
  color: red;
  font-size: 0.9rem;
}

.input-error {
  border: 2px solid red;
}
```

## JavaScript

```javascript
const form = document.querySelector("#formationForm");

const prenomInput = document.querySelector("#prenom");
const typeProfilSelect = document.querySelector("#typeProfil");
const ecoleInput = document.querySelector("#ecole");
const entrepriseInput = document.querySelector("#entreprise");

const ecoleSection = document.querySelector("#ecoleSection");
const entrepriseSection = document.querySelector("#entrepriseSection");

const prenomError = document.querySelector("#prenomError");
const typeProfilError = document.querySelector("#typeProfilError");
const ecoleError = document.querySelector("#ecoleError");
const entrepriseError = document.querySelector("#entrepriseError");

typeProfilSelect.addEventListener("change", function () {
  const typeProfil = typeProfilSelect.value;

  ecoleSection.classList.add("hidden");
  entrepriseSection.classList.add("hidden");

  ecoleInput.required = false;
  entrepriseInput.required = false;

  ecoleInput.value = "";
  entrepriseInput.value = "";

  ecoleError.textContent = "";
  entrepriseError.textContent = "";

  if (typeProfil === "etudiant") {
    ecoleSection.classList.remove("hidden");
    ecoleInput.required = true;
  }

  if (typeProfil === "professionnel") {
    entrepriseSection.classList.remove("hidden");
    entrepriseInput.required = true;
  }
});

form.addEventListener("submit", function (event) {
  event.preventDefault();

  prenomError.textContent = "";
  typeProfilError.textContent = "";
  ecoleError.textContent = "";
  entrepriseError.textContent = "";

  const prenom = prenomInput.value.trim();
  const typeProfil = typeProfilSelect.value;
  const ecole = ecoleInput.value.trim();
  const entreprise = entrepriseInput.value.trim();

  let formulaireValide = true;

  if (prenom === "") {
    prenomError.textContent = "Le prénom est obligatoire.";
    formulaireValide = false;
  }

  if (typeProfil === "") {
    typeProfilError.textContent = "Veuillez choisir un type de profil.";
    formulaireValide = false;
  }

  if (typeProfil === "etudiant" && ecole === "") {
    ecoleError.textContent = "Le nom de l’école est obligatoire.";
    formulaireValide = false;
  }

  if (typeProfil === "professionnel" && entreprise === "") {
    entrepriseError.textContent = "Le nom de l’entreprise est obligatoire.";
    formulaireValide = false;
  }

  if (!formulaireValide) {
    return;
  }

  const formData = new FormData(form);
  const donnees = Object.fromEntries(formData.entries());

  console.log(donnees);
});
```

# 33. Explication du formulaire dynamique complet

Dans cet exemple, le formulaire change selon le type de profil choisi.

Si l’utilisateur choisit :

```txt
Étudiant
```

alors le champ “Nom de l’école” apparaît.

Si l’utilisateur choisit :

```txt
Professionnel
```

alors le champ “Nom de l’entreprise” apparaît.

Le JavaScript fait plusieurs choses :

```txt
1. Il écoute le changement du select.
2. Il affiche ou cache certaines sections.
3. Il rend certains champs obligatoires selon le choix.
4. Il vide les champs cachés.
5. Il valide les champs visibles au moment de la soumission.
6. Il affiche les messages d’erreur appropriés.
```

Cette logique est très fréquente dans les vrais formulaires.

# 34. Erreurs fréquentes à éviter

## Erreur 1 : afficher une erreur sans dire quoi corriger

À éviter :

```txt
Erreur.
```

Mieux :

```txt
Veuillez entrer votre adresse courriel.
```

---

## Erreur 2 : utiliser `innerHTML` pour afficher une donnée utilisateur

À éviter :

```javascript
message.innerHTML = nomInput.value;
```

Mieux :

```javascript
message.textContent = nomInput.value;
```

---

## Erreur 3 : valider sans nettoyer la valeur

À éviter :

```javascript
if (nomInput.value === "") {
  // erreur
}
```

Mieux :

```javascript
if (nomInput.value.trim() === "") {
  // erreur
}
```

---

## Erreur 4 : oublier de vider les anciens messages d’erreur

À éviter :

```javascript
if (nom === "") {
  nomError.textContent = "Le nom est obligatoire.";
}
```

Le problème est que l’ancien message peut rester affiché même après correction.

Mieux :

```javascript
nomError.textContent = "";

if (nom === "") {
  nomError.textContent = "Le nom est obligatoire.";
}
```

---

## Erreur 5 : mettre un bouton dynamique en `submit`

À éviter :

```html
<button id="addFieldButton">Ajouter un champ</button>
```

Dans un formulaire, ce bouton peut soumettre le formulaire.

Mieux :

```html
<button type="button" id="addFieldButton">Ajouter un champ</button>
```

---

## Erreur 6 : cacher un champ sans vider sa valeur

Si un champ est caché mais contient encore une valeur, cette valeur peut être récupérée ou envoyée.

Mieux :

```javascript
champCache.value = "";
```

quand on cache la section.

---

## Erreur 7 : valider tous les champs, même ceux qui ne sont pas pertinents

Si un champ est caché parce qu’il ne concerne pas l’utilisateur, il ne faut pas toujours le valider.

Exemple :

```javascript
if (typeProfil === "etudiant" && ecole === "") {
  ecoleError.textContent = "Le nom de l’école est obligatoire.";
}
```

La validation dépend du contexte.

# 35. Ce qu’il faut absolument retenir

## Validation HTML

La validation HTML utilise des attributs comme :

```html
required type="email" minlength maxlength min max
```

Elle est simple et pratique, mais limitée.

---

## Validation JavaScript

La validation JavaScript permet de créer des règles personnalisées.

Structure fréquente :

```javascript
form.addEventListener("submit", function (event) {
  event.preventDefault();

  let formulaireValide = true;

  if (champ.value.trim() === "") {
    messageErreur.textContent = "Ce champ est obligatoire.";
    formulaireValide = false;
  }

  if (!formulaireValide) {
    return;
  }

  console.log("Formulaire valide.");
});
```

---

## Messages d’erreur

Un bon message d’erreur doit être :

```txt
clair
précis
visible
utile
proche du champ
```

---

## Événements importants

```txt
input  → se déclenche pendant la saisie
change → se déclenche lorsqu’une valeur change
focus  → se déclenche quand l’utilisateur entre dans un champ
blur   → se déclenche quand l’utilisateur quitte un champ
submit → se déclenche quand le formulaire est soumis
```

---

## Formulaires dynamiques

Un formulaire dynamique change selon les actions de l’utilisateur.

Il peut :

- afficher une section ;
- cacher un champ ;
- rendre un champ obligatoire ;
- désactiver un bouton ;
- ajouter des champs ;
- afficher un aperçu en direct.

---

# Conclusion

La validation et les formulaires dynamiques transforment un formulaire simple en véritable interface interactive.

La validation HTML permet de poser des règles simples rapidement.
La validation JavaScript permet de personnaliser l’expérience et de mieux guider l’utilisateur.
Les messages d’erreur aident l’utilisateur à comprendre et corriger ses erreurs.
Les événements comme `input`, `change`, `focus` et `blur` permettent au formulaire de réagir pendant l’utilisation.
Les formulaires dynamiques permettent d’afficher seulement les champs nécessaires selon le contexte.

La logique générale à retenir est la suivante :

```txt
L’utilisateur interagit avec le formulaire.
JavaScript écoute les événements.
Le programme récupère les valeurs.
Le programme vérifie les règles.
Les messages guident l’utilisateur.
Le formulaire s’adapte selon les choix.
Si tout est valide, les données peuvent être traitées.
```

À partir de cette étape, vous pouvez commencer à construire des formulaires plus proches de ceux utilisés dans de vraies applications web.
