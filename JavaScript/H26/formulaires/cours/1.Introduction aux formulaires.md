# Support de cours

# Introduction aux formulaires HTML : rôle, logique, structure et champs essentiels

## Présentation générale

Les formulaires occupent une place centrale dans le développement web. Ils permettent à une page web de ne plus être seulement un espace d’affichage, mais aussi un espace d’interaction.

Une page sans formulaire peut présenter de l’information.
Une page avec formulaire peut recevoir de l’information.

Cette différence est fondamentale. Dès qu’un utilisateur écrit son nom, choisit une option, coche une case, sélectionne une date ou clique sur un bouton d’envoi, il communique avec l’application.

Les formulaires sont donc un point de rencontre entre trois parties importantes du web :

```txt
HTML  →  structure du formulaire
CSS   →  apparence du formulaire
JavaScript  →  comportement du formulaire
```

Dans ce premier cours, nous allons surtout nous concentrer sur la partie HTML. Avant de valider un formulaire, de récupérer les données avec JavaScript ou de les envoyer à un serveur, il faut d’abord comprendre comment un formulaire est construit.

---

# 1. Qu’est-ce qu’un formulaire ?

Un formulaire est une zone de la page web qui permet à l’utilisateur de saisir, choisir ou envoyer des informations.

Par exemple, un formulaire peut servir à :

- créer un compte ;
- se connecter à une application ;
- envoyer un message ;
- faire une recherche ;
- remplir une commande ;
- réserver une place ;
- ajouter un commentaire ;
- modifier un profil ;
- répondre à un sondage ;
- choisir des préférences.

Dans tous ces cas, le principe est le même : l’utilisateur donne une information à la page.

Exemple très simple :

```html
<form>
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <button type="submit">Envoyer</button>
</form>
```

Même si ce formulaire est très court, il contient déjà les éléments essentiels :

- une balise `<form>` ;
- un texte qui décrit le champ avec `<label>` ;
- un champ de saisie avec `<input>` ;
- un bouton d’envoi avec `<button>`.

---

# 2. Le rôle des formulaires dans une application web

Un formulaire sert à recueillir une intention ou une information de l’utilisateur.

Prenons quelques exemples.

## Exemple 1 : formulaire de connexion

L’utilisateur veut accéder à son compte.

Il doit fournir :

- son adresse courriel ;
- son mot de passe.

```html
<form>
  <label for="email">Adresse courriel</label>
  <input type="email" id="email" name="email" />

  <label for="password">Mot de passe</label>
  <input type="password" id="password" name="password" />

  <button type="submit">Connexion</button>
</form>
```

Ici, le formulaire permet à l’utilisateur de dire :

> Je veux me connecter avec ces informations.

---

## Exemple 2 : formulaire de recherche

L’utilisateur veut chercher une information.

```html
<form>
  <label for="recherche">Recherche</label>
  <input type="search" id="recherche" name="recherche" />

  <button type="submit">Rechercher</button>
</form>
```

Ici, le formulaire permet à l’utilisateur de dire :

> Je cherche quelque chose.

---

## Exemple 3 : formulaire de contact

L’utilisateur veut envoyer un message.

```html
<form>
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <label for="message">Message</label>
  <textarea id="message" name="message"></textarea>

  <button type="submit">Envoyer</button>
</form>
```

Ici, le formulaire permet à l’utilisateur de dire :

> Je veux transmettre ce message.

---

# 3. La logique générale d’un formulaire

Un formulaire suit généralement une logique simple :

```txt
1. L’utilisateur voit des champs.
2. L’utilisateur remplit ou sélectionne des informations.
3. L’utilisateur clique sur un bouton.
4. Le formulaire est soumis.
5. Les données peuvent ensuite être traitées.
```

Pour l’instant, nous ne parlons pas encore du traitement JavaScript ou serveur. Nous observons seulement la base HTML.

Un formulaire bien construit prépare les données pour la suite. Si les champs sont mal nommés, mal structurés ou difficiles à comprendre, la récupération des données deviendra plus difficile.

---

# 4. La balise `<form>`

La balise `<form>` représente le formulaire complet.

Elle sert à regrouper les champs qui appartiennent à la même action.

```html
<form>
  <!-- Les champs du formulaire seront placés ici -->
</form>
```

Tout ce qui se trouve entre `<form>` et `</form>` fait partie du formulaire.

Exemple :

```html
<form>
  <label for="prenom">Prénom</label>
  <input type="text" id="prenom" name="prenom" />

  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" />

  <button type="submit">Envoyer</button>
</form>
```

Dans cet exemple, les champs `prenom` et `nom` appartiennent au même formulaire.

---

# 5. La structure de base d’un formulaire

Un formulaire lisible est généralement organisé en groupes.

Exemple :

```html
<form>
  <div>
    <label for="prenom">Prénom</label>
    <input type="text" id="prenom" name="prenom" />
  </div>

  <div>
    <label for="email">Adresse courriel</label>
    <input type="email" id="email" name="email" />
  </div>

  <button type="submit">Envoyer</button>
</form>
```

Les balises `<div>` ne sont pas obligatoires, mais elles permettent de mieux organiser le code.

Chaque groupe contient souvent :

```txt
un label
un champ
éventuellement un message d’aide ou d’erreur
```

Exemple plus structuré :

```html
<form>
  <div class="form-group">
    <label for="email">Adresse courriel</label>
    <input type="email" id="email" name="email" />
    <p>Nous utiliserons cette adresse pour vous contacter.</p>
  </div>

  <button type="submit">Envoyer</button>
</form>
```

Cette structure sera très utile plus tard avec CSS et JavaScript.

---

# 6. Le rôle du `<label>`

La balise `<label>` sert à décrire un champ.

Exemple :

```html
<label for="email">Adresse courriel</label>
<input type="email" id="email" name="email" />
```

Le texte “Adresse courriel” indique à l’utilisateur ce qu’il doit écrire dans le champ.

Le lien entre le label et le champ se fait grâce à deux attributs :

```html
<label for="email"> <input id="email" /></label>
```

Le `for` du label doit correspondre au `id` du champ.

```html
<label for="prenom">Prénom</label>
<input type="text" id="prenom" name="prenom" />
```

Ici :

```txt
for="prenom"
id="prenom"
```

Les deux valeurs sont identiques. Le navigateur comprend donc que ce label décrit ce champ.

---

## Pourquoi le label est-il important ?

Le label est important pour plusieurs raisons.

Il rend le formulaire plus clair pour l’utilisateur.
Il améliore l’accessibilité.
Il permet aux lecteurs d’écran de mieux comprendre le formulaire.
Il permet aussi de cliquer sur le texte du label pour activer le champ.

Exemple :

```html
<label for="conditions">J’accepte les conditions</label>
<input type="checkbox" id="conditions" name="conditions" />
```

Si le label est bien lié au champ, cliquer sur le texte peut cocher la case.

---

## Mauvaise pratique

```html
<input type="email" placeholder="Email" />
```

Ce champ n’a pas de label. Il peut sembler clair au début, mais il devient moins accessible et moins professionnel.

---

## Meilleure pratique

```html
<label for="email">Adresse courriel</label>
<input type="email" id="email" name="email" placeholder="exemple@email.com" />
```

Le label donne le sens du champ.
Le placeholder donne seulement un exemple.

---

# 7. La balise `<input>`

La balise `<input>` est l’un des éléments les plus utilisés dans les formulaires.

Elle permet de créer différents types de champs selon la valeur de son attribut `type`.

Exemple :

```html
<input type="text" />
```

L’attribut `type` indique au navigateur quel type de donnée est attendu.

Quelques types courants :

```html
<input type="text" />
<input type="email" />
<input type="password" />
<input type="number" />
<input type="date" />
<input type="checkbox" />
<input type="radio" />
<input type="file" />
<input type="search" />
<input type="tel" />
<input type="url" />
```

Chaque type a un comportement différent.

---

# 8. Le champ texte : `type="text"`

Le champ texte est utilisé pour une courte saisie textuelle.

Exemple :

```html
<label for="prenom">Prénom</label>
<input type="text" id="prenom" name="prenom" />
```

On l’utilise pour des informations comme :

- prénom ;
- nom ;
- ville ;
- titre ;
- pseudo ;
- nom d’utilisateur.

Le champ texte est très flexible, mais il ne donne pas beaucoup d’indications particulières au navigateur. Il accepte presque tous les textes.

---

# 9. Le champ courriel : `type="email"`

Le champ courriel sert à demander une adresse courriel.

```html
<label for="email">Adresse courriel</label>
<input type="email" id="email" name="email" />
```

Le navigateur comprend qu’il s’agit d’une adresse courriel. Il peut donc faire une vérification de base lorsque le formulaire est soumis.

Par exemple, si l’utilisateur écrit seulement :

```txt
bonjour
```

le navigateur peut considérer que ce n’est pas une adresse courriel valide.

Cependant, il faut comprendre une chose importante : `type="email"` ne garantit pas que l’adresse existe réellement. Il vérifie seulement que la valeur ressemble à une adresse courriel.

---

# 10. Le champ mot de passe : `type="password"`

Le champ mot de passe masque les caractères saisis.

```html
<label for="password">Mot de passe</label>
<input type="password" id="password" name="password" />
```

Ce type de champ est utilisé pour éviter que le mot de passe soit visible à l’écran pendant la saisie.

Cependant, il faut éviter une confusion fréquente : le champ `password` ne sécurise pas réellement les données à lui seul. Il masque seulement l’affichage.

La vraie sécurité dépend ensuite d’autres éléments, comme :

- l’utilisation de HTTPS ;
- le traitement côté serveur ;
- le stockage sécurisé des mots de passe ;
- la validation côté serveur.

Pour l’instant, les il faut retenir que `type="password"` sert surtout à masquer visuellement la saisie.

---

# 11. Le champ nombre : `type="number"`

Le champ nombre permet de saisir une valeur numérique.

```html
<label for="age">Âge</label> <input type="number" id="age" name="age" />
```

On peut lui ajouter des limites :

```html
<label for="age">Âge</label>
<input type="number" id="age" name="age" min="0" max="120" />
```

Ici :

```txt
min="0"      → valeur minimale
max="120"   → valeur maximale
```

On peut aussi ajouter un pas avec `step`.

```html
<label for="quantite">Quantité</label>
<input type="number" id="quantite" name="quantite" min="1" step="1" />
```

Le champ nombre est utile pour :

- âge ;
- quantité ;
- prix ;
- nombre d’articles ;
- note ;
- durée.

---

# 12. Le champ date : `type="date"`

Le champ date permet à l’utilisateur de sélectionner une date.

```html
<label for="naissance">Date de naissance</label>
<input type="date" id="naissance" name="naissance" />
```

Selon le navigateur, un petit calendrier peut apparaître.

Ce champ est utile pour :

- date de naissance ;
- date de réservation ;
- date de début ;
- date de fin ;
- échéance.

On peut aussi utiliser `min` et `max`.

```html
<label for="dateReservation">Date de réservation</label>
<input
  type="date"
  id="dateReservation"
  name="dateReservation"
  min="2026-01-01"
  max="2026-12-31"
/>
```

Cela limite les dates que l’utilisateur peut choisir.

---

# 13. Le champ recherche : `type="search"`

Le type `search` est utilisé pour les formulaires de recherche.

```html
<label for="recherche">Recherche</label>
<input type="search" id="recherche" name="recherche" />
```

Visuellement, il ressemble souvent à un champ texte. Sur certains navigateurs, il peut offrir des comportements pratiques, comme une icône pour effacer la recherche.

Il est plus précis sémantiquement que `type="text"` lorsque le champ sert vraiment à faire une recherche.

---

# 14. Le champ téléphone : `type="tel"`

Le type `tel` sert à demander un numéro de téléphone.

```html
<label for="telephone">Téléphone</label>
<input type="tel" id="telephone" name="telephone" />
```

Il ne vérifie pas automatiquement tous les formats de téléphone, car les formats varient selon les pays.

Il peut toutefois adapter le clavier sur mobile pour faciliter la saisie d’un numéro.

---

# 15. Le champ URL : `type="url"`

Le type `url` sert à demander une adresse web.

```html
<label for="site">Site web</label> <input type="url" id="site" name="site" />
```

Le navigateur peut vérifier que la valeur ressemble à une URL.

Exemple attendu :

```txt
https://exemple.com
```

---

# 16. Le champ fichier : `type="file"`

Le type `file` permet à l’utilisateur de sélectionner un fichier depuis son appareil.

```html
<label for="cv">Téléverser un fichier</label>
<input type="file" id="cv" name="cv" />
```

On peut limiter les types de fichiers acceptés avec `accept`.

```html
<label for="image">Choisir une image</label>
<input type="file" id="image" name="image" accept="image/*" />
```

Ici, le navigateur comprend que l’utilisateur doit choisir une image.

On peut aussi permettre plusieurs fichiers avec `multiple`.

```html
<label for="documents">Documents</label>
<input type="file" id="documents" name="documents" multiple />
```

Le champ fichier sera étudié plus en détail plus tard, car son traitement est différent des champs textuels.

---

# 17. La zone de texte longue : `<textarea>`

La balise `<textarea>` sert à saisir un texte plus long.

Exemple :

```html
<label for="message">Message</label>
<textarea id="message" name="message"></textarea>
```

Contrairement à `<input>`, la balise `<textarea>` possède une balise ouvrante et une balise fermante.

```html
<textarea></textarea>
```

Elle est utilisée pour :

- message ;
- commentaire ;
- description ;
- justification ;
- biographie ;
- contenu long.

On peut aussi prévoir un texte initial :

```html
<textarea id="message" name="message">Bonjour,</textarea>
```

Cependant, dans la plupart des formulaires, on laisse la zone vide.

---

## Différence entre `<input>` et `<textarea>`

Un `<input>` sert généralement à une information courte.

```html
<input type="text" name="titre" />
```

Un `<textarea>` sert à une information plus longue.

```html
<textarea name="description"></textarea>
```

Exemple concret :

```html
<label for="titre">Titre</label>
<input type="text" id="titre" name="titre" />

<label for="description">Description</label>
<textarea id="description" name="description"></textarea>
```

Le titre est court.
La description peut être longue.

---

# 18. La liste déroulante : `<select>` et `<option>`

La balise `<select>` permet à l’utilisateur de choisir une option dans une liste.

```html
<label for="niveau">Niveau</label>

<select id="niveau" name="niveau">
  <option value="">Choisir un niveau</option>
  <option value="debutant">Débutant</option>
  <option value="intermediaire">Intermédiaire</option>
  <option value="avance">Avancé</option>
</select>
```

La balise `<select>` représente la liste complète.
Chaque balise `<option>` représente un choix possible.

---

## Comprendre `value` dans une option

Prenons cette option :

```html
<option value="debutant">Débutant</option>
```

Il y a deux informations importantes :

```txt
value="debutant"  → valeur utilisée par le programme
Débutant          → texte affiché à l’utilisateur
```

Le texte affiché peut être joli, lisible et accentué.
La valeur peut être plus simple à manipuler en JavaScript.

Autre exemple :

```html
<option value="qc">Québec</option>
<option value="on">Ontario</option>
<option value="bc">Colombie-Britannique</option>
```

L’utilisateur voit :

```txt
Québec
Ontario
Colombie-Britannique
```

Le programme reçoit :

```txt
qc
on
bc
```

---

## Option vide au début

Il est souvent recommandé de placer une option vide au début.

```html
<option value="">Choisir une option</option>
```

Cela évite qu’un choix soit sélectionné automatiquement sans que l’utilisateur ait vraiment choisi.

Exemple :

```html
<label for="programme">Programme</label>

<select id="programme" name="programme" required>
  <option value="">Choisir un programme</option>
  <option value="web">Développement web</option>
  <option value="reseau">Réseaux informatiques</option>
  <option value="design">Design numérique</option>
</select>
```

Ici, comme le champ est `required`, l’utilisateur devra choisir une vraie option.

---

# 19. Les cases à cocher : `checkbox`

Une case à cocher permet à l’utilisateur de choisir une option indépendante.

Exemple :

```html
<label>
  <input type="checkbox" name="conditions" value="accepte" />
  J’accepte les conditions d’utilisation
</label>
```

Une case à cocher peut être cochée ou non cochée.

Elle est souvent utilisée pour :

- accepter des conditions ;
- s’inscrire à une infolettre ;
- choisir plusieurs préférences ;
- sélectionner plusieurs catégories ;
- activer ou désactiver une option.

---

## Checkbox simple

Exemple :

```html
<label>
  <input type="checkbox" id="newsletter" name="newsletter" value="oui" />
  Je souhaite recevoir l’infolettre
</label>
```

Ici, l’utilisateur peut choisir oui ou non en cochant ou non la case.

Dans ce cas, le texte du label est souvent placé autour du champ. Cette structure est acceptable :

```html
<label>
  <input type="checkbox" name="newsletter" value="oui" />
  Je souhaite recevoir l’infolettre
</label>
```

Elle permet de cliquer sur le texte pour cocher la case.

---

## Plusieurs cases à cocher

Les cases à cocher permettent aussi de choisir plusieurs réponses.

```html
<p>Technologies connues</p>

<label>
  <input type="checkbox" name="technologies" value="html" />
  HTML
</label>

<label>
  <input type="checkbox" name="technologies" value="css" />
  CSS
</label>

<label>
  <input type="checkbox" name="technologies" value="javascript" />
  JavaScript
</label>
```

Ici, l’utilisateur peut cocher :

- HTML seulement ;
- CSS seulement ;
- JavaScript seulement ;
- HTML et CSS ;
- les trois ;
- aucune option.

Toutes les cases ont le même `name` :

```html
name="technologies"
```

Cela indique qu’elles appartiennent au même groupe de données.

Chaque case a cependant une valeur différente :

```html
value="html" value="css" value="javascript"
```

---

# 20. Les boutons radio : `radio`

Les boutons radio permettent de choisir une seule option dans un groupe.

Exemple :

```html
<p>Niveau actuel</p>

<label>
  <input type="radio" name="niveau" value="debutant" />
  Débutant
</label>

<label>
  <input type="radio" name="niveau" value="intermediaire" />
  Intermédiaire
</label>

<label>
  <input type="radio" name="niveau" value="avance" />
  Avancé
</label>
```

L’utilisateur peut choisir un seul niveau.

La règle importante est la suivante :

> Les boutons radio d’un même groupe doivent avoir le même `name`.

Ici :

```html
name="niveau"
```

est répété pour chaque bouton radio.

Cela permet au navigateur de comprendre que ces choix s’excluent mutuellement.

---

## Différence entre checkbox et radio

Une checkbox permet de cocher ou décocher une option.
Plusieurs checkbox peuvent être cochées en même temps.

Un radio permet de choisir une seule option dans un groupe.
Quand une option est sélectionnée, les autres options du même groupe sont désélectionnées.

Exemple :

```txt
Checkbox  → plusieurs choix possibles
Radio     → un seul choix possible
```

---

# 21. Les boutons dans les formulaires

Un formulaire contient souvent un bouton.

```html
<button type="submit">Envoyer</button>
```

Le bouton peut avoir plusieurs types.

---

## `type="submit"`

```html
<button type="submit">Envoyer</button>
```

Ce bouton soumet le formulaire.

C’est le bouton le plus courant dans un formulaire.

---

## `type="button"`

```html
<button type="button">Afficher plus d’options</button>
```

Ce bouton ne soumet pas le formulaire automatiquement.

Il est utilisé quand on veut déclencher une action JavaScript sans envoyer le formulaire.

Par exemple :

- afficher une section ;
- ajouter un champ ;
- ouvrir une fenêtre ;
- calculer un résultat ;
- afficher un aperçu.

---

## `type="reset"`

```html
<button type="reset">Réinitialiser</button>
```

Ce bouton remet les champs du formulaire à leur valeur initiale.

Il faut l’utiliser avec prudence, car l’utilisateur peut effacer toutes ses données par accident.

---

## Attention au type par défaut

Dans un formulaire, un bouton sans type peut se comporter comme un bouton de soumission.

Exemple :

```html
<button>Cliquer</button>
```

Dans un formulaire, ce bouton peut être interprété comme :

```html
<button type="submit">Cliquer</button>
```

Il est donc recommandé d’écrire explicitement le type du bouton.

```html
<button type="submit">Envoyer</button> <button type="button">Annuler</button>
```

# 22. Les attributs importants des champs

Les champs de formulaire utilisent plusieurs attributs importants. Ces attributs donnent des informations au navigateur, à JavaScript et parfois au serveur.

Les plus importants à comprendre au début sont :

```txt
id
name
value
placeholder
required
type
min
max
minlength
maxlength
checked
selected
disabled
readonly
```

Nous allons les étudier progressivement.

---

# 23. L’attribut `id`

L’attribut `id` sert à identifier un élément précis dans la page.

Exemple :

```html
<input type="text" id="prenom" />
```

Le `id` doit être unique dans la page. Cela signifie qu’on ne devrait pas avoir deux éléments avec le même `id`.

Exemple à éviter :

```html
<input id="email" /> <input id="email" />
```

Exemple correct :

```html
<input id="email" /> <input id="confirmationEmail" />
```

---

## À quoi sert le `id` ?

Le `id` sert principalement à trois choses :

Premièrement, il permet de lier un label à un champ.

```html
<label for="email">Adresse courriel</label> <input type="email" id="email" />
```

Deuxièmement, il permet de cibler l’élément en CSS.

```css
#email {
  border: 1px solid black;
}
```

Troisièmement, il permet de récupérer l’élément avec JavaScript.

```javascript
const emailInput = document.querySelector("#email");
```

Le `id` identifie donc l’élément dans la page.

---

# 24. L’attribut `name`

L’attribut `name` sert à nommer la donnée du formulaire.

Exemple :

```html
<input type="text" id="prenom" name="prenom" />
```

Le `name` est essentiel lorsque les données du formulaire sont envoyées ou récupérées.

Si l’utilisateur écrit :

```txt
Sara
```

dans ce champ :

```html
<input type="text" name="prenom" />
```

la donnée peut être représentée ainsi :

```txt
prenom = Sara
```

---

## Différence entre `id` et `name`

C’est une distinction fondamentale.

```html
<input type="email" id="email" name="email" />
```

Le `id` sert à identifier l’élément dans la page.
Le `name` sert à identifier la donnée envoyée.

Autrement dit :

```txt
id    → pour le HTML, le CSS, le DOM et JavaScript
name  → pour les données du formulaire
```

Dans un formulaire bien construit, on utilise souvent les deux.

---

## Exemple important

```html
<label for="courriel">Adresse courriel</label>
<input type="email" id="courriel" name="email" />
```

Ici :

```txt
id="courriel"
name="email"
```

Le label utilise le `id` :

```html
<label for="courriel"></label>
```

Mais la donnée envoyée s’appelle :

```txt
email
```

Il faut donc comprendre que `id` et `name` peuvent être différents, même si on choisit souvent de leur donner le même nom pour simplifier.

---

# 25. L’attribut `value`

L’attribut `value` représente la valeur d’un champ.

Avec un champ texte, il peut définir une valeur déjà présente au chargement de la page.

```html
<label for="ville">Ville</label>
<input type="text" id="ville" name="ville" value="Montréal" />
```

Dans cet exemple, le champ contient déjà :

```txt
Montréal
```

---

## `value` avec les options

Dans une liste déroulante :

```html
<option value="debutant">Débutant</option>
```

La valeur envoyée est :

```txt
debutant
```

Le texte affiché est :

```txt
Débutant
```

---

## `value` avec les boutons radio

```html
<label>
  <input type="radio" name="niveau" value="debutant" />
  Débutant
</label>
```

Si l’utilisateur choisit cette option, la valeur envoyée pour `niveau` sera :

```txt
debutant
```

---

## `value` avec les checkbox

```html
<label>
  <input type="checkbox" name="newsletter" value="oui" />
  Je souhaite recevoir l’infolettre
</label>
```

Si la case est cochée, la valeur associée peut être :

```txt
newsletter = oui
```

Si elle n’est pas cochée, la donnée peut ne pas être envoyée.

---

# 26. L’attribut `placeholder`

L’attribut `placeholder` affiche un exemple ou une indication temporaire dans un champ.

```html
<label for="email">Adresse courriel</label>
<input type="email" id="email" name="email" placeholder="exemple@email.com" />
```

Le placeholder disparaît généralement quand l’utilisateur commence à écrire.

Il sert à donner un exemple, mais il ne doit pas remplacer le label.

---

## Mauvaise utilisation du placeholder

```html
<input type="text" placeholder="Nom" />
<input type="email" placeholder="Email" />
```

Ces champs semblent compréhensibles, mais ils n’ont pas de vrais labels.

Quand l’utilisateur commence à écrire, l’indication disparaît. Cela peut rendre le formulaire moins clair.

---

## Bonne utilisation du placeholder

```html
<label for="nom">Nom complet</label>
<input type="text" id="nom" name="nom" placeholder="Ex. Sara Dupont" />
```

Le label indique ce que le champ représente.
Le placeholder donne seulement un exemple.

---

# 27. L’attribut `required`

L’attribut `required` rend un champ obligatoire.

```html
<label for="nom">Nom</label> <input type="text" id="nom" name="nom" required />
```

Si l’utilisateur essaie de soumettre le formulaire sans remplir ce champ, le navigateur peut bloquer l’envoi et afficher un message.

Exemple avec plusieurs champs obligatoires :

```html
<form>
  <label for="nom">Nom</label>
  <input type="text" id="nom" name="nom" required />

  <label for="email">Adresse courriel</label>
  <input type="email" id="email" name="email" required />

  <button type="submit">Envoyer</button>
</form>
```

Ici, le nom et le courriel sont obligatoires.

---

## Important : `required` aide, mais ne suffit pas

L’attribut `required` est une validation côté navigateur. Il améliore l’expérience utilisateur, mais il ne remplace pas la validation JavaScript ou serveur.

Un utilisateur avancé peut modifier le HTML dans le navigateur. Il ne faut donc jamais considérer `required` comme une protection complète.

Pour le moment, il faut retenir ceci :

```txt
required = le navigateur demande une valeur avant de soumettre le formulaire
```

---

# 28. Les attributs `minlength` et `maxlength`

Ces attributs contrôlent la longueur d’un texte.

```html
<label for="username">Nom d’utilisateur</label>
<input type="text" id="username" name="username" minlength="3" maxlength="20" />
```

Ici :

```txt
minlength="3"   → au moins 3 caractères
maxlength="20"  → au maximum 20 caractères
```

Cela peut être utile pour :

- nom d’utilisateur ;
- mot de passe ;
- commentaire court ;
- code ;
- titre.

Exemple :

```html
<label for="motdepasse">Mot de passe</label>
<input
  type="password"
  id="motdepasse"
  name="motdepasse"
  minlength="8"
  required
/>
```

Le navigateur peut refuser un mot de passe trop court.

---

# 29. Les attributs `min`, `max` et `step`

Ces attributs sont souvent utilisés avec les champs numériques ou les dates.

Exemple avec un nombre :

```html
<label for="age">Âge</label>
<input type="number" id="age" name="age" min="0" max="120" />
```

Exemple avec une quantité :

```html
<label for="quantite">Quantité</label>
<input type="number" id="quantite" name="quantite" min="1" max="10" step="1" />
```

Ici :

```txt
min="1"   → valeur minimale
max="10"  → valeur maximale
step="1"  → progression de 1 en 1
```

Exemple avec une date :

```html
<label for="date">Date</label>
<input type="date" id="date" name="date" min="2026-01-01" />
```

---

# 30. L’attribut `checked`

L’attribut `checked` sert à cocher une case ou un bouton radio par défaut.

Exemple avec une checkbox :

```html
<label>
  <input type="checkbox" name="newsletter" value="oui" checked />
  Je souhaite recevoir l’infolettre
</label>
```

Ici, la case est déjà cochée au chargement de la page.

Exemple avec un radio :

```html
<p>Niveau</p>

<label>
  <input type="radio" name="niveau" value="debutant" checked />
  Débutant
</label>

<label>
  <input type="radio" name="niveau" value="avance" />
  Avancé
</label>
```

Ici, “Débutant” est sélectionné par défaut.

Il faut utiliser cet attribut avec prudence. Dans certains contextes, il vaut mieux laisser l’utilisateur faire un choix volontairement.

---

# 31. L’attribut `selected`

L’attribut `selected` sert à sélectionner une option par défaut dans un `<select>`.

```html
<label for="pays">Pays</label>

<select id="pays" name="pays">
  <option value="ca" selected>Canada</option>
  <option value="fr">France</option>
  <option value="ma">Maroc</option>
</select>
```

Ici, Canada est sélectionné par défaut.

Cependant, dans un formulaire où l’utilisateur doit faire un choix réel, on préfère souvent une option vide au début.

```html
<select id="pays" name="pays" required>
  <option value="">Choisir un pays</option>
  <option value="ca">Canada</option>
  <option value="fr">France</option>
  <option value="ma">Maroc</option>
</select>
```

---

# 32. L’attribut `disabled`

L’attribut `disabled` désactive un champ.

```html
<input type="text" name="code" disabled />
```

Un champ désactivé ne peut pas être modifié par l’utilisateur.

Il apparaît souvent grisé.

Attention : un champ `disabled` n’est généralement pas envoyé avec les données du formulaire.

Exemple :

```html
<label for="statut">Statut</label>
<input type="text" id="statut" name="statut" value="En attente" disabled />
```

Ce champ est visible, mais l’utilisateur ne peut pas le modifier.

---

# 33. L’attribut `readonly`

L’attribut `readonly` rend un champ non modifiable, mais sa valeur peut tout de même être incluse dans les données du formulaire.

```html
<label for="numero">Numéro de dossier</label>
<input type="text" id="numero" name="numero" value="A-12345" readonly />
```

Différence importante :

```txt
disabled  → non modifiable, souvent non envoyé
readonly  → non modifiable, mais valeur conservée
```

---

# 34. Organiser un formulaire avec `<fieldset>` et `<legend>`

Pour les formulaires plus longs, on peut regrouper certains champs avec `<fieldset>`.

La balise `<legend>` donne un titre au groupe.

Exemple :

```html
<form>
  <fieldset>
    <legend>Informations personnelles</legend>

    <label for="prenom">Prénom</label>
    <input type="text" id="prenom" name="prenom" />

    <label for="nom">Nom</label>
    <input type="text" id="nom" name="nom" />
  </fieldset>

  <fieldset>
    <legend>Préférences</legend>

    <label>
      <input type="checkbox" name="newsletter" value="oui" />
      Recevoir l’infolettre
    </label>
  </fieldset>

  <button type="submit">Envoyer</button>
</form>
```

Cette structure est particulièrement utile pour :

- améliorer la lisibilité ;
- organiser les grands formulaires ;
- améliorer l’accessibilité ;
- regrouper les boutons radio ou les cases à cocher.

# 36. Lecture pédagogique du formulaire complet

Dans le formulaire précédent, chaque partie a un rôle précis.

## La balise `<form>`

Elle regroupe tout le formulaire.

```html
<form></form>
```

C’est le conteneur principal.

---

## Les `<fieldset>`

Ils organisent le formulaire en sections.

```html
<fieldset>
  <legend>Informations personnelles</legend>
</fieldset>
```

Cela rend le formulaire plus clair.

---

## Les labels

Chaque champ important possède un label.

```html
<label for="email">Adresse courriel</label>
<input type="email" id="email" name="email" />
```

Cela permet à l’utilisateur de comprendre ce qu’il doit saisir.

---

## Les attributs `id`

Ils permettent de lier les labels aux champs.

```html
<label for="prenom">Prénom</label> <input id="prenom" />
```

Le `for` correspond au `id`.

---

## Les attributs `name`

Ils nomment les données.

```html
<input name="prenom" />
```

Plus tard, quand on récupérera les données du formulaire, cette donnée s’appellera `prenom`.

---

## Les attributs `required`

Ils rendent certains champs obligatoires.

```html
<input type="email" name="email" required />
```

Le navigateur demandera à l’utilisateur de remplir ce champ avant l’envoi.

# 37. Erreurs fréquentes à éviter

## Erreur 1 : oublier le label

Exemple incomplet :

```html
<input type="text" name="prenom" />
```

Même si le champ fonctionne, l’utilisateur ne sait pas clairement quoi écrire.

Correction :

```html
<label for="prenom">Prénom</label>
<input type="text" id="prenom" name="prenom" />
```

---

## Erreur 2 : utiliser un placeholder à la place du label

À éviter :

```html
<input type="text" placeholder="Prénom" />
```

Meilleur choix :

```html
<label for="prenom">Prénom</label>
<input type="text" id="prenom" name="prenom" placeholder="Ex. Sara" />
```

Le label reste visible.
Le placeholder sert seulement d’exemple.

---

## Erreur 3 : oublier l’attribut `name`

À éviter :

```html
<input type="email" id="email" />
```

Le champ a un `id`, mais pas de `name`.

Correction :

```html
<input type="email" id="email" name="email" />
```

Le `name` sera essentiel pour récupérer ou envoyer la donnée.

---

## Erreur 4 : donner le même `id` à plusieurs champs

À éviter :

```html
<input type="text" id="nom" name="prenom" />
<input type="text" id="nom" name="nom" />
```

Correction :

```html
<input type="text" id="prenom" name="prenom" />
<input type="text" id="nom" name="nom" />
```

Chaque `id` doit être unique.

---

## Erreur 5 : mal regrouper les boutons radio

À éviter :

```html
<input type="radio" name="debutant" value="debutant" />
<input type="radio" name="avance" value="avance" />
```

Ici, chaque radio a un `name` différent. Le navigateur ne comprend pas qu’ils appartiennent au même groupe.

Correction :

```html
<input type="radio" name="niveau" value="debutant" />
<input type="radio" name="niveau" value="avance" />
```

Les deux boutons radio partagent le même `name`.

---

## Erreur 6 : oublier le type du bouton

À éviter :

```html
<button>Annuler</button>
```

Dans un formulaire, ce bouton peut soumettre le formulaire sans qu’on le veuille.

Correction :

```html
<button type="button">Annuler</button> <button type="submit">Envoyer</button>
```

---

# 38. Ce qu’il faut absolument retenir

Un formulaire HTML sert à recueillir des informations auprès de l’utilisateur.

La balise principale est :

```html
<form></form>
```

Les champs les plus courants sont :

```html
<input />
<textarea></textarea>
<select></select>
<option></option>
```

Le label sert à décrire un champ :

```html
<label for="email">Adresse courriel</label> <input id="email" />
```

Le `id` sert à identifier un élément dans la page.

Le `name` sert à nommer la donnée du formulaire.

```html
<input id="email" name="email" />
```

Le `value` représente la valeur d’un champ ou d’une option.

Le `placeholder` donne un exemple, mais ne remplace pas le label.

Le `required` rend un champ obligatoire.

Les checkbox permettent plusieurs choix.

Les boutons radio permettent un seul choix dans un groupe.

Le bouton `submit` permet de soumettre le formulaire.

---

# Conclusion

Les formulaires sont une base essentielle du développement web. Ils permettent à l’utilisateur de communiquer avec une page ou une application.

Avant d’utiliser JavaScript, il faut savoir construire un formulaire HTML correctement. Un formulaire bien structuré facilite ensuite la validation, la récupération des données et l’envoi vers un serveur.

La logique à retenir est simple :

```txt
Le formulaire regroupe les champs.
Les labels expliquent les champs.
Les inputs permettent la saisie.
Les attributs précisent le comportement.
Les names nomment les données.
Le bouton submit déclenche l’envoi.
```
