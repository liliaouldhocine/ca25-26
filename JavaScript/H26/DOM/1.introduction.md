# Le DOM en JavaScript

## Objectifs du cours

À la fin de ce cours, vous devriez être capables de :

- expliquer ce qu’est le DOM
- comprendre son utilité dans une page web
- repérer la logique de sa structure
- sélectionner des éléments HTML avec JavaScript
- modifier du contenu, des classes, des attributs et du style
- réagir aux actions de l’utilisateur
- créer, ajouter et supprimer des éléments dans une page

---

# 1. Qu’est-ce que le DOM ?

Le **DOM** signifie **Document Object Model**.

C’est la **représentation de la page HTML en mémoire**, sous forme d’objets que JavaScript peut lire et modifier.

Autrement dit :

- le **HTML** construit la structure de la page
- le **CSS** gère l’apparence
- le **DOM** permet à **JavaScript d’interagir avec la page**

Quand un navigateur lit un fichier HTML, il ne garde pas simplement du texte brut. Il transforme ce code en une structure organisée que JavaScript peut manipuler.

## Idée importante

Le DOM est le **pont entre le HTML et JavaScript**.

Sans le DOM, JavaScript ne pourrait pas :

- changer un texte
- afficher ou masquer un élément
- réagir à un clic
- ajouter un élément dans une liste
- valider un formulaire

---

# 2. Pourquoi le DOM est-il important ?

Une page web moderne n’est pas seulement un document statique.

On veut souvent qu’elle puisse :

- réagir à l’utilisateur
- se mettre à jour sans recharger la page
- afficher du contenu dynamiquement
- changer d’apparence selon certaines actions

Le DOM rend tout cela possible.

## Exemples concrets

Grâce au DOM, on peut :

- changer le titre d’une page après un clic
- afficher un message d’erreur si un champ est vide
- ajouter un nouvel élément à une liste
- ouvrir ou fermer un menu
- modifier la couleur d’un bouton

---

# 3. La logique du DOM : un arbre d’éléments

Le DOM organise la page comme un **arbre**.

Prenons ce HTML :

```html
<!DOCTYPE html>
<html>
  <body>
    <h1>Bonjour</h1>
    <p>Bienvenue sur mon site</p>
    <button>Clique ici</button>
  </body>
</html>
```

Le navigateur le comprend un peu comme ceci :

- document
  - html
    - body
      - h1
      - p
      - button

Chaque élément de la page devient un **nœud** dans cet arbre.

## Vocabulaire utile

- **parent** : un élément qui contient d’autres éléments
- **enfant** : un élément contenu dans un autre
- **frères/sœurs** : des éléments au même niveau

Dans l’exemple :

- `body` est le parent de `h1`, `p` et `button`
- `h1`, `p` et `button` sont des enfants de `body`

Cette logique est importante, car elle aide à comprendre comment JavaScript repère et modifie les éléments.

---

# 4. Le point d’entrée : `document`

En JavaScript, l’objet principal pour accéder au DOM est `document`.

C’est lui qui représente la page HTML chargée dans le navigateur.

Exemple :

```js
console.log(document);
```

Cela permet de voir l’objet document dans la console.

On utilisera `document` pour :

- trouver des éléments
- créer des éléments
- modifier la page

---

# 5. Sélectionner des éléments du DOM

Avant de modifier un élément, il faut d’abord le **sélectionner**.

## a) `getElementById()`

Permet de récupérer un élément grâce à son `id`.

### HTML

```html
<h1 id="title">Bonjour</h1>
```

### JavaScript

```js
const title = document.getElementById("title");
console.log(title);
```

Ici, `title` contient l’élément `<h1>`.

---

## b) `querySelector()`

Permet de sélectionner le **premier élément** qui correspond à un sélecteur CSS.

```js
const title = document.querySelector("#title");
const button = document.querySelector("button");
const paragraph = document.querySelector(".message");
```

On peut utiliser :

- un `id` avec `#`
- une classe avec `.`
- une balise comme `button`, `p`, `h1`

---

## c) `querySelectorAll()`

Permet de sélectionner **tous les éléments** qui correspondent à un sélecteur.

### HTML

```html
<p class="item">Premier</p>
<p class="item">Deuxième</p>
<p class="item">Troisième</p>
```

### JavaScript

```js
const items = document.querySelectorAll(".item");
console.log(items);
```

Cette méthode retourne une collection sur laquelle on peut souvent utiliser `forEach`.

```js
items.forEach((item) => {
  console.log(item.textContent);
});
```

## À retenir

- `querySelector()` retourne un seul élément
- `querySelectorAll()` retourne plusieurs éléments

---

# 6. Lire et modifier le contenu d’un élément

Une fois l’élément sélectionné, on peut lire ou modifier son contenu.

## a) `textContent`

Permet de lire ou modifier le texte d’un élément.

### HTML

```html
<h1 id="title">Bonjour</h1>
```

### JavaScript

```js
const title = document.getElementById("title");

console.log(title.textContent); // Bonjour

title.textContent = "Salut tout le monde";
```

Le texte affiché sur la page change.

---

## b) `innerHTML`

Permet de lire ou modifier le contenu HTML à l’intérieur d’un élément.

### HTML

```html
<div id="box"></div>
```

### JavaScript

```js
const box = document.getElementById("box");
box.innerHTML = "<p>Un nouveau paragraphe</p>";
```

Cela ajoute du HTML à l’intérieur de `box`.

## Attention

`innerHTML` est pratique, mais il faut l’utiliser avec prudence.
Pour les débutants, il est souvent plus simple et plus sûr de commencer par `textContent`.

---

# 7. Modifier le style et les classes

## a) Modifier directement le style

```js
const title = document.getElementById("title");
title.style.color = "blue";
title.style.fontSize = "40px";
```

Cela fonctionne, mais ce n’est pas toujours la meilleure approche.

---

## b) Utiliser les classes avec `classList`

Souvent, il est préférable d’ajouter ou retirer des classes CSS.

### HTML

```html
<h1 id="title">Bonjour</h1>
```

### CSS

```css
.highlight {
  color: red;
  background-color: yellow;
}
```

### JavaScript

```js
const title = document.getElementById("title");
title.classList.add("highlight");
```

On peut aussi :

```js
title.classList.remove("highlight");
title.classList.toggle("highlight");
```

## Pourquoi `classList` est souvent préférable ?

Parce qu’on sépare mieux les rôles :

- JavaScript gère le comportement
- CSS gère l’apparence

---

# 8. Modifier les attributs

Les éléments HTML ont souvent des attributs, par exemple :

- `src`
- `href`
- `alt`
- `value`
- `placeholder`

## Exemple avec un lien

### HTML

```html
<a id="link" href="https://example.com">Visiter</a>
```

### JavaScript

```js
const link = document.getElementById("link");

console.log(link.getAttribute("href"));
link.setAttribute("href", "https://openai.com");
```

## Exemple avec une image

```js
const image = document.querySelector("img");
image.setAttribute("src", "photo2.jpg");
```

# 9. Réagir aux actions de l’utilisateur : les événements

Le DOM devient vraiment intéressant quand on y ajoute des **événements**.

Un événement est quelque chose qui se produit sur la page, par exemple :

- un clic
- une saisie dans un champ
- une soumission de formulaire
- le passage de la souris

## `addEventListener()`

Cette méthode permet d’écouter un événement.

### HTML

```html
<h1 id="title">Bonjour</h1>
<button id="btn">Changer le titre</button>
```

### JavaScript

```js
const title = document.getElementById("title");
const button = document.getElementById("btn");

button.addEventListener("click", () => {
  title.textContent = "Tu as cliqué sur le bouton";
});
```

Quand l’utilisateur clique sur le bouton, le texte change.

## Idée essentielle

Avec le DOM, JavaScript ne fait pas juste modifier une page.
Il peut aussi **attendre une action**, puis réagir.

---

# 10. Exemple simple d’interaction

### HTML

```html
<p id="message">Aucun clic pour le moment</p>
<button id="btn">Clique-moi</button>
```

### JavaScript

```js
const message = document.getElementById("message");
const button = document.getElementById("btn");

button.addEventListener("click", () => {
  message.textContent = "Le bouton a été cliqué";
});
```

## Ce qui se passe

1. JavaScript sélectionne le paragraphe
2. JavaScript sélectionne le bouton
3. Il écoute le clic sur le bouton
4. Quand le clic arrive, il modifie le texte du paragraphe

C’est une excellente première démonstration du DOM.

# 11. Créer et ajouter des éléments

Le DOM ne sert pas seulement à modifier des éléments existants.
Il permet aussi d’en créer de nouveaux.

## Exemple : ajouter un élément à une liste

### HTML

```html
<ul id="list">
  <li>Pomme</li>
</ul>

<button id="addBtn">Ajouter un fruit</button>
```

### JavaScript

```js
const list = document.getElementById("list");
const addBtn = document.getElementById("addBtn");

addBtn.addEventListener("click", () => {
  const newItem = document.createElement("li");
  newItem.textContent = "Banane";
  list.appendChild(newItem);
});
```

## Étapes importantes

- `createElement("li")` crée un nouvel élément
- `textContent` lui donne un texte
- `appendChild()` l’ajoute dans la page

---

# 12. Supprimer un élément

On peut aussi retirer un élément du DOM.

### HTML

```html
<p id="note">Ce message va disparaître</p>
<button id="removeBtn">Supprimer</button>
```

### JavaScript

```js
const note = document.getElementById("note");
const removeBtn = document.getElementById("removeBtn");

removeBtn.addEventListener("click", () => {
  note.remove();
});
```

---

# 13. Lire la valeur d’un champ

Les champs de formulaire sont très fréquents dans les manipulations DOM.

### HTML

```html
<input id="nameInput" type="text" placeholder="Votre nom" />
<button id="showBtn">Afficher</button>
<p id="result"></p>
```

### JavaScript

```js
const input = document.getElementById("nameInput");
const button = document.getElementById("showBtn");
const result = document.getElementById("result");

button.addEventListener("click", () => {
  result.textContent = "Bonjour " + input.value;
});
```

Ici, on récupère ce que l’utilisateur a écrit avec `input.value`.

# 14. Une logique simple pour travailler avec le DOM

Quand vous manipulez le DOM, pensez presque toujours en 4 étapes :

## 1. Sélectionner

Quel élément veux-je manipuler ?

```js
const title = document.getElementById("title");
```

## 2. Observer

Que contient-il déjà ?

```js
console.log(title.textContent);
```

## 3. Modifier

Qu’est-ce que je veux changer ?

```js
title.textContent = "Nouveau titre";
```

## 4. Réagir

Quand ce changement doit-il arriver ?

```js
button.addEventListener("click", () => {
  title.textContent = "Nouveau titre";
});
```

Cette logique aide beaucoup à ne pas se perdre.

# 15. Erreurs fréquentes chez les débutants

## a) Essayer de modifier un élément non sélectionné

```js
title.textContent = "Bonjour";
```

Ici, si `title` n’a jamais été défini, cela provoque une erreur.

Il faut d’abord sélectionner :

```js
const title = document.getElementById("title");
```

---

## b) Se tromper de sélecteur

```js
document.querySelector("title");
```

Cela cherche une balise `<title>`, pas un élément avec `id="title"`.

Pour un `id`, il faut écrire :

```js
document.querySelector("#title");
```

---

## c) Oublier que `querySelectorAll()` retourne plusieurs éléments

```js
const items = document.querySelectorAll(".item");
items.textContent = "Test";
```

Cela ne fonctionne pas, car `items` contient plusieurs éléments.

Il faut parcourir la collection :

```js
items.forEach((item) => {
  item.textContent = "Test";
});
```

---

## d) Exécuter le JavaScript avant que le HTML soit prêt

Si JavaScript s’exécute avant que les éléments HTML existent, la sélection peut échouer.

Une solution simple consiste à placer le script à la fin du `body` :

```html
<body>
  <h1 id="title">Bonjour</h1>
  <script src="script.js"></script>
</body>
```

On peut aussi écouter l’événement `DOMContentLoaded`, mais pour un premier cours, placer le script à la fin du `body` est souvent suffisant.

# 16. Exemple complet

Voici un petit exemple simple et cohérent.

### HTML

```html
<!DOCTYPE html>
<html lang="fr">
  <head>
    <meta charset="UTF-8" />
    <title>Exemple DOM</title>
    <style>
      .highlight {
        color: white;
        background-color: teal;
        padding: 10px;
      }
    </style>
  </head>
  <body>
    <h1 id="title">Bienvenue</h1>
    <button id="changeTextBtn">Changer le texte</button>
    <button id="toggleClassBtn">Ajouter ou retirer la classe</button>
    <button id="addItemBtn">Ajouter un élément</button>

    <ul id="list">
      <li>Élément 1</li>
    </ul>

    <script>
      const title = document.getElementById("title");
      const changeTextBtn = document.getElementById("changeTextBtn");
      const toggleClassBtn = document.getElementById("toggleClassBtn");
      const addItemBtn = document.getElementById("addItemBtn");
      const list = document.getElementById("list");

      changeTextBtn.addEventListener("click", () => {
        title.textContent = "Le titre a changé";
      });

      toggleClassBtn.addEventListener("click", () => {
        title.classList.toggle("highlight");
      });

      addItemBtn.addEventListener("click", () => {
        const newItem = document.createElement("li");
        newItem.textContent = "Nouvel élément";
        list.appendChild(newItem);
      });
    </script>
  </body>
</html>
```

## Ce que cet exemple montre

- la sélection d’éléments
- la modification du texte
- l’ajout ou le retrait d’une classe
- la création d’un élément
- l’ajout dans une liste
- la réaction aux clics

C’est un très bon premier laboratoire DOM.

# 17. Résumé du cours

Le DOM est la représentation de la page HTML sous forme d’objets manipulables par JavaScript.

Il permet de :

- sélectionner des éléments
- lire leur contenu
- modifier leur texte, leurs classes, leurs styles et leurs attributs
- réagir aux actions de l’utilisateur
- créer, ajouter et supprimer des éléments

## Les idées les plus importantes à retenir

- JavaScript manipule la page grâce au DOM
- le DOM organise la page comme un arbre
- on doit d’abord sélectionner un élément avant de le modifier
- les événements permettent de rendre la page interactive
- le DOM est au cœur du développement web dynamique

# 19. Conclusion

Comprendre le DOM, c’est comprendre comment une page web devient interactive.

Le HTML donne une structure à la page.
Le CSS lui donne une apparence.
Le DOM permet à JavaScript d’y intervenir.

C’est à partir du DOM que les étudiants commencent vraiment à voir le pouvoir de JavaScript dans le navigateur.
