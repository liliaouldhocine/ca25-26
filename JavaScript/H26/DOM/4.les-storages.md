# Cours 4 — Les storages en JavaScript

## Objectifs du cours

À la fin de ce cours, vous devriez être capables de :

- comprendre à quoi servent les storages dans le navigateur
- distinguer `localStorage` et `sessionStorage`
- enregistrer, lire, modifier et supprimer une donnée stockée
- comprendre que les données sont enregistrées sous forme de chaînes de caractères
- utiliser `JSON.stringify()` et `JSON.parse()` pour stocker des tableaux ou des objets
- relier le storage au DOM pour conserver un état dans une page web

---

# 1. Pourquoi ce cours vient maintenant

Dans les cours précédents, nous avons vu que JavaScript peut :

- sélectionner des éléments dans la page
- réagir aux événements
- lire des formulaires
- modifier l’interface
- afficher ou masquer des éléments

Mais jusqu’ici, toutes ces interactions restent **temporaires**.

Par exemple :

- un utilisateur écrit son nom
- un thème sombre est activé
- une liste de tâches est remplie

Tout cela fonctionne… jusqu’à ce que la page soit rechargée.

C’est là que les **storages** deviennent utiles.

Ils permettent de **conserver certaines données dans le navigateur**, pour que la page puisse les retrouver plus tard.

---

# 2. Qu’est-ce qu’un storage ?

Un storage est un espace de stockage fourni par le navigateur.

Il permet à JavaScript d’enregistrer des données directement dans le navigateur de l’utilisateur.

Ces données peuvent ensuite être relues plus tard.

## Idée importante

Le storage ne sert pas à afficher quelque chose par lui-même.
Il sert à **mémoriser une information**.

Ensuite, JavaScript peut utiliser cette information pour remettre la page dans le bon état.

---

# 3. Pourquoi les storages sont utiles

Les storages sont utiles quand on veut **retenir un choix ou une information** entre deux moments d’utilisation.

## Exemples concrets

On peut s’en servir pour :

- mémoriser un prénom entré dans un formulaire
- conserver une liste de tâches
- retenir un thème clair ou sombre
- sauvegarder un filtre choisi par l’utilisateur
- conserver un état d’interface même après rechargement

Autrement dit, le storage aide la page à **se souvenir**.

---

# 4. Les deux storages les plus utilisés

En JavaScript, pour débuter, on rencontre surtout :

- `localStorage`
- `sessionStorage`

Ils fonctionnent presque de la même manière, mais ils ne gardent pas les données aussi longtemps.

---

# 5. `localStorage`

`localStorage` conserve les données même si :

- on recharge la page
- on ferme l’onglet
- on ferme le navigateur
- on revient plus tard sur le site

Les données restent enregistrées jusqu’à ce qu’on les supprime.

## Idée simple

`localStorage` sert à garder une donnée **sur une durée plus longue**.

---

# 6. `sessionStorage`

`sessionStorage` conserve les données seulement pendant la session en cours.

En pratique :

- si on recharge la page, les données restent
- si on ferme l’onglet, les données disparaissent

## Idée simple

`sessionStorage` sert à garder une donnée **temporairement**, pendant que l’onglet est ouvert.

---

# 7. Différence entre les deux

## `localStorage`

Les données restent disponibles même après fermeture du navigateur.

## `sessionStorage`

Les données disparaissent lorsque la session de l’onglet se termine.

## À retenir

- `localStorage` : plus durable
- `sessionStorage` : plus temporaire

---

# 8. La logique générale du storage

Quand on utilise un storage, on travaille souvent avec une logique très simple :

1. on choisit une **clé**
2. on associe une **valeur**
3. on enregistre cette valeur
4. on la relit plus tard avec la même clé

Exemple d’idée :

- clé : `"username"`
- valeur : `"Lina"`

Le navigateur peut alors retenir que `"username"` correspond à `"Lina"`.

---

# 9. Important : les valeurs sont stockées comme du texte

Dans les storages, les données sont enregistrées sous forme de **chaînes de caractères**.

Cela veut dire que même si on stocke un nombre ou un booléen, le navigateur l’enregistre comme du texte.

C’est très important à comprendre.

---

# 10. Enregistrer une donnée avec `setItem()`

Pour enregistrer une donnée, on utilise :

```js
localStorage.setItem("username", "Lina");
```

Ici :

- `"username"` est la clé
- `"Lina"` est la valeur enregistrée

On peut ensuite aller la relire plus tard.

---

# 11. Lire une donnée avec `getItem()`

Pour lire une donnée, on utilise :

```js
const name = localStorage.getItem("username");
console.log(name);
```

Si la clé existe, JavaScript récupère la valeur.

Ici, `name` va contenir `"Lina"`.

---

# 12. Supprimer une donnée avec `removeItem()`

Pour supprimer une donnée précise :

```js
localStorage.removeItem("username");
```

Cela enlève uniquement la donnée associée à cette clé.

---

# 13. Vider complètement le storage avec `clear()`

Pour supprimer toutes les données du storage concerné :

```js
localStorage.clear();
```

Attention : cela supprime tout ce qui est enregistré dans ce `localStorage` pour le site concerné.

---

# 14. Premier exemple simple

### JavaScript

```js
localStorage.setItem("city", "Montréal");

const city = localStorage.getItem("city");
console.log(city);
```

## Ce que fait ce code

- on enregistre `"Montréal"` sous la clé `"city"`
- on relit la valeur
- on l’affiche dans la console

---

# 15. Exemple avec le DOM

Le storage devient plus intéressant quand on le relie à la page.

### HTML

```html
<input id="nameInput" type="text" placeholder="Votre prénom" />
<button id="saveBtn">Enregistrer</button>
<p id="message"></p>
```

### JavaScript

```js
const nameInput = document.getElementById("nameInput");
const saveBtn = document.getElementById("saveBtn");
const message = document.getElementById("message");

saveBtn.addEventListener("click", () => {
  localStorage.setItem("firstName", nameInput.value);
  message.textContent = "Prénom enregistré.";
});
```

## Ce que cela permet

Quand l’utilisateur clique, son prénom est enregistré dans le navigateur.

---

# 16. Relire la donnée au chargement de la page

Le vrai intérêt du storage apparaît quand on relit la donnée plus tard.

### JavaScript

```js
const savedName = localStorage.getItem("firstName");

if (savedName !== null) {
  message.textContent = "Bonjour " + savedName;
}
```

## Idée importante

La page peut maintenant retrouver une information d’une visite précédente ou d’un rechargement.

---

# 17. Exemple complet : mémoriser un prénom

### HTML

```html
<!DOCTYPE html>
<html lang="fr">
  <head>
    <meta charset="UTF-8" />
    <title>Storage simple</title>
  </head>
  <body>
    <input id="nameInput" type="text" placeholder="Votre prénom" />
    <button id="saveBtn">Enregistrer</button>
    <p id="message"></p>

    <script>
      const nameInput = document.getElementById("nameInput");
      const saveBtn = document.getElementById("saveBtn");
      const message = document.getElementById("message");

      const savedName = localStorage.getItem("firstName");

      if (savedName !== null) {
        message.textContent = "Bonjour " + savedName;
      }

      saveBtn.addEventListener("click", () => {
        localStorage.setItem("firstName", nameInput.value);
        message.textContent = "Bonjour " + nameInput.value;
      });
    </script>
  </body>
</html>
```

## Ce que montre cet exemple

- lecture du storage au chargement
- enregistrement d’une valeur
- affichage de cette valeur dans la page

C’est un excellent premier exemple pédagogique.

---

# 18. Utiliser `sessionStorage`

`sessionStorage` fonctionne presque pareil.

### Exemple

```js
sessionStorage.setItem("theme", "dark");

const theme = sessionStorage.getItem("theme");
console.log(theme);
```

Ici, la donnée est conservée tant que l’onglet reste ouvert.

---

# 19. Quand utiliser `localStorage` ou `sessionStorage` ?

## Utiliser `localStorage` quand :

- on veut garder une donnée plus longtemps
- on veut qu’elle survive à la fermeture du navigateur
- on veut retrouver une préférence utilisateur plus tard

## Utiliser `sessionStorage` quand :

- la donnée n’est utile que temporairement
- elle doit disparaître à la fermeture de l’onglet
- on veut garder un état seulement pendant la session actuelle

---

# 20. Attention : les storages ne stockent pas directement les objets

Les storages stockent du texte.

Donc ce code pose problème :

```js
const user = {
  name: "Lina",
  age: 17,
};

localStorage.setItem("user", user);
```

Ce n’est pas la bonne manière de stocker un objet.

Pour stocker un objet ou un tableau, il faut d’abord le transformer en texte JSON.

---

# 21. `JSON.stringify()`

`JSON.stringify()` transforme un objet ou un tableau en chaîne de caractères.

### Exemple

```js
const user = {
  name: "Lina",
  age: 17,
};

const userString = JSON.stringify(user);
console.log(userString);
```

Résultat :

```js
{"name":"Lina","age":17}
```

C’est cette chaîne que l’on peut stocker.

---

# 22. Stocker un objet dans `localStorage`

### JavaScript

```js
const user = {
  name: "Lina",
  age: 17,
};

localStorage.setItem("user", JSON.stringify(user));
```

Ici, l’objet est converti en texte avant d’être enregistré.

---

# 23. `JSON.parse()`

Quand on relit une donnée JSON, on récupère du texte.

Pour retrouver un vrai objet JavaScript, il faut utiliser `JSON.parse()`.

### JavaScript

```js
const savedUser = localStorage.getItem("user");
const userObject = JSON.parse(savedUser);

console.log(userObject.name);
```

---

# 24. Stocker un tableau

Le même principe fonctionne avec un tableau.

### JavaScript

```js
const fruits = ["pomme", "banane", "orange"];

localStorage.setItem("fruits", JSON.stringify(fruits));
```

Puis pour le relire :

```js
const savedFruits = localStorage.getItem("fruits");
const fruitsArray = JSON.parse(savedFruits);

console.log(fruitsArray);
```

---

# 25. Pourquoi `JSON.stringify()` et `JSON.parse()` sont importants

Sans eux, le storage est surtout pratique pour de petites valeurs simples :

- un nom
- un thème
- un état
- une préférence

Avec eux, on peut aussi mémoriser :

- des objets
- des tableaux
- une liste de tâches
- une petite configuration utilisateur

C’est une étape très importante pour rendre le storage vraiment utile.

---

# 26. Exemple : sauvegarder une liste de tâches

### JavaScript

```js
const tasks = ["Faire les devoirs", "Lire", "Réviser"];
localStorage.setItem("tasks", JSON.stringify(tasks));
```

Puis :

```js
const savedTasks = localStorage.getItem("tasks");

if (savedTasks !== null) {
  const tasksArray = JSON.parse(savedTasks);
  console.log(tasksArray);
}
```

---

# 27. Exemple complet simple avec une liste

### HTML

```html
<input id="taskInput" type="text" placeholder="Nouvelle tâche" />
<button id="addBtn">Ajouter</button>
<ul id="taskList"></ul>
```

### JavaScript

```js
const taskInput = document.getElementById("taskInput");
const addBtn = document.getElementById("addBtn");
const taskList = document.getElementById("taskList");

let tasks = [];

const savedTasks = localStorage.getItem("tasks");

if (savedTasks !== null) {
  tasks = JSON.parse(savedTasks);
  tasks.forEach((task) => {
    const li = document.createElement("li");
    li.textContent = task;
    taskList.appendChild(li);
  });
}

addBtn.addEventListener("click", () => {
  if (taskInput.value !== "") {
    tasks.push(taskInput.value);
    localStorage.setItem("tasks", JSON.stringify(tasks));

    const li = document.createElement("li");
    li.textContent = taskInput.value;
    taskList.appendChild(li);

    taskInput.value = "";
  }
});
```

## Ce que cet exemple montre

- lecture d’un tableau stocké
- conversion avec `JSON.parse()`
- affichage des éléments dans le DOM
- ajout d’une nouvelle tâche
- mise à jour du tableau
- réenregistrement dans `localStorage`

C’est un très bon pont entre DOM, événements, formulaires et storage.

---

# 28. Vérifier si une donnée existe

Quand on lit une clé qui n’existe pas, `getItem()` retourne `null`.

### Exemple

```js
const savedTheme = localStorage.getItem("theme");

if (savedTheme !== null) {
  console.log("Un thème est enregistré.");
}
```

Cette vérification est très importante pour éviter des erreurs.

---

# 29. Exemple : mémoriser un thème

### HTML

```html
<button id="themeBtn">Changer le thème</button>
```

### JavaScript

```js
const themeBtn = document.getElementById("themeBtn");

const savedTheme = localStorage.getItem("theme");

if (savedTheme === "dark") {
  document.body.classList.add("dark");
}

themeBtn.addEventListener("click", () => {
  document.body.classList.toggle("dark");

  if (document.body.classList.contains("dark")) {
    localStorage.setItem("theme", "dark");
  } else {
    localStorage.setItem("theme", "light");
  }
});
```

## Ce que cela permet

Le choix du thème est conservé après rechargement de la page.

---

# 30. Ce qu’il ne faut pas stocker dans `localStorage`

Il est important de ne pas présenter `localStorage` comme un espace sûr pour tout.

Il ne faut pas y stocker des informations sensibles, par exemple :

- des mots de passe
- des données secrètes
- des informations très privées

## Pourquoi ?

Parce que ce storage est accessible côté navigateur.
Ce n’est pas un endroit fait pour protéger des données sensibles.

---

# 31. Limites importantes du storage

Pour des débutants, voici les limites les plus importantes à connaître :

## 1. Les données sont stockées comme du texte

Il faut convertir les objets et tableaux avec JSON.

## 2. Ce n’est pas une base de données

C’est pratique pour de petites données simples, pas pour de gros systèmes complexes.

## 3. Ce n’est pas un espace sécurisé

On évite les données sensibles.

## 4. Le storage est lié au site

Un site ne lit pas directement le storage d’un autre site.

---

# 32. Différence avec une variable JavaScript

Une variable classique :

```js
let userName = "Lina";
```

n’existe que pendant l’exécution actuelle du script.

Mais une donnée enregistrée dans `localStorage` :

```js
localStorage.setItem("userName", "Lina");
```

peut être retrouvée plus tard, même après rechargement.

## Idée essentielle

- variable : temporaire en mémoire
- storage : mémorisé dans le navigateur

---

# 33. Quelques méthodes utiles à connaître

En plus de `setItem()`, `getItem()`, `removeItem()` et `clear()`, on peut rencontrer :

## `localStorage.length`

Indique combien d’éléments sont enregistrés.

```js
console.log(localStorage.length);
```

## `localStorage.key(index)`

Permet d’obtenir la clé située à une certaine position.

```js
console.log(localStorage.key(0));
```

Pour un premier cours, ces méthodes sont moins essentielles, mais elles peuvent être mentionnées.

---

# 34. Erreurs fréquentes chez les débutants

## a) Oublier que le storage enregistre du texte

Exemple problématique :

```js
localStorage.setItem("age", 17);
```

Cela fonctionne, mais la valeur sera relue comme une chaîne.

---

## b) Oublier `JSON.stringify()`

Si on essaie de stocker directement un tableau ou un objet, le résultat ne sera pas celui attendu.

---

## c) Oublier `JSON.parse()`

Si on relit un tableau JSON sans le parser, on récupère du texte, pas un vrai tableau JavaScript.

---

## d) Oublier de vérifier `null`

Lire une clé inexistante sans vérification peut compliquer le code.

---

## e) Confondre storage et DOM

Le storage conserve la donnée.
Le DOM affiche ou modifie la page.

Ils travaillent souvent ensemble, mais ce ne sont pas la même chose.

---

# 35. Méthode de travail recommandée

Quand vous utilisez le storage, vous pouvez suivre cette logique :

## 1. Identifier la donnée à mémoriser

Exemple : nom, thème, liste de tâches.

## 2. Choisir le bon storage

`localStorage` ou `sessionStorage`.

## 3. Choisir une clé claire

Exemple : `"theme"`, `"tasks"`, `"firstName"`.

## 4. Enregistrer la donnée

Avec `setItem()`.

## 5. Relire la donnée au bon moment

Souvent au chargement de la page.

## 6. Mettre à jour le DOM

Pour afficher la donnée ou remettre l’interface dans le bon état.

---

# 36. Exemple intégrateur

Voici un exemple simple qui relie formulaire, DOM et `localStorage`.

### HTML

```html
<!DOCTYPE html>
<html lang="fr">
  <head>
    <meta charset="UTF-8" />
    <title>Exemple storage</title>
  </head>
  <body>
    <form id="nameForm">
      <input id="nameInput" type="text" placeholder="Votre prénom" />
      <button type="submit">Enregistrer</button>
    </form>

    <p id="message"></p>

    <script>
      const nameForm = document.getElementById("nameForm");
      const nameInput = document.getElementById("nameInput");
      const message = document.getElementById("message");

      const savedName = localStorage.getItem("firstName");

      if (savedName !== null) {
        message.textContent = "Bienvenue " + savedName;
        nameInput.value = savedName;
      }

      nameForm.addEventListener("submit", (event) => {
        event.preventDefault();

        if (nameInput.value !== "") {
          localStorage.setItem("firstName", nameInput.value);
          message.textContent = "Bienvenue " + nameInput.value;
        }
      });
    </script>
  </body>
</html>
```

## Ce que cet exemple regroupe

- formulaire
- événement `submit`
- `preventDefault()`
- lecture d’un input
- stockage dans `localStorage`
- récupération au chargement
- mise à jour du DOM

# 37. Résumé du cours

Les storages permettent à JavaScript de conserver certaines données dans le navigateur.

Nous avons vu que :

- `localStorage` garde les données plus longtemps
- `sessionStorage` garde les données pendant la session
- les données sont stockées sous forme de texte
- `setItem()` enregistre
- `getItem()` lit
- `removeItem()` supprime une donnée
- `clear()` vide tout le storage
- `JSON.stringify()` et `JSON.parse()` permettent de stocker tableaux et objets
- le storage devient vraiment utile lorsqu’il est lié au DOM

## Les notions les plus importantes

- le storage sert à mémoriser une donnée dans le navigateur
- `localStorage` et `sessionStorage` ne gardent pas les données aussi longtemps

- les valeurs sont stockées comme du texte
- pour les objets et tableaux, il faut utiliser JSON
- le storage permet à une page de retrouver un état après rechargement

# 39. Mini synthèse des méthodes essentielles

## Enregistrer une valeur

```js
localStorage.setItem("key", "value");
```

## Lire une valeur

```js
localStorage.getItem("key");
```

## Supprimer une valeur

```js
localStorage.removeItem("key");
```

## Tout supprimer

```js
localStorage.clear();
```

## Stocker un objet ou un tableau

```js
localStorage.setItem("tasks", JSON.stringify(tasks));
```

## Relire un objet ou un tableau

```js
const tasks = JSON.parse(localStorage.getItem("tasks"));
```
