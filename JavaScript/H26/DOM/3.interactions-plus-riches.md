# Interactions plus riches avec le DOM en JavaScript

## Objectifs du cours

À la fin de ce cours, vous devriez être capables de :

- afficher et masquer des éléments dans une page
- créer des interactions plus dynamiques avec le DOM
- gérer plusieurs événements sur une même interface
- comprendre l’intérêt de `classList.toggle()`
- organiser la logique d’une page interactive
- découvrir la propagation des événements
- comprendre l’idée générale de la délégation d’événements

---

# 1. Pourquoi ce cours vient après les événements et les formulaires

Dans les cours précédents, nous avons vu que JavaScript peut :

- sélectionner des éléments
- modifier leur contenu
- réagir à un clic ou à une saisie
- lire des valeurs dans un formulaire
- afficher des messages

Mais dans une vraie interface, on veut souvent aller un peu plus loin.

Par exemple, on veut :

- ouvrir ou fermer un menu
- afficher un message au bon moment
- masquer une section inutile
- changer l’état d’un élément
- faire réagir plusieurs parties de la page ensemble

Autrement dit, on ne veut plus seulement une page qui réagit une fois.
On veut une page qui **change d’état**.

C’est exactement l’objectif de ce cours.

---

# 2. Une idée centrale : l’interface a des états

Dans beaucoup d’interfaces, un élément peut avoir plusieurs états.

Par exemple :

- visible / caché
- ouvert / fermé
- actif / inactif
- sélectionné / non sélectionné
- terminé / non terminé

Le rôle de JavaScript est souvent de faire passer un élément d’un état à un autre.

## Exemple simple

Un paragraphe peut être :

- affiché au début
- caché après un clic
- réaffiché après un autre clic

On ne change pas seulement son texte.
On change son **état dans l’interface**.

---

# 3. Afficher ou masquer un élément

Une des premières interactions riches à apprendre consiste à afficher ou masquer un élément.

## Exemple

### HTML

```html
<p id="info">Voici une information importante.</p>
<button id="toggleBtn">Afficher / masquer</button>
```

### JavaScript

```js
const info = document.getElementById("info");
const toggleBtn = document.getElementById("toggleBtn");

toggleBtn.addEventListener("click", () => {
  if (info.style.display === "none") {
    info.style.display = "block";
  } else {
    info.style.display = "none";
  }
});
```

## Ce que fait ce code

- on sélectionne le paragraphe
- on sélectionne le bouton
- à chaque clic, on vérifie l’état actuel
- si l’élément est caché, on l’affiche
- sinon, on le masque

---

# 4. Pourquoi cette approche fonctionne, mais n’est pas toujours idéale

Modifier directement `style.display` fonctionne bien pour comprendre le principe.

Cependant, dans un projet plus propre, on préfère souvent utiliser des classes CSS.

Cela permet de mieux séparer :

- le comportement en JavaScript
- l’apparence en CSS

---

# 5. Afficher ou masquer avec une classe CSS

## HTML

```html
<p id="info">Voici une information importante.</p>
<button id="toggleBtn">Afficher / masquer</button>
```

## CSS

```css
.hidden {
  display: none;
}
```

## JavaScript

```js
const info = document.getElementById("info");
const toggleBtn = document.getElementById("toggleBtn");

toggleBtn.addEventListener("click", () => {
  info.classList.toggle("hidden");
});
```

## Pourquoi cette version est souvent préférable

Parce que :

- JavaScript déclenche le changement
- CSS définit ce que signifie “caché”

Le code est souvent plus clair et plus facile à maintenir.

---

# 6. `classList.toggle()` : une méthode très utile

`toggle()` est très pratique pour gérer les états.

Il signifie :

- si la classe est absente, on l’ajoute
- si la classe est présente, on la retire

C’est très utile pour :

- afficher / masquer
- activer / désactiver
- sélectionner / désélectionner
- ouvrir / fermer

## Exemple

```js
menu.classList.toggle("open");
```

Ici, un menu peut devenir ouvert, puis fermé, puis ouvert à nouveau.

---

# 7. Changer le texte selon l’état

Une interface plus riche ne change pas seulement l’apparence.
Elle peut aussi changer le texte selon ce qui se passe.

## Exemple

### HTML

```html
<p id="details">Détails visibles</p>
<button id="toggleBtn">Masquer</button>
```

### JavaScript

```js
const details = document.getElementById("details");
const toggleBtn = document.getElementById("toggleBtn");

toggleBtn.addEventListener("click", () => {
  if (details.style.display === "none") {
    details.style.display = "block";
    toggleBtn.textContent = "Masquer";
  } else {
    details.style.display = "none";
    toggleBtn.textContent = "Afficher";
  }
});
```

## Idée importante

L’interface devient plus claire pour l’utilisateur, car le bouton indique l’action suivante.

---

# 8. Gérer plusieurs événements sur un même élément

Un même élément peut réagir à plusieurs types d’événements.

## Exemple

### HTML

```html
<button id="boxBtn">Passez la souris ou cliquez</button>
<p id="message"></p>
```

### JavaScript

```js
const boxBtn = document.getElementById("boxBtn");
const message = document.getElementById("message");

boxBtn.addEventListener("mouseover", () => {
  message.textContent = "La souris est sur le bouton.";
});

boxBtn.addEventListener("mouseout", () => {
  message.textContent = "La souris a quitté le bouton.";
});

boxBtn.addEventListener("click", () => {
  message.textContent = "Le bouton a été cliqué.";
});
```

## Ce que cela montre

Le même bouton peut :

- réagir au survol
- réagir au départ de la souris
- réagir au clic

Cela permet de créer des interfaces plus vivantes.

---

# 9. Gérer plusieurs éléments sur une même page

Dans une vraie page, il n’y a pas qu’un seul bouton ou un seul champ.

On peut avoir plusieurs éléments interactifs à gérer en même temps.

## Exemple

### HTML

```html
<button id="showBtn">Afficher le message</button>
<button id="hideBtn">Masquer le message</button>
<p id="message">Bonjour tout le monde</p>
```

### JavaScript

```js
const showBtn = document.getElementById("showBtn");
const hideBtn = document.getElementById("hideBtn");
const message = document.getElementById("message");

showBtn.addEventListener("click", () => {
  message.style.display = "block";
});

hideBtn.addEventListener("click", () => {
  message.style.display = "none";
});
```

## Ce qu’il faut comprendre

Chaque événement peut avoir une responsabilité différente :

- un bouton affiche
- un autre masque
- le paragraphe reçoit le changement

---

# 10. Organiser sa logique d’interaction

Quand une page devient un peu plus riche, il est important d’organiser sa pensée.

Une bonne méthode consiste à se poser ces questions :

## 1. Quel élément change ?

Exemple : un menu, un paragraphe, une carte, une liste.

## 2. Quel événement déclenche ce changement ?

Exemple : clic, saisie, survol.

## 3. Quel état faut-il appliquer ?

Exemple : visible, caché, actif, terminé.

## 4. Comment représenter cet état ?

Exemple : avec `textContent`, `style`, une classe CSS, un attribut.

Cette logique aide beaucoup à ne pas se perdre.

---

# 11. Exemple : ouvrir et fermer une section

### HTML

```html
<h2 id="title">Section d’aide</h2>
<button id="toggleHelpBtn">Afficher l’aide</button>
<p id="helpText" class="hidden">
  Voici des informations supplémentaires pour l’utilisateur.
</p>
```

### CSS

```css
.hidden {
  display: none;
}
```

### JavaScript

```js
const toggleHelpBtn = document.getElementById("toggleHelpBtn");
const helpText = document.getElementById("helpText");

toggleHelpBtn.addEventListener("click", () => {
  helpText.classList.toggle("hidden");

  if (helpText.classList.contains("hidden")) {
    toggleHelpBtn.textContent = "Afficher l’aide";
  } else {
    toggleHelpBtn.textContent = "Masquer l’aide";
  }
});
```

## Nouvelle idée introduite ici

`classList.contains("hidden")` permet de vérifier si une classe est présente.

Cela est très utile pour connaître l’état actuel d’un élément.

---

# 12. Utiliser `classList.contains()`

Cette méthode permet de répondre à une question simple :

> Est-ce que cette classe est présente ?

## Exemple

```js
if (box.classList.contains("active")) {
  console.log("La boîte est active");
}
```

Cette méthode est utile quand on veut :

- adapter le texte d’un bouton
- vérifier un état avant d’agir
- prendre une décision selon la classe actuelle

# 13. Exemple : sélectionner une carte


## Ce que les étudiants doivent voir

Une interaction riche, ce n’est souvent qu’une suite simple :

- un clic

# 14. Gérer plusieurs éléments semblables

Lorsqu’on a plusieurs éléments du même type, on peut vouloir leur appliquer la même logique.

## Exemple

### HTML

```html
<p class="item">Élément 1</p>
<p class="item">Élément 2</p>
<p class="item">Élément 3</p>
<button id="highlightBtn">Mettre en évidence</button>
```

### JavaScript

```js
const items = document.querySelectorAll(".item");
const highlightBtn = document.getElementById("highlightBtn");

highlightBtn.addEventListener("click", () => {
  items.forEach((item) => {
    item.classList.toggle("highlight");
  });
});
```

## Ce que cela montre

On peut sélectionner plusieurs éléments avec `querySelectorAll()` puis les parcourir avec `forEach()` pour leur appliquer la même transformation.

# 16. Introduire la propagation des événements

Lorsque l’on clique sur un élément, l’événement ne concerne pas toujours seulement cet élément.

Il peut aussi “remonter” dans les éléments parents.

C’est ce qu’on appelle la **propagation des événements**.

Pour des débutants, il n’est pas nécessaire d’entrer trop loin dans les détails, mais l’idée mérite d’être introduite.

---

# 17. Exemple simple de propagation

### HTML

```html
<div id="parent">
  <button id="childBtn">Clique ici</button>
</div>
```

### JavaScript

```js
const parent = document.getElementById("parent");
const childBtn = document.getElementById("childBtn");

parent.addEventListener("click", () => {
  console.log("Le parent a reçu le clic");
});

childBtn.addEventListener("click", () => {
  console.log("Le bouton a reçu le clic");
});
```

## Que se passe-t-il ?

Quand on clique sur le bouton :

- le bouton reçoit le clic
- puis le parent reçoit aussi ce clic

C’est une première illustration de la propagation.

---

# 18. Pourquoi cette idée est utile

Parce que dans certaines interfaces :

- un clic sur un enfant déclenche aussi un effet sur le parent
- cela peut être utile
- mais cela peut aussi surprendre si on ne s’y attend pas

Le plus important à ce niveau est simplement de savoir que cela existe.

---

# 19. Empêcher la propagation avec `stopPropagation()`

Dans certains cas, on veut empêcher un événement de remonter au parent.

### Exemple

```js
childBtn.addEventListener("click", (event) => {
  event.stopPropagation();
  console.log("Le bouton a reçu le clic");
});
```

## Effet

Le clic reste lié au bouton et ne remonte pas au parent.

## Remarque pédagogique

Pour un premier contact, il suffit d’expliquer que :

- `preventDefault()` empêche le comportement automatique
- `stopPropagation()` empêche la remontée de l’événement

Les deux ont des rôles différents.

---

# 20. Introduire doucement la délégation d’événements

La délégation d’événements est une idée très utile lorsqu’on a beaucoup d’éléments semblables, ou lorsque des éléments sont ajoutés dynamiquement.

L’idée générale est la suivante :

> Au lieu d’ajouter un écouteur à chaque enfant, on place parfois un seul écouteur sur le parent.

Pour des novices, il ne faut pas la rendre trop technique.
Il suffit de montrer à quoi elle sert.

---

# 21. Pourquoi la délégation peut être utile

Imaginons une liste de tâches :

- les tâches existent déjà au chargement
- ou bien elles sont ajoutées plus tard avec JavaScript

Au lieu de mettre un écouteur sur chaque tâche, on peut parfois écouter le clic sur toute la liste, puis déterminer quel élément a été visé.

Cela évite de répéter beaucoup de code.

---

# 22. Exemple simple d’idée de délégation

### HTML

```html
<ul id="taskList">
  <li>Tâche 1</li>
  <li>Tâche 2</li>
  <li>Tâche 3</li>
</ul>
```

### JavaScript

```js
const taskList = document.getElementById("taskList");

taskList.addEventListener("click", (event) => {
  if (event.target.tagName === "LI") {
    event.target.classList.toggle("done");
  }
});
```

## Ce qu’il faut comprendre ici

- l’écouteur est placé sur `ul`
- mais le clic peut venir d’un `li`
- `event.target` représente l’élément réellement cliqué

Pour l’instant, l’objectif n’est pas de tout maîtriser, mais de voir qu’il existe une autre manière de gérer les événements.

---

# 23. Ce qu’il faut retenir de la délégation à ce stade

Pour des étudiants débutants, il suffit de retenir ceci :

- on peut écouter un événement sur un parent
- puis vérifier quel enfant a été cliqué
- cela est souvent utile pour les listes ou les éléments créés dynamiquement

C’est déjà une excellente base.

# 24. Exemple intégrateur

Voici un exemple qui regroupe plusieurs idées du cours.

### HTML

```html
<!DOCTYPE html>
<html lang="fr">
  <head>
    <meta charset="UTF-8" />
    <title>Interactions riches</title>
    <style>
      .hidden {
        display: none;
      }

      .selected {
        background-color: lightgray;
        padding: 10px;
        font-weight: bold;
      }
    </style>
  </head>
  <body>
    <button id="toggleSectionBtn">Afficher la section</button>

    <div id="section" class="hidden">
      <p id="sectionText">Contenu de la section.</p>
      <button id="selectTextBtn">Sélectionner le texte</button>
    </div>

    <script>
      const toggleSectionBtn = document.getElementById("toggleSectionBtn");
      const section = document.getElementById("section");
      const sectionText = document.getElementById("sectionText");
      const selectTextBtn = document.getElementById("selectTextBtn");

      toggleSectionBtn.addEventListener("click", () => {
        section.classList.toggle("hidden");

        if (section.classList.contains("hidden")) {
          toggleSectionBtn.textContent = "Afficher la section";
        } else {
          toggleSectionBtn.textContent = "Masquer la section";
        }
      });

      selectTextBtn.addEventListener("click", () => {
        sectionText.classList.toggle("selected");

        if (sectionText.classList.contains("selected")) {
          selectTextBtn.textContent = "Désélectionner le texte";
        } else {
          selectTextBtn.textContent = "Sélectionner le texte";
        }
      });
    </script>
  </body>
</html>
```

## Ce que cet exemple montre

- afficher / masquer une section
- utiliser une classe CSS pour représenter un état
- vérifier une classe avec `contains()`
- changer le texte d’un bouton selon l’état
- créer plusieurs interactions dans la même page

# 25. Erreurs fréquentes chez les débutants

## a) Oublier quel élément change réellement

Parfois, l’étudiant modifie le bouton alors qu’il voulait modifier le paragraphe, ou inversement.

Il faut toujours se demander :

> Quel élément doit être transformé ?

---

## b) Mélanger le comportement et le style

Écrire tout avec `style` fonctionne, mais devient vite difficile à gérer.

Il est souvent préférable d’utiliser des classes CSS.

---

## c) Se perdre entre plusieurs écouteurs

Quand plusieurs éléments réagissent, il faut bien nommer les variables et garder une logique claire.

Par exemple :

- `toggleBtn`
- `message`
- `menu`
- `card`
- `taskList`

De bons noms aident beaucoup.

---

## d) Ne pas comprendre pourquoi un parent réagit aussi

Quand cela arrive, il faut penser à la propagation des événements.

---

## e) Oublier que `toggle()` change l’état à chaque fois

Si un étudiant clique plusieurs fois et pense que le comportement est “incohérent”, il faut lui rappeler que `toggle()` alterne entre deux états.

# 26. Méthode de travail recommandée

Pour construire une interaction un peu plus riche, on peut suivre cette méthode :

## 1. Identifier l’élément concerné

Que veut-on afficher, masquer ou modifier ?

## 2. Identifier l’événement

Quel geste utilisateur déclenche l’action ?

## 3. Identifier l’état visé

Visible ? Caché ? Sélectionné ? Actif ?

## 4. Choisir la technique

Changer du texte ? Ajouter une classe ? Vérifier `contains()` ? Utiliser `toggle()` ?

## 5. Tester plusieurs fois

Une interaction doit être testée plus d’une fois pour vérifier les deux états.
