# Les événements et les formulaires en JavaScript

## Objectifs du cours

À la fin de ce cours, vous devriez être capables de :

- comprendre ce qu’est un événement en JavaScript
- expliquer à quoi servent les événements dans une page web
- utiliser `addEventListener()`
- réagir à des actions comme un clic, une saisie ou une soumission
- lire les valeurs d’un formulaire
- empêcher le comportement par défaut d’un formulaire
- valider des champs simples avec JavaScript

---

# 1. Pourquoi ce cours vient juste après le DOM

Dans le cours précédent, nous avons vu que le DOM permet à JavaScript de :

- sélectionner des éléments
- lire leur contenu
- les modifier
- en créer de nouveaux

Mais une page web ne devient vraiment interactive que lorsqu’elle **réagit à l’utilisateur**.

C’est ici que les **événements** entrent en jeu.

Un événement est une action qui se produit dans la page, par exemple :

- un clic sur un bouton
- du texte saisi dans un champ
- un formulaire envoyé
- une touche du clavier pressée
- la souris qui passe sur un élément

Le rôle de JavaScript est alors de dire :

> Quand cette action arrive, exécute ce code.

---

# 2. Qu’est-ce qu’un événement ?

Un **événement** est quelque chose qui se produit dans la page web.

Par exemple :

- l’utilisateur clique sur un bouton
- l’utilisateur écrit dans un champ
- l’utilisateur soumet un formulaire

Ces actions peuvent être détectées par JavaScript.

## Idée importante

Le DOM représente la page.
Les événements représentent ce qui **se passe sur la page**.

Autrement dit :

- avec le DOM, on manipule les éléments
- avec les événements, on sait **quand** agir

---

# 3. La méthode principale : `addEventListener()`

Pour écouter un événement, on utilise généralement :

```js
element.addEventListener("typeEvenement", fonction);
```

Cette méthode signifie :

> Sur cet élément, écoute cet événement, puis exécute cette fonction.

---

# 4. Premier exemple : écouter un clic

### HTML

```html
<h1 id="title">Bonjour</h1>
<button id="btn">Clique ici</button>
```

### JavaScript

```js
const title = document.getElementById("title");
const button = document.getElementById("btn");

button.addEventListener("click", () => {
  title.textContent = "Tu as cliqué sur le bouton";
});
```

## Ce qui se passe

1. on sélectionne le titre
2. on sélectionne le bouton
3. on écoute le clic sur le bouton
4. lorsque le clic arrive, on change le texte du titre

C’est l’un des exemples les plus simples et les plus importants pour commencer.

---

# 5. Pourquoi utiliser `addEventListener()` ?

Parce qu’une page web n’est pas seulement là pour afficher de l’information.
Elle doit souvent **répondre à l’utilisateur**.

Par exemple :

- ouvrir un menu
- changer un texte
- afficher un message
- vérifier un formulaire
- ajouter un élément à une liste

Les événements permettent ce comportement interactif.

---

# 6. Les événements les plus utiles pour débuter

Pour des étudiants novices, les plus utiles sont souvent :

- `click`
- `input`
- `change`
- `submit`

Nous allons les voir progressivement.

---

# 7. L’événement `click`

`click` se produit lorsqu’un utilisateur clique sur un élément.

### Exemple

```html
<p id="message">Aucune action</p>
<button id="actionBtn">Faire une action</button>
```

```js
const message = document.getElementById("message");
const actionBtn = document.getElementById("actionBtn");

actionBtn.addEventListener("click", () => {
  message.textContent = "Une action a été déclenchée";
});
```

## Utilité

`click` est très utilisé pour :

- les boutons
- les cartes interactives
- les liens spéciaux
- les actions d’ajout, suppression ou modification

---

# 8. L’événement `input`

`input` se produit chaque fois que le contenu d’un champ change pendant que l’utilisateur écrit.

### HTML

```html
<input id="nameInput" type="text" placeholder="Entrez votre nom" />
<p id="result"></p>
```

### JavaScript

```js
const nameInput = document.getElementById("nameInput");
const result = document.getElementById("result");

nameInput.addEventListener("input", () => {
  result.textContent = "Vous écrivez : " + nameInput.value;
});
```

## Ce qui se passe

À chaque caractère tapé ou supprimé, le texte affiché se met à jour.

## Utilité

`input` est pratique pour :

- afficher un aperçu en direct
- vérifier la longueur d’un texte
- réagir immédiatement à la saisie

---

# 9. L’événement `change`

`change` se produit lorsqu’une valeur est modifiée puis validée.

Il est souvent utilisé avec :

- les listes déroulantes
- les cases à cocher
- les boutons radio
- certains champs de formulaire

### HTML

```html
<select id="city">
  <option value="Montréal">Montréal</option>
  <option value="Paris">Paris</option>
  <option value="Tokyo">Tokyo</option>
</select>

<p id="selectedCity"></p>
```

### JavaScript

```js
const city = document.getElementById("city");
const selectedCity = document.getElementById("selectedCity");

city.addEventListener("change", () => {
  selectedCity.textContent = "Ville choisie : " + city.value;
});
```

## Différence entre `input` et `change`

- `input` réagit immédiatement pendant la saisie
- `change` réagit lorsque la nouvelle valeur est réellement changée ou validée

# 10. Les formulaires : pourquoi sont-ils importants ?

Les formulaires sont partout sur le web :

- connexion
- inscription
- recherche
- envoi de message
- ajout d’une tâche
- commande en ligne

Un formulaire sert à **recueillir des informations auprès de l’utilisateur**.

Avec JavaScript, on peut :

- lire les valeurs saisies
- vérifier si elles sont valides
- afficher des messages d’erreur
- empêcher l’envoi du formulaire si nécessaire

---

# 11. Structure simple d’un formulaire

### HTML

```html
<form id="signupForm">
  <input id="username" type="text" placeholder="Nom d'utilisateur" />
  <button type="submit">Envoyer</button>
</form>
```

Ici, le formulaire contient :

- un champ texte
- un bouton d’envoi

---

# 12. L’événement `submit`

L’événement `submit` se produit lorsqu’un formulaire est envoyé.

### HTML

```html
<form id="signupForm">
  <input id="username" type="text" placeholder="Nom d'utilisateur" />
  <button type="submit">Envoyer</button>
</form>

<p id="message"></p>
```

### JavaScript

```js
const signupForm = document.getElementById("signupForm");
const username = document.getElementById("username");
const message = document.getElementById("message");

signupForm.addEventListener("submit", () => {
  message.textContent = "Formulaire envoyé pour : " + username.value;
});
```

## Problème

Dans la plupart des cas, lorsqu’un formulaire est soumis, la page recharge ou tente d’envoyer les données.
Cela peut empêcher de bien voir le résultat de notre script.

C’est pour cela qu’on utilise souvent `preventDefault()`.

---

# 13. `preventDefault()` : empêcher le comportement par défaut

### Version corrigée

```js
const signupForm = document.getElementById("signupForm");
const username = document.getElementById("username");
const message = document.getElementById("message");

signupForm.addEventListener("submit", (event) => {
  event.preventDefault();
  message.textContent = "Formulaire envoyé pour : " + username.value;
});
```

## Pourquoi ?

Par défaut, un formulaire veut être envoyé.
`preventDefault()` empêche ce comportement automatique.

Cela permet à JavaScript de gérer lui-même ce qui doit se passer.

## Idée importante

Quand on travaille avec des formulaires, `preventDefault()` est très souvent nécessaire.

---

# 14. L’objet `event`

Dans beaucoup d’événements, on reçoit un paramètre souvent appelé `event`.

Exemple :

```js
button.addEventListener("click", (event) => {
  console.log(event);
});
```

Cet objet contient des informations sur l’événement qui vient de se produire.

Dans un premier cours, l’usage le plus important est :

```js
event.preventDefault();
```

Pour des débutants, cela suffit largement au départ.

---

# 15. Lire les valeurs d’un formulaire

Pour lire ce que l’utilisateur écrit dans un champ, on utilise souvent `.value`.

### HTML

```html
<form id="contactForm">
  <input id="firstName" type="text" placeholder="Prénom" />
  <input id="email" type="email" placeholder="Email" />
  <button type="submit">Envoyer</button>
</form>

<p id="output"></p>
```

### JavaScript

```js
const contactForm = document.getElementById("contactForm");
const firstName = document.getElementById("firstName");
const email = document.getElementById("email");
const output = document.getElementById("output");

contactForm.addEventListener("submit", (event) => {
  event.preventDefault();

  output.textContent =
    "Prénom : " + firstName.value + " | Email : " + email.value;
});
```

# 16. Première validation simple

Valider un formulaire, c’est vérifier si les données semblent acceptables avant d’aller plus loin.

### Exemple : vérifier qu’un champ n’est pas vide

```html
<form id="loginForm">
  <input id="userName" type="text" placeholder="Nom" />
  <button type="submit">Connexion</button>
</form>

<p id="feedback"></p>
```

```js
const loginForm = document.getElementById("loginForm");
const userName = document.getElementById("userName");
const feedback = document.getElementById("feedback");

loginForm.addEventListener("submit", (event) => {
  event.preventDefault();

  if (userName.value === "") {
    feedback.textContent = "Veuillez entrer votre nom.";
  } else {
    feedback.textContent = "Bienvenue " + userName.value;
  }
});
```

## Ce que montre cet exemple

- on écoute l’envoi du formulaire
- on empêche le rechargement
- on vérifie la valeur saisie
- on affiche un message

C’est la base de presque toutes les validations de formulaire.

---

# 17. Exemple avec plusieurs conditions simples

```js
const registerForm = document.getElementById("registerForm");
const nameInput = document.getElementById("nameInput");
const emailInput = document.getElementById("emailInput");
const result = document.getElementById("result");

registerForm.addEventListener("submit", (event) => {
  event.preventDefault();

  if (nameInput.value === "" || emailInput.value === "") {
    result.textContent = "Tous les champs doivent être remplis.";
  } else {
    result.textContent = "Le formulaire est prêt à être envoyé.";
  }
});
```

Ici, on vérifie simplement que les deux champs ne sont pas vides.

---

# 18. Afficher un message d’erreur ou de succès

Dans un formulaire, JavaScript sert souvent à informer l’utilisateur.

### Exemple

```js
form.addEventListener("submit", (event) => {
  event.preventDefault();

  if (input.value === "") {
    message.textContent = "Erreur : ce champ est obligatoire.";
  } else {
    message.textContent = "Succès : le formulaire est valide.";
  }
});
```

C’est une logique simple, mais très utile.

---

# 19. Vider un champ après une action

Après avoir utilisé un champ, on peut vouloir le vider.

```js
input.value = "";
```

### Exemple

```js
addTaskForm.addEventListener("submit", (event) => {
  event.preventDefault();

  console.log(taskInput.value);
  taskInput.value = "";
});
```

Cela est très utile pour :

- une liste de tâches
- un champ de recherche
- un mini formulaire dynamique

# 20. Petit exemple complet : mini formulaire

### HTML

```html
<form id="nameForm">
  <input id="nameInput" type="text" placeholder="Votre prénom" />
  <button type="submit">Valider</button>
</form>

<p id="response"></p>
```

### JavaScript

```js
const nameForm = document.getElementById("nameForm");
const nameInput = document.getElementById("nameInput");

nameForm.addEventListener("submit", (event) => {
  event.preventDefault();

  if (nameInput.value === "") {
    response.textContent = "Veuillez entrer un prénom.";
  } else {
    response.textContent = "Bonjour " + nameInput.value;
    nameInput.value = "";
  }

```

## Ce que cet exemple regroupe

- sélection d’éléments
- événement `submit`
- `preventDefault()`
- lecture de `value`
- condition simple
- affichage d’un message
- remise à zéro du champ

C’est un très bon modèle pour débuter.

# 21. Différence entre cliquer sur un bouton et envoyer un formulaire

Il est important que les étudiants comprennent ceci :

## Bouton simple

```html
<button id="myBtn">Cliquer</button>
```

Ici, on écoute souvent un `click`.

## Bouton dans un formulaire

```html
<form>
  <button type="submit">Envoyer</button>
</form>
```

Ici, l’action principale concerne souvent le `submit` du formulaire.

## Idée à retenir

- `click` concerne le bouton
- `submit` concerne le formulaire

Pour la logique d’un formulaire, on préfère souvent écouter `submit`.

# 22. Erreurs fréquentes chez les débutants

## a) Oublier `preventDefault()`

Conséquence : la page recharge et le résultat ne reste pas affiché.

## b) Écouter `click` sur le bouton au lieu de `submit` sur le formulaire

Ce n’est pas toujours faux, mais cela contourne parfois la logique naturelle du formulaire.

## c) Oublier de lire la valeur avec `.value`

Écrire :

```js
console.log(input);
```

n’affiche pas ce que l’utilisateur a tapé, mais l’élément HTML lui-même.

Il faut écrire :

```js
console.log(input.value);
```

## d) Ne pas gérer les champs vides

Un formulaire sans validation simple peut donner des résultats incomplets ou incohérents.

## e) Confondre l’élément et son contenu

- `input` est l’élément
- `input.value` est la valeur saisie

# 23. Logique générale à retenir

Quand on travaille avec les événements et les formulaires, la démarche est souvent la suivante :

## 1. Sélectionner les éléments

```js
const form = document.getElementById("form");
const input = document.getElementById("input");
const message = document.getElementById("message");
```

## 2. Écouter l’événement

```js
form.addEventListener("submit", (event) => {
```

## 3. Empêcher le comportement par défaut si nécessaire

```js
event.preventDefault();
```

## 4. Lire la valeur saisie

```js
const value = input.value;
```

## 5. Vérifier ou utiliser cette valeur

```js
if (value === "") {
  message.textContent = "Champ obligatoire.";
}
```

## 6. Mettre à jour la page

```js
message.textContent = "Bonjour " + value;
```

Cette structure revient très souvent.

# 24. Exemple intégrateur

### HTML

```html
<!DOCTYPE html>
<html lang="fr">
  <head>
    <meta charset="UTF-8" />
    <title>Événements et formulaires</title>
  </head>
  <body>
    <form id="taskForm">
      <input id="taskInput" type="text" placeholder="Nouvelle tâche" />
      <button type="submit">Ajouter</button>
    </form>

    <p id="feedback"></p>
    <ul id="taskList"></ul>

    <script>
      const taskForm = document.getElementById("taskForm");
      const taskInput = document.getElementById("taskInput");
      const feedback = document.getElementById("feedback");
      const taskList = document.getElementById("taskList");

      taskForm.addEventListener("submit", (event) => {
        event.preventDefault();

        if (taskInput.value === "") {
          feedback.textContent = "Veuillez entrer une tâche.";
        } else {
          const newTask = document.createElement("li");
          newTask.textContent = taskInput.value;
          taskList.appendChild(newTask);

          feedback.textContent = "Tâche ajoutée avec succès.";
          taskInput.value = "";
        }
      });
    </script>
  </body>
</html>
```

## Ce que cet exemple permet de revoir

- le DOM
- la sélection d’éléments
- l’événement `submit`
- `preventDefault()`
- la lecture d’un champ
- une validation simple
- la création d’un élément
- l’ajout dans une liste

- réagir à un clic

# 26. Ce qu’il faut retenir en priorité

## Les notions les plus importantes

- un événement est une action qui se produit sur la page
- `addEventListener()` sert à écouter un événement
- `click`, `input`, `change` et `submit` sont très fréquents
- un formulaire utilise souvent `submit`
- `event.preventDefault()` empêche l’envoi automatique du formulaire
- `input.value` permet de lire ce que l’utilisateur a écrit
- une validation simple vérifie que les données ont du sens
