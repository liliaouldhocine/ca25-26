# Les unités relatives en CSS : `em` et `rem`

## Objectifs du cours

À la fin de ce cours, vous serez capables de :

- expliquer ce qu’est une unité relative en CSS ;
- comprendre le fonctionnement de l’unité `em` ;
- comprendre le fonctionnement de l’unité `rem` ;
- distinguer `em` et `rem` ;
- choisir l’unité la plus adaptée selon le contexte ;
- comprendre pourquoi `rem` est souvent plus facile à maintenir dans un projet Web.

---

# 1. Mise en contexte

En CSS, plusieurs propriétés utilisent des mesures.

Par exemple :

```css
p {
  font-size: 16px;
  margin-bottom: 20px;
}
```

Dans cet exemple, `16px` et `20px` sont des valeurs fixes exprimées en pixels.

Mais en CSS, il existe aussi des unités relatives.

Une unité relative permet de définir une taille par rapport à une autre taille.

Cela permet de créer des interfaces plus flexibles et plus faciles à adapter.

---

# 2. Qu’est-ce qu’une unité relative ?

Une unité relative donne une mesure à un élément en fonction d’une autre valeur de référence.

Autrement dit, la taille ne dépend pas seulement de la valeur écrite dans le CSS.

Elle dépend aussi du contexte.

Par exemple :

```css
font-size: 2em;
```

ne veut pas toujours dire la même chose.

La valeur finale dépendra de la taille de référence utilisée par l’unité `em`.

---

# 3. L’unité `em`

L’unité `em` est relative à la taille de police de l’élément parent.

Par exemple, si l’élément parent a une taille de police de `16px`, alors :

```text
1em = 16px
2em = 32px
0.5em = 8px
```

Donc :

```css
.enfant {
  font-size: 2em;
}
```

Si le parent fait `16px`, l’enfant fera :

```text
2 × 16px = 32px
```

---

# 4. Exemple simple avec `em`

HTML :

```html
<div class="parent">
  <p class="enfant">Bonjour</p>
</div>
```

CSS :

```css
.parent {
  font-size: 16px;
}

.enfant {
  font-size: 2em;
}
```

Dans cet exemple :

- `.parent` a une taille de texte de `16px` ;
- `.enfant` a une taille de texte de `2em` ;
- `2em` correspond à deux fois la taille du parent.

Résultat :

```text
.enfant = 32px
```

---

# 5. Les tailles par défaut dans le navigateur

Par défaut, la taille de police de l’élément `html` est généralement de `16px`.

Cela signifie que, si aucune autre taille n’est définie, beaucoup de calculs partent de cette base.

Exemple :

```css
html {
  font-size: 16px;
}
```

Dans les navigateurs, plusieurs éléments HTML ont aussi des tailles par défaut.

Par exemple :

- un paragraphe `<p>` est généralement à `1em` ;
- un titre `<h1>` est généralement plus grand ;
- un titre `<h2>` est généralement plus petit qu’un `<h1>`, mais plus grand qu’un paragraphe.

Ces tailles peuvent varier légèrement selon les styles par défaut du navigateur.

---

# 6. Pourquoi `em` peut devenir difficile à suivre ?

L’unité `em` dépend du parent.

Cela peut sembler simple au départ, mais cela devient plus complexe lorsqu’il y a plusieurs niveaux d’imbrication.

Exemple :

```html
<div class="niveau-1">
  <div class="niveau-2">
    <p class="niveau-3">Texte</p>
  </div>
</div>
```

CSS :

```css
.niveau-1 {
  font-size: 20px;
}

.niveau-2 {
  font-size: 2em;
}

.niveau-3 {
  font-size: 2em;
}
```

Analyse :

```text
.niveau-1 = 20px
.niveau-2 = 2 × 20px = 40px
.niveau-3 = 2 × 40px = 80px
```

Le paragraphe final aura donc une taille de `80px`.

Cela peut devenir surprenant si on ne fait pas attention.

---

# 7. Problème de maintenabilité avec `em`

Le principal problème de `em` est qu’il dépend du contexte.

Si on modifie la taille d’un parent, tous les enfants qui utilisent `em` peuvent être affectés.

Exemple :

```css
.card {
  font-size: 18px;
}

.card-title {
  font-size: 2em;
}
```

Ici, `.card-title` aura une taille de :

```text
2 × 18px = 36px
```

Si on change ensuite la taille de `.card` :

```css
.card {
  font-size: 20px;
}
```

alors `.card-title` devient :

```text
2 × 20px = 40px
```

La taille du titre change indirectement.

Dans un petit projet, ce n’est pas forcément un problème.

Mais dans une application plus grande, cela peut rendre le code plus difficile à maintenir.

---

# 8. L’unité `rem`

L’unité `rem` signifie `root em`.

Elle est relative à la taille de police de l’élément racine, c’est-à-dire l’élément `html`.

Par défaut, cette taille est généralement de `16px`.

Donc, si `html` vaut `16px` :

```text
1rem = 16px
2rem = 32px
0.5rem = 8px
1.5rem = 24px
```

Contrairement à `em`, `rem` ne dépend pas du parent direct.

Il dépend toujours de la même référence : l’élément `html`.

---

# 9. Exemple simple avec `rem`

HTML :

```html
<div class="parent">
  <p class="enfant">Bonjour</p>
</div>
```

CSS :

```css
html {
  font-size: 16px;
}

.parent {
  font-size: 20px;
}

.enfant {
  font-size: 2rem;
}
```

Dans cet exemple :

- `.parent` a une taille de `20px` ;
- `.enfant` utilise `2rem` ;
- `rem` dépend de `html`, pas de `.parent`.

Donc :

```text
.enfant = 2 × 16px = 32px
```

Même si le parent fait `20px`, l’enfant garde une taille basée sur `html`.

---

# 10. Comparaison entre `em` et `rem`

Prenons le même HTML :

```html
<div class="parent">
  <p class="texte-em">Texte avec em</p>
  <p class="texte-rem">Texte avec rem</p>
</div>
```

CSS :

```css
html {
  font-size: 16px;
}

.parent {
  font-size: 20px;
}

.texte-em {
  font-size: 2em;
}

.texte-rem {
  font-size: 2rem;
}
```

Analyse :

```text
.texte-em = 2 × 20px = 40px
.texte-rem = 2 × 16px = 32px
```

Même si les deux valeurs utilisent le chiffre `2`, elles ne donnent pas le même résultat.

Pourquoi ?

Parce qu’elles n’utilisent pas la même référence.

---

# 11. Tableau comparatif

| Unité | Référence                  | Exemple | Résultat si la référence vaut 16px |
| ----- | -------------------------- | ------- | ---------------------------------- |
| `em`  | taille de police du parent | `2em`   | `32px`                             |
| `rem` | taille de police de `html` | `2rem`  | `32px`                             |

La différence importante est la référence utilisée.

Avec `em`, la référence peut changer selon le parent.

Avec `rem`, la référence reste stable dans toute la page.

---

# 12. Pourquoi `rem` est souvent plus facile à utiliser ?

L’unité `rem` est souvent recommandée pour les interfaces modernes parce qu’elle est plus prévisible.

Quand on écrit :

```css
.title {
  font-size: 2rem;
}
```

on sait que la taille dépend de l’élément `html`.

Elle ne dépend pas d’un parent qui pourrait changer selon l’endroit où le titre est placé.

Cela permet de créer un système de tailles plus cohérent.

Exemple :

```css
html {
  font-size: 16px;
}

h1 {
  font-size: 2rem;
}

h2 {
  font-size: 1.5rem;
}

p {
  font-size: 1rem;
}

.card {
  padding: 1.5rem;
}
```

Dans cet exemple :

- les titres ;
- les paragraphes ;
- les espacements ;

sont tous basés sur la même référence.

---

# 13. Exemple pratique : carte avec `rem`

HTML :

```html
<article class="card">
  <h2>Article important</h2>
  <p>Voici un court paragraphe dans une carte.</p>
  <button>Lire plus</button>
</article>
```

CSS :

```css
html {
  font-size: 16px;
}

.card {
  padding: 1.5rem;
  max-width: 24rem;
  border: 1px solid black;
}

.card h2 {
  font-size: 1.5rem;
  margin-bottom: 1rem;
}

.card p {
  font-size: 1rem;
  margin-bottom: 1rem;
}

.card button {
  font-size: 1rem;
  padding: 0.5rem 1rem;
}
```

Analyse :

```text
1rem = 16px
1.5rem = 24px
24rem = 384px
0.5rem = 8px
```

La carte utilise des tailles cohérentes parce que tout est calculé à partir de la même base.

---

# 14. Quand utiliser `em` ?

Même si `rem` est souvent plus simple, `em` reste utile dans certains cas.

On peut utiliser `em` lorsqu’on veut qu’un élément dépende volontairement de la taille de son parent.

Par exemple, pour un bouton :

```css
.button {
  font-size: 1rem;
  padding: 0.5em 1em;
}
```

Ici, le `padding` dépend de la taille du texte du bouton.

Si le texte du bouton devient plus grand, l’espace intérieur du bouton grandit aussi.

Cela peut être pratique pour garder des proportions naturelles.

---

# 15. Quand utiliser `rem` ?

On utilise souvent `rem` pour :

- les tailles de police principales ;
- les marges ;
- les espacements ;
- les largeurs maximales ;
- les paddings généraux ;
- les composants d’interface.

Exemple :

```css
.page {
  padding: 2rem;
}

.title {
  font-size: 2rem;
}

.card {
  padding: 1.5rem;
  max-width: 30rem;
}
```

Ce code est plus facile à comprendre, parce que toutes les tailles dépendent de la même référence.

---

# 16. Bonne pratique générale

Dans la plupart des projets Web, on peut suivre cette règle simple :

```text
Utiliser rem pour les tailles globales et em pour les proportions locales.
```

Par exemple :

```css
.card {
  padding: 1.5rem;
}

.card-title {
  font-size: 1.5rem;
}

.button {
  font-size: 1rem;
  padding: 0.5em 1em;
}
```

Ici :

- la carte utilise `rem`, car son espacement doit être cohérent avec le reste du site ;
- le titre utilise `rem`, car sa taille doit être prévisible ;
- le bouton utilise `em` pour son `padding`, afin que l’espace intérieur suive la taille du texte.

---

# 17. Points d’attention

## 17.1 Ne pas supposer que `em` vaut toujours `16px`

Un `em` ne vaut pas toujours `16px`.

Il dépend de la taille de police du parent.

Exemple :

```css
.parent {
  font-size: 20px;
}

.enfant {
  font-size: 1em;
}
```

Ici :

```text
1em = 20px
```

---

## 17.2 Ne pas oublier que `rem` dépend de `html`

Un `rem` dépend de la taille de police de l’élément `html`.

Exemple :

```css
html {
  font-size: 18px;
}

.title {
  font-size: 2rem;
}
```

Ici :

```text
2rem = 36px
```

---

## 17.3 Éviter les imbrications difficiles avec `em`

Avec plusieurs niveaux d’imbrication, `em` peut produire des résultats difficiles à prévoir.

Exemple :

```css
.parent {
  font-size: 1.2em;
}

.enfant {
  font-size: 1.2em;
}

.petit-enfant {
  font-size: 1.2em;
}
```

Chaque niveau dépend du précédent.

Le calcul peut devenir difficile à suivre.

---

## 17.4 Ne pas mélanger trop d’unités sans logique

Il est normal d’utiliser plusieurs unités CSS dans un projet.

Mais il faut avoir une logique claire.

Par exemple :

```text
rem pour les tailles générales
em pour les proportions locales
px pour certains petits détails fixes
% ou vw/vh pour certaines dimensions liées à l’écran
```

---

# 18. Mini-bilan

Les unités relatives permettent de définir une taille par rapport à une autre taille.

L’unité `em` dépend de la taille de police du parent.

```css
.enfant {
  font-size: 2em;
}
```

Si le parent fait `16px`, alors l’enfant fait `32px`.

L’unité `rem` dépend de la taille de police de l’élément racine `html`.

```css
.enfant {
  font-size: 2rem;
}
```

Si `html` fait `16px`, alors l’enfant fait `32px`.

La grande différence est donc :

```text
em  → dépend du parent
rem → dépend de html
```

Pour cette raison, `rem` est souvent plus facile à utiliser dans les projets Web, surtout lorsque le code devient plus grand.

# Cheatsheet — `em` et `rem`

## `em`

```css
.element {
  font-size: 2em;
}
```

Référence :

```text
la taille de police du parent
```

Exemple :

```text
Parent = 16px
2em = 32px
```

---

## `rem`

```css
.element {
  font-size: 2rem;
}
```

Référence :

```text
la taille de police de html
```

Exemple :

```text
html = 16px
2rem = 32px
```

---

## Différence principale

| Unité | Dépend de      | Facilité à maintenir                       |
| ----- | -------------- | ------------------------------------------ |
| `em`  | parent         | plus difficile avec beaucoup d’imbrication |
| `rem` | élément `html` | plus prévisible                            |

---

## Recommandation

```css
/* Tailles globales */
.title {
  font-size: 2rem;
}

/* Espacements globaux */
.card {
  padding: 1.5rem;
}

/* Proportions locales */
.button {
  padding: 0.5em 1em;
}
```

À retenir :

```text
rem pour la cohérence globale.
em pour les proportions liées à un composant.
```
