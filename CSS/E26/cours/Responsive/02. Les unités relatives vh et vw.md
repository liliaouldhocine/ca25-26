# Les unités relatives `vh` et `vw`

## Objectifs du cours

À la fin de ce cours, vous serez capables de :

- expliquer ce qu’est le viewport ;
- utiliser l’unité `vw` pour définir une largeur relative à l’écran ;
- utiliser l’unité `vh` pour définir une hauteur relative à l’écran ;
- comprendre pourquoi `vh` est utile pour créer des sections pleine hauteur ;
- distinguer `height: 100%` et `height: 100vh` ;
- utiliser `min-height: 100vh` dans une mise en page Web.

---

# 1. Mise en contexte

Dans les cours précédents, nous avons vu les unités relatives `em` et `rem`.

Ces unités dépendent principalement de la taille de police.

Il existe aussi des unités relatives qui dépendent directement de la taille de la fenêtre du navigateur.

Ces unités sont très utiles pour créer des interfaces qui s’adaptent à l’écran de l’utilisateur.

Les deux unités principales sont :

```css
vw
vh
```

---

# 2. Le viewport

Le viewport correspond à la zone visible de la page dans le navigateur.

Autrement dit, c’est l’espace disponible à l’écran pour afficher le site.

Sur un ordinateur, le viewport est généralement large.

Sur un téléphone, le viewport est plus étroit.

Sur une tablette, il peut changer selon l’orientation de l’appareil.

```text
Viewport = zone visible de la page dans le navigateur
```

Les unités `vw` et `vh` sont calculées à partir de cette zone visible.

---

# 3. L’unité `vw`

L’unité `vw` signifie **viewport width**.

Elle représente un pourcentage de la largeur du viewport.

```text
1vw = 1% de la largeur du viewport
```

Donc :

```text
100vw = 100% de la largeur du viewport
50vw = 50% de la largeur du viewport
25vw = 25% de la largeur du viewport
```

---

# 4. Exemple avec `vw`

```css
.banner {
  width: 100vw;
}
```

Dans cet exemple, la bannière prend toute la largeur visible de l’écran.

Autre exemple :

```css
.card {
  width: 50vw;
}
```

Ici, la carte prend la moitié de la largeur du viewport.

Si le viewport mesure `1200px` de large :

```text
50vw = 600px
```

Si le viewport mesure `800px` de large :

```text
50vw = 400px
```

La taille s’adapte donc à la largeur de l’écran.

---

# 5. Attention avec `100vw`

Il faut être prudent avec :

```css
width: 100vw;
```

Dans certains navigateurs, `100vw` peut inclure la largeur de la barre de défilement verticale.

Cela peut créer un léger débordement horizontal.

Dans plusieurs cas, pour prendre toute la largeur disponible dans le parent, on préfère utiliser :

```css
width: 100%;
```

On utilise donc `100vw` lorsqu’on veut vraiment baser la largeur sur le viewport.

---

# 6. L’unité `vh`

L’unité `vh` signifie **viewport height**.

Elle représente un pourcentage de la hauteur du viewport.

```text
1vh = 1% de la hauteur du viewport
```

Donc :

```text
100vh = 100% de la hauteur du viewport
50vh = 50% de la hauteur du viewport
25vh = 25% de la hauteur du viewport
```

---

# 7. Exemple avec `vh`

```css
.hero {
  height: 100vh;
}
```

Dans cet exemple, la section `.hero` prend toute la hauteur visible de l’écran.

Ce type de section est souvent utilisé pour une page d’accueil.

Exemple :

```html
<section class="hero">
  <h1>Bienvenue sur mon site</h1>
  <p>Une section pleine hauteur.</p>
</section>
```

```css
.hero {
  height: 100vh;
}
```

La section occupera toute la hauteur de la fenêtre du navigateur.

---

# 8. Pourquoi `vh` est utile ?

Par défaut, les éléments de type bloc prennent la hauteur de leur contenu.

Par exemple :

```html
<section>
  <h1>Bonjour</h1>
  <p>Petit texte.</p>
</section>
```

Si la section contient peu de texte, elle sera peu haute.

Elle ne prendra pas automatiquement toute la hauteur de l’écran.

Les éléments comme `html`, `body`, `section`, `div`, `main` sont généralement affichés en bloc.

```css
html,
body {
  display: block;
}
```

Un élément bloc prend naturellement toute la largeur disponible, mais sa hauteur dépend de son contenu.

C’est pour cela que `vh` est très utile lorsqu’on veut donner une hauteur liée à l’écran.

---

# 9. Pourquoi `height: 100%` ne suffit pas toujours ?

On pourrait penser qu’il suffit d’écrire :

```css
section {
  height: 100%;
}
```

Mais cela ne donne pas toujours le résultat attendu.

La raison est que `height: 100%` dépend de la hauteur du parent.

Si le parent n’a pas de hauteur définie, le navigateur ne sait pas clairement à quoi correspond ce `100%`.

Par exemple :

```css
section {
  height: 100%;
}
```

Cette règle signifie :

```text
Prends 100% de la hauteur du parent.
```

Mais si le parent n’a pas une hauteur explicite, le calcul peut ne pas produire une section pleine hauteur.

---

# 10. Différence entre `100%` et `100vh`

La différence importante est la référence utilisée.

| Valeur          | Référence           |
| --------------- | ------------------- |
| `height: 100%`  | hauteur du parent   |
| `height: 100vh` | hauteur du viewport |

Avec :

```css
height: 100%;
```

l’élément dépend de son parent.

Avec :

```css
height: 100vh;
```

l’élément dépend directement de la hauteur visible de la fenêtre du navigateur.

C’est pour cela que `100vh` est souvent plus simple lorsqu’on veut créer une section pleine hauteur.

---

# 11. Exemple avec `height: 100vh`

HTML :

```html
<main class="page">
  <section class="hero">
    <h1>Bienvenue</h1>
    <p>Cette section prend toute la hauteur visible.</p>
  </section>
</main>
```

CSS :

```css
body {
  margin: 0;
}

.hero {
  height: 100vh;
  border: 1px solid black;
}
```

Dans cet exemple :

- `body { margin: 0; }` retire la marge par défaut du navigateur ;
- `.hero { height: 100vh; }` donne à la section toute la hauteur du viewport.

---

# 12. Utiliser `min-height: 100vh`

Dans beaucoup de cas, il est préférable d’utiliser :

```css
min-height: 100vh;
```

plutôt que :

```css
height: 100vh;
```

Pourquoi ?

Parce que `height: 100vh` force exactement la hauteur à `100vh`.

Si le contenu est plus grand que la hauteur disponible, il peut dépasser ou créer une mise en page moins flexible.

Avec :

```css
min-height: 100vh;
```

on dit plutôt :

```text
Prends au moins toute la hauteur visible, mais agrandis-toi si le contenu en a besoin.
```

C’est souvent plus sécuritaire pour les vraies pages Web.

---

# 13. Exemple recommandé avec `min-height`

```html
<section class="hero">
  <div class="hero-content">
    <h1>Bienvenue</h1>
    <p>Une section pleine hauteur, mais flexible.</p>
  </div>
</section>
```

```css
body {
  margin: 0;
}

.hero {
  min-height: 100vh;
  display: flex;
  justify-content: center;
  align-items: center;
}

.hero-content {
  max-width: 40rem;
  padding: 2rem;
}
```

Dans cet exemple :

- `.hero` prend au moins toute la hauteur visible ;
- `display: flex` active Flexbox ;
- `justify-content: center` centre le contenu sur l’axe principal ;
- `align-items: center` centre le contenu sur l’axe secondaire ;
- `min-height: 100vh` permet à la section de grandir si le contenu devient plus long.

---

# 14. Exemple combinant `vw` et `vh`

On peut utiliser `vw` et `vh` dans la même interface.

```css
.hero {
  min-height: 100vh;
  width: 100%;
}

.hero h1 {
  font-size: 5vw;
}
```

Ici :

- la section prend au moins toute la hauteur visible ;
- le titre a une taille basée sur la largeur du viewport.

Si la fenêtre est plus large, le titre devient plus grand.

Si la fenêtre est plus étroite, le titre devient plus petit.

---

# 15. Attention avec les textes en `vw`

Il faut être prudent lorsqu’on utilise `vw` pour la taille du texte.

Exemple :

```css
h1 {
  font-size: 8vw;
}
```

Sur un très grand écran, le texte peut devenir énorme.

Sur un très petit écran, il peut devenir trop petit.

Pour un meilleur contrôle, on peut utiliser `clamp`.

Exemple :

```css
h1 {
  font-size: clamp(2rem, 5vw, 4rem);
}
```

Cela signifie :

```text
Minimum : 2rem
Valeur flexible : 5vw
Maximum : 4rem
```

Le texte reste donc flexible, mais avec des limites.

---

# 16. Exemple pratique : page d’accueil pleine hauteur

HTML :

```html
<header class="hero">
  <nav class="navbar">
    <strong>Mon site</strong>
    <a href="#">Contact</a>
  </nav>

  <section class="intro">
    <h1>Créer des interfaces responsives</h1>
    <p>Les unités vh et vw permettent d’adapter les dimensions au viewport.</p>
  </section>
</header>
```

CSS :

```css
body {
  margin: 0;
  font-family: Arial, sans-serif;
}

.hero {
  min-height: 100vh;
  padding: 2rem;
  box-sizing: border-box;
  display: flex;
  flex-direction: column;
}

.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

.intro {
  flex: 1;
  display: flex;
  flex-direction: column;
  justify-content: center;
}

.intro h1 {
  font-size: clamp(2rem, 5vw, 4rem);
  margin: 0;
}

.intro p {
  font-size: 1.2rem;
  max-width: 40rem;
}
```

Analyse :

- `.hero` prend au moins toute la hauteur visible ;
- `padding: 2rem` ajoute un espace intérieur ;
- `box-sizing: border-box` permet d’inclure le padding dans les dimensions ;
- `.intro` utilise `flex: 1` pour prendre l’espace disponible sous la navigation ;
- le titre utilise `clamp` pour rester lisible sur plusieurs tailles d’écran.

# 17. Points d’attention

## 17.1 `vh` dépend de la hauteur visible

L’unité `vh` dépend de la hauteur du viewport.

Si la hauteur de la fenêtre change, la valeur change aussi.

```css
.section {
  height: 50vh;
}
```

Cette section prendra la moitié de la hauteur visible.

---

## 17.2 `vw` dépend de la largeur visible

L’unité `vw` dépend de la largeur du viewport.

```css
.box {
  width: 50vw;
}
```

Cette boîte prendra la moitié de la largeur visible.

---

## 17.3 `height: 100%` dépend du parent

```css
.section {
  height: 100%;
}
```

Cela fonctionne seulement si la hauteur du parent est clairement définie.

Sinon, le résultat peut être différent de ce qu’on attend.

---

## 17.4 `min-height: 100vh` est souvent préférable

Pour créer une section pleine hauteur, on utilise souvent :

```css
.section {
  min-height: 100vh;
}
```

Cette solution est plus flexible que :

```css
.section {
  height: 100vh;
}
```

car elle permet au contenu de prendre plus de place si nécessaire.

---

## 17.5 Penser aux appareils mobiles

Sur mobile, la hauteur visible peut être influencée par la barre d’adresse du navigateur.

Cela peut parfois créer des variations avec `100vh`.

Dans plusieurs cas simples, `min-height: 100vh` reste une bonne solution.

Pour des interfaces plus avancées, il existe aussi des unités modernes comme :

```css
svh
lvh
dvh
```

Mais pour commencer, `vh` et `vw` sont les unités principales à comprendre.

---

# 18. Mini-bilan

Les unités `vw` et `vh` sont des unités relatives au viewport.

```text
1vw = 1% de la largeur du viewport
1vh = 1% de la hauteur du viewport
```

`vw` est utile pour créer des dimensions basées sur la largeur de l’écran.

```css
.box {
  width: 50vw;
}
```

`vh` est utile pour créer des sections qui dépendent de la hauteur visible.

```css
.hero {
  min-height: 100vh;
}
```

Il faut retenir que :

```text
height: 100% dépend du parent.
height: 100vh dépend du viewport.
```

Dans les pages Web modernes, on utilise souvent :

```css
.section {
  min-height: 100vh;
}
```

pour créer une section pleine hauteur, mais suffisamment flexible pour s’adapter au contenu.
