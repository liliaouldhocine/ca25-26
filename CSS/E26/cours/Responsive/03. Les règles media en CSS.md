# Les règles `@media` en CSS

## Objectifs du cours

À la fin de ce cours, vous serez capables de :

- comprendre ce qu’est une règle `@` en CSS ;
- expliquer le rôle de la règle `@media` ;
- utiliser une requête média pour appliquer du CSS selon certaines conditions ;
- cibler un type de média comme l’écran ou l’impression ;
- adapter une page selon la largeur ou la hauteur du viewport ;
- utiliser l’orientation de l’appareil dans une règle CSS ;
- comprendre le rôle des media queries dans le responsive design.

---

# 1. Mise en contexte

Dans une page Web, le même contenu peut être consulté sur plusieurs types d’appareils :

- ordinateur de bureau ;
- ordinateur portable ;
- tablette ;
- téléphone ;
- imprimante ;
- lecteur d’écran.

Un site ne devrait donc pas toujours avoir exactement la même présentation partout.

Par exemple :

- une page peut avoir trois colonnes sur un grand écran ;
- une seule colonne sur un téléphone ;
- une taille de police différente à l’impression ;
- un affichage adapté en mode portrait ou paysage.

Pour gérer ces situations, CSS permet d’utiliser des règles particulières appelées règles `@`.

---

# 2. Les règles `@` en CSS

En CSS, il existe des règles spéciales qui commencent par le symbole `@`.

On les appelle des règles `@`, ou **at-rules**.

Exemples :

```css
@charset "UTF-8";
```

```css
@import url("style.css");
```

```css
@keyframes apparition {
  from {
    opacity: 0;
  }

  to {
    opacity: 1;
  }
}
```

Ces règles permettent d’ajouter des comportements particuliers à une feuille de styles.

La règle qui nous intéresse ici est :

```css
@media;
```

---

# 3. Qu’est-ce que `@media` ?

La règle `@media` permet d’appliquer un ensemble de règles CSS seulement si une condition est respectée.

Cette condition s’appelle une **requête média**, ou **media query**.

Exemple :

```css
@media (max-width: 600px) {
  body {
    font-size: 14px;
  }
}
```

Ce code signifie :

```text
Si la largeur du viewport est de 600px ou moins,
alors la taille du texte du body devient 14px.
```

Si la condition n’est pas respectée, le CSS à l’intérieur du bloc `@media` ne s’applique pas.

---

# 4. À quoi servent les media queries ?

Les media queries sont très utilisées pour créer des interfaces responsives.

Une interface responsive est une interface qui s’adapte à la taille de l’écran ou au contexte d’utilisation.

Par exemple :

```css
.container {
  display: flex;
  gap: 1rem;
}

@media (max-width: 700px) {
  .container {
    flex-direction: column;
  }
}
```

Dans cet exemple :

- sur un grand écran, les éléments sont placés côte à côte ;
- sur un petit écran, les éléments sont empilés verticalement.

---

# 5. Syntaxe générale d’une règle `@media`

La syntaxe générale est la suivante :

```css
@media condition {
  selecteur {
    propriete: valeur;
  }
}
```

Exemple :

```css
@media (max-width: 800px) {
  .card {
    width: 100%;
  }
}
```

La partie entre parenthèses est la condition :

```css
(max-width: 800px)
```

Les règles CSS à l’intérieur s’appliquent seulement si cette condition est vraie.

---

# 6. Les types de médias

Une requête média peut cibler un type de média.

Les types principaux sont :

| Type     | Utilité                  |
| -------- | ------------------------ |
| `screen` | pour les écrans          |
| `print`  | pour l’impression        |
| `speech` | pour les lecteurs vocaux |

---

# 7. Cibler l’impression avec `print`

On peut appliquer du CSS seulement lorsqu’une page est imprimée.

Exemple :

```css
@media print {
  body {
    font-size: 10pt;
  }
}
```

Ce code signifie :

```text
Quand la page est imprimée,
le texte du body utilise une taille de 10pt.
```

Cela peut être utile pour :

- retirer certains éléments inutiles à l’impression ;
- changer la taille du texte ;
- simplifier la mise en page ;
- économiser de l’encre.

Exemple :

```css
@media print {
  nav,
  footer {
    display: none;
  }

  body {
    font-size: 10pt;
  }
}
```

Dans cet exemple, la navigation et le pied de page ne seront pas imprimés.

---

# 8. Cibler les écrans avec `screen`

On peut aussi cibler seulement les écrans.

```css
@media screen {
  body {
    font-size: 16px;
  }
}
```

Cette règle s’applique lorsque la page est affichée sur un écran.

Dans la majorité des sites Web, on travaille principalement avec le type `screen`.

---

# 9. Cibler les lecteurs vocaux avec `speech`

Le type `speech` concerne les lecteurs vocaux.

```css
@media speech {
  body {
    font-size: 13px;
  }
}
```

Ce type est moins souvent utilisé dans les projets courants, mais il existe dans les spécifications CSS.

---

# 10. Cibler la largeur du viewport

Le cas le plus courant avec `@media` est l’adaptation à la largeur du viewport.

Pour rappel, le viewport est la zone visible de la page dans le navigateur.

On peut utiliser des conditions comme :

```css
(min-width: 800px)
```

ou :

```css
(max-width: 800px)
```

---

# 11. Utiliser `max-width`

La condition `max-width` signifie :

```text
jusqu’à cette largeur maximale
```

Exemple :

```css
@media (max-width: 600px) {
  .menu {
    flex-direction: column;
  }
}
```

Ce code signifie :

```text
Si le viewport mesure 600px de large ou moins,
alors le menu devient vertical.
```

C’est très utile pour adapter une interface aux petits écrans.

---

# 12. Exemple avec `max-width`

HTML :

```html
<nav class="menu">
  <a href="#">Accueil</a>
  <a href="#">Services</a>
  <a href="#">Contact</a>
</nav>
```

CSS :

```css
.menu {
  display: flex;
  gap: 1rem;
}

@media (max-width: 600px) {
  .menu {
    flex-direction: column;
  }
}
```

Résultat :

- sur un grand écran, les liens sont côte à côte ;
- sur un petit écran, les liens sont empilés.

---

# 13. Utiliser `min-width`

La condition `min-width` signifie :

```text
à partir de cette largeur minimale
```

Exemple :

```css
@media (min-width: 900px) {
  .container {
    max-width: 1200px;
  }
}
```

Ce code signifie :

```text
Si le viewport mesure au moins 900px de large,
alors le conteneur peut avoir une largeur maximale de 1200px.
```

---

# 14. Différence entre `min-width` et `max-width`

| Condition   | Signification                   | Utilisation fréquente                  |
| ----------- | ------------------------------- | -------------------------------------- |
| `max-width` | jusqu’à une certaine largeur    | adapter aux petits écrans              |
| `min-width` | à partir d’une certaine largeur | enrichir l’interface sur grands écrans |

Exemple avec `max-width` :

```css
@media (max-width: 600px) {
  .card {
    width: 100%;
  }
}
```

Exemple avec `min-width` :

```css
@media (min-width: 900px) {
  .cards {
    display: flex;
  }
}
```

---

# 15. Approche mobile-first

Une approche fréquente en développement Web est l’approche **mobile-first**.

Cela signifie qu’on écrit d’abord le CSS pour les petits écrans.

Ensuite, on ajoute des règles pour les écrans plus larges avec `min-width`.

Exemple :

```css
.cards {
  display: flex;
  flex-direction: column;
  gap: 1rem;
}

@media (min-width: 800px) {
  .cards {
    flex-direction: row;
  }
}
```

Dans cet exemple :

- par défaut, les cartes sont empilées ;
- à partir de `800px`, les cartes passent côte à côte.

Cette approche est souvent plus simple à maintenir.

---

# 16. Approche desktop-first

Il existe aussi l’approche **desktop-first**.

On écrit d’abord le CSS pour les grands écrans, puis on adapte pour les petits écrans avec `max-width`.

Exemple :

```css
.cards {
  display: flex;
  gap: 1rem;
}

@media (max-width: 800px) {
  .cards {
    flex-direction: column;
  }
}
```

Dans cet exemple :

- par défaut, les cartes sont côte à côte ;
- à `800px` ou moins, elles s’empilent.

Les deux approches existent, mais il est important de rester cohérent dans un même projet.

---

# 17. Cibler la hauteur du viewport

On peut aussi utiliser la hauteur du viewport dans une media query.

Exemple :

```css
@media (max-height: 700px) {
  .hero {
    padding-top: 1rem;
    padding-bottom: 1rem;
  }
}
```

Ce code signifie :

```text
Si la hauteur visible est de 700px ou moins,
alors on réduit l’espace vertical dans la section hero.
```

Cela peut être utile pour les écrans peu hauts.

---

# 18. Largeur ou hauteur exacte

Il est possible de cibler une largeur ou une hauteur exacte.

Exemples :

```css
@media (width: 300px) {
  body {
    background: lightgray;
  }
}
```

```css
@media (height: 360px) {
  body {
    background: lightgray;
  }
}
```

Cependant, en pratique, on utilise rarement une taille exacte.

Pourquoi ?

Parce qu’il existe beaucoup de tailles d’écrans différentes.

On préfère généralement utiliser des intervalles avec `min-width` ou `max-width`.

---

# 19. Intervalles de tailles

On peut cibler une zone entre deux tailles.

Une façon très compatible consiste à combiner `min-width` et `max-width`.

Exemple :

```css
@media (min-width: 400px) and (max-width: 700px) {
  .container {
    padding: 2rem;
  }
}
```

Ce code signifie :

```text
Si le viewport mesure entre 400px et 700px,
alors le conteneur reçoit un padding de 2rem.
```

---

# 20. Syntaxe moderne avec comparateurs

CSS permet aussi une syntaxe plus moderne pour exprimer certains intervalles.

Exemple :

```css
@media (400px <= width <= 700px) {
  .container {
    padding: 2rem;
  }
}
```

On peut aussi écrire :

```css
@media (height > 600px) {
  .section {
    min-height: 100vh;
  }
}
```

Cette syntaxe est plus lisible, mais l’écriture avec `min-width` et `max-width` reste encore très courante dans les projets.

---

# 21. Orientation de l’appareil

Une media query peut aussi cibler l’orientation du viewport.

Il existe deux orientations principales :

| Orientation | Signification                             |
| ----------- | ----------------------------------------- |
| `portrait`  | la hauteur est plus grande que la largeur |
| `landscape` | la largeur est plus grande que la hauteur |

---

# 22. Exemple avec `orientation: portrait`

```css
@media (orientation: portrait) {
  p {
    padding: 15px;
  }
}
```

Ce code signifie :

```text
Si le viewport est en mode portrait,
alors les paragraphes ont un padding de 15px.
```

Le mode portrait est fréquent sur les téléphones tenus verticalement.

---

# 23. Exemple avec `orientation: landscape`

```css
@media (orientation: landscape) {
  p {
    padding: 10px;
  }
}
```

Ce code signifie :

```text
Si le viewport est en mode paysage,
alors les paragraphes ont un padding de 10px.
```

Le mode paysage correspond à une zone d’affichage plus large que haute.

---

# 24. Exemple pratique avec orientation

HTML :

```html
<section class="intro">
  <h1>Bienvenue</h1>
  <p>Cette section s’adapte à l’orientation de l’écran.</p>
</section>
```

CSS :

```css
.intro {
  padding: 2rem;
}

@media (orientation: portrait) {
  .intro {
    text-align: center;
  }
}

@media (orientation: landscape) {
  .intro {
    display: flex;
    align-items: center;
    justify-content: space-between;
  }
}
```

Dans cet exemple :

- en portrait, le contenu est centré ;
- en paysage, la section peut utiliser une disposition plus horizontale.

---

# 25. Combiner plusieurs conditions

On peut combiner plusieurs conditions avec `and`.

Exemple :

```css
@media screen and (min-width: 800px) {
  .container {
    max-width: 1200px;
  }
}
```

Ce code signifie :

```text
Si le média est un écran
et si le viewport mesure au moins 800px de large,
alors la règle s’applique.
```

Autre exemple :

```css
@media screen and (max-width: 600px) and (orientation: portrait) {
  .menu {
    flex-direction: column;
  }
}
```

Ce code signifie :

```text
Si on est sur un écran,
avec une largeur de 600px ou moins,
en orientation portrait,
alors le menu devient vertical.
```

---

# 26. Autres caractéristiques média

Les media queries peuvent cibler d’autres caractéristiques que la taille du viewport.

Par exemple :

| Caractéristique          | Rôle                                   |
| ------------------------ | -------------------------------------- |
| `aspect-ratio`           | ratio entre la largeur et la hauteur   |
| `resolution`             | résolution de l’appareil               |
| `prefers-color-scheme`   | préférence thème clair ou sombre       |
| `scripting`              | disponibilité des scripts              |
| `hover`                  | capacité de survol avec la souris      |
| `pointer`                | précision du pointeur                  |
| `prefers-reduced-motion` | préférence pour réduire les animations |

---

# 27. Exemple avec `prefers-color-scheme`

On peut adapter une interface selon la préférence de thème de l’utilisateur.

```css
@media (prefers-color-scheme: dark) {
  body {
    background: #111;
    color: #f5f5f5;
  }
}
```

Ce code signifie :

```text
Si l’utilisateur préfère le thème sombre,
alors on applique un fond sombre et un texte clair.
```

---

# 28. Exemple avec `prefers-reduced-motion`

Certaines personnes préfèrent réduire les animations dans leur système.

On peut respecter cette préférence avec :

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none;
    transition: none;
  }
}
```

Ce code signifie :

```text
Si l’utilisateur préfère réduire les animations,
alors on désactive les animations et transitions.
```

C’est une bonne pratique d’accessibilité.

---

# 29. Exemple complet : page responsive

HTML :

```html
<header class="hero">
  <nav class="navbar">
    <strong>Mon site</strong>
    <div class="links">
      <a href="#">Accueil</a>
      <a href="#">Services</a>
      <a href="#">Contact</a>
    </div>
  </nav>

  <section class="intro">
    <h1>Créer une page responsive</h1>
    <p>Les media queries permettent d’adapter l’interface à l’écran.</p>
  </section>
</header>
```

CSS :

```css
body {
  margin: 0;
  font-family: Arial, sans-serif;
}

.hero {
  min-height: 100vh;
  padding: 1rem;
  box-sizing: border-box;
}

.navbar {
  display: flex;
  flex-direction: column;
  gap: 1rem;
}

.links {
  display: flex;
  flex-direction: column;
  gap: 0.5rem;
}

.intro {
  margin-top: 3rem;
}

.intro h1 {
  font-size: 2rem;
}

.intro p {
  font-size: 1rem;
  max-width: 40rem;
}

@media (min-width: 700px) {
  .hero {
    padding: 2rem;
  }

  .navbar {
    flex-direction: row;
    justify-content: space-between;
    align-items: center;
  }

  .links {
    flex-direction: row;
    gap: 1rem;
  }

  .intro h1 {
    font-size: 3rem;
  }
}
```

Analyse :

- par défaut, le design est adapté aux petits écrans ;
- la navigation est verticale ;
- à partir de `700px`, la navigation devient horizontale ;
- le titre devient plus grand sur les écrans plus larges ;
- le code suit une logique mobile-first.

# 30. Points d’attention

## 30.1 Ne pas multiplier les breakpoints sans raison

Un breakpoint est une largeur à laquelle on change la mise en page.

Exemple :

```css
@media (min-width: 700px) {
  ...
}
```

Il faut éviter d’ajouter trop de breakpoints.

Il vaut mieux en ajouter seulement lorsque le design en a besoin.

---

## 30.2 Penser au contenu, pas seulement aux appareils

Il ne faut pas choisir les breakpoints uniquement en fonction de modèles d’appareils.

Par exemple :

```text
Téléphone = 375px
Tablette = 768px
Ordinateur = 1200px
```

Ces repères peuvent aider, mais le plus important est d’observer quand le contenu commence à être moins lisible.

Un bon breakpoint est souvent placé là où la mise en page commence à se briser.

---

## 30.3 Garder une logique cohérente

Il est préférable de choisir une approche principale :

```text
mobile-first avec min-width
```

ou :

```text
desktop-first avec max-width
```

Mélanger les deux sans logique peut rendre le CSS difficile à maintenir.

---

## 30.4 Éviter les tailles exactes trop strictes

Les règles comme :

```css
@media (width: 375px) {
  ...
}
```

sont rarement idéales.

Elles s’appliquent seulement à une taille précise.

On préfère généralement :

```css
@media (max-width: 600px) {
  ...
}
```

ou :

```css
@media (min-width: 700px) {
  ...
}
```

---

# 31. Mini-bilan

La règle `@media` permet d’appliquer du CSS seulement si certaines conditions sont respectées.

Elle est essentielle pour créer des interfaces responsives.

Exemple simple :

```css
@media (max-width: 600px) {
  .menu {
    flex-direction: column;
  }
}
```

Les media queries peuvent cibler :

- le type de média ;
- la largeur du viewport ;
- la hauteur du viewport ;
- l’orientation ;
- certaines préférences de l’utilisateur ;
- d’autres caractéristiques de l’appareil ou du navigateur.

Pour créer un design responsive, on utilise souvent une approche mobile-first :

```css
.element {
  /* style de base pour petits écrans */
}

@media (min-width: 800px) {
  .element {
    /* style pour écrans plus larges */
  }
}
```

```css
@media (condition) {

  }
}
```

---

## Largeur maximale

```css
@media (max-width: 600px) {
  .menu {
    flex-direction: column;
  }
}
```

S’applique à `600px` ou moins.

```css
@media (min-width: 800px) {
  .container {
    max-width: 1200px;
  }
}
```

S’applique à partir de `800px`.

---

```css
@media (min-width: 400px) and (max-width: 700px) {
  .box {
    padding: 2rem;
  }
}
```

S’applique entre `400px` et `700px`.

---

```css
@media (orientation: portrait) {
  .section {
```

```css
@media (orientation: landscape) {

  }
```
