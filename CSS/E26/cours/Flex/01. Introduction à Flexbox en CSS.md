# Introduction à Flexbox en CSS

## Objectifs du cours

À la fin de ce cours, vous serez capables de :

- expliquer ce qu’est Flexbox ;
- transformer un élément HTML en conteneur flexible ;
- distinguer le conteneur flexible et les éléments flexibles ;
- comprendre la différence entre l’axe principal et l’axe secondaire ;
- utiliser `flex-direction` pour changer la direction des éléments ;
- utiliser `flex-wrap` pour autoriser ou non le retour à la ligne ;
- utiliser `flex-flow` comme raccourci.

---

# 1. Mise en contexte

Lorsqu’on crée une page Web, on doit souvent placer plusieurs éléments les uns à côté des autres ou les uns sous les autres.

Par exemple, on peut vouloir créer :

- une barre de navigation ;
- une rangée de cartes ;
- une section avec une image et du texte ;
- une liste d’éléments alignés ;
- une interface qui s’adapte à la taille de l’écran.

Avant Flexbox, il était parfois difficile d’aligner proprement les éléments, surtout lorsque leur taille était inconnue ou variable.

Flexbox a été créé pour simplifier ce travail.

---

# 2. Qu’est-ce que Flexbox ?

Flexbox est un modèle de disposition CSS qui permet de gérer plus facilement l’organisation des éléments HTML dans un conteneur.

Flexbox permet notamment de :

- placer des éléments horizontalement ou verticalement ;
- aligner des éléments ;
- gérer l’espace entre les éléments ;
- adapter la taille des éléments selon l’espace disponible ;
- créer plus facilement des interfaces responsives.

Le mot **Flexbox** vient de **Flexible Box Layout**, ce qui signifie qu’on travaille avec des boîtes flexibles.

---

# 3. Conteneur et éléments flexibles

Pour comprendre Flexbox, il faut distinguer deux éléments importants :

## 3.1 Le conteneur flexible

Un conteneur est un élément HTML qui contient d’autres éléments HTML.

Exemple :

```html
<div class="container">
  <div>Élément 1</div>
  <div>Élément 2</div>
  <div>Élément 3</div>
</div>
```

Ici, le `div` avec la classe `.container` est le conteneur.

---

## 3.2 Les éléments flexibles

Les éléments qui se trouvent directement à l’intérieur du conteneur deviennent des **éléments flexibles**, aussi appelés **flex items**.

Dans l’exemple suivant :

```html
<div class="container">
  <div class="item">Élément 1</div>
  <div class="item">Élément 2</div>
  <div class="item">Élément 3</div>
</div>
```

Les trois éléments `.item` sont les enfants directs du conteneur `.container`.

Lorsque le conteneur devient flexible, ces éléments deviennent des éléments flexibles.

---

# 4. Créer un conteneur flexible

Pour créer un conteneur flexible, on utilise la propriété CSS `display` avec la valeur `flex`.

```css
.container {
  display: flex;
}
```

À partir de ce moment, les enfants directs de `.container` deviennent des éléments flexibles.

Exemple complet :

```html
<div class="container">
  <div class="item">Un</div>
  <div class="item">Deux</div>
  <div class="item">Trois</div>
</div>
```

```css
.container {
  display: flex;
}

.item {
  padding: 1rem;
  border: 1px solid black;
}
```

Par défaut, les éléments seront placés sur une ligne, de gauche à droite.

---

# 5. L’axe principal et l’axe secondaire

Flexbox fonctionne avec deux axes :

- l’axe principal ;
- l’axe secondaire.

Ces axes permettent de comprendre dans quelle direction les éléments sont placés.

---

## 5.1 L’axe principal

L’axe principal est l’axe sur lequel les éléments flexibles sont disposés.

Par défaut, l’axe principal est horizontal.

Cela signifie que les éléments sont placés de gauche à droite.

```text
Élément 1  →  Élément 2  →  Élément 3
```

---

## 5.2 L’axe secondaire

L’axe secondaire est l’axe perpendiculaire à l’axe principal.

Si l’axe principal est horizontal, l’axe secondaire est vertical.

```text
Axe principal : horizontal
Axe secondaire : vertical
```

Si l’axe principal est vertical, l’axe secondaire devient horizontal.

```text
Axe principal : vertical
Axe secondaire : horizontal
```

Cette distinction est importante, car plusieurs propriétés Flexbox dépendent de ces axes.

---

# 6. Définir l’axe principal avec `flex-direction`

La propriété `flex-direction` permet de choisir la direction de l’axe principal.

Elle s’utilise sur le conteneur flexible.

```css
.container {
  display: flex;
  flex-direction: row;
}
```

La propriété `flex-direction` peut prendre quatre valeurs principales :

| Valeur           | Effet                                                  |
| ---------------- | ------------------------------------------------------ |
| `row`            | place les éléments horizontalement, de gauche à droite |
| `row-reverse`    | place les éléments horizontalement, de droite à gauche |
| `column`         | place les éléments verticalement, de haut en bas       |
| `column-reverse` | place les éléments verticalement, de bas en haut       |

Par défaut, la valeur est `row`.

---

# 7. `flex-direction: row`

La valeur `row` place les éléments sur une ligne horizontale, de gauche à droite.

```css
.container {
  display: flex;
  flex-direction: row;
}
```

Résultat logique :

```text
Élément 1  Élément 2  Élément 3
```

C’est le comportement par défaut de Flexbox.

Même si on n’écrit pas `flex-direction: row`, le conteneur flexible utilise déjà cette direction.

---

# 8. `flex-direction: row-reverse`

La valeur `row-reverse` place les éléments horizontalement, mais dans l’ordre inverse.

```css
.container {
  display: flex;
  flex-direction: row-reverse;
}
```

Résultat logique :

```text
Élément 3  Élément 2  Élément 1
```

Les éléments sont toujours placés sur une ligne horizontale, mais l’ordre visuel est inversé.

---

# 9. `flex-direction: column`

La valeur `column` place les éléments verticalement, de haut en bas.

```css
.container {
  display: flex;
  flex-direction: column;
}
```

Résultat logique :

```text
Élément 1
Élément 2
Élément 3
```

Cette direction est utile lorsqu’on veut empiler des éléments verticalement tout en utilisant les autres avantages de Flexbox.

---

# 10. `flex-direction: column-reverse`

La valeur `column-reverse` place les éléments verticalement, mais dans l’ordre inverse.

```css
.container {
  display: flex;
  flex-direction: column-reverse;
}
```

Résultat logique :

```text
Élément 3
Élément 2
Élément 1
```

Les éléments sont toujours sur un axe vertical, mais ils sont affichés du bas vers le haut.

---

# 11. Comprendre le lien entre axe principal et axe secondaire

La direction choisie avec `flex-direction` change l’axe principal.

Si on utilise :

```css
flex-direction: row;
```

Alors :

```text
Axe principal : horizontal
Axe secondaire : vertical
```

Si on utilise :

```css
flex-direction: column;
```

Alors :

```text
Axe principal : vertical
Axe secondaire : horizontal
```

Cela aura une importance lorsqu’on utilisera plus tard des propriétés comme :

```css
justify-content
align-items
```

Ces propriétés servent à aligner les éléments, mais elles ne travaillent pas sur le même axe.

---

# 12. Autoriser ou non le retour à la ligne avec `flex-wrap`

Par défaut, Flexbox essaie de placer tous les éléments sur une seule ligne.

Cela signifie que si plusieurs éléments sont dans un conteneur, Flexbox va essayer de les faire tenir sur la même ligne.

La propriété `flex-wrap` permet de définir si les éléments peuvent aller ou non à la ligne.

Elle s’utilise sur le conteneur flexible.

```css
.container {
  display: flex;
  flex-wrap: wrap;
}
```

Les valeurs principales sont :

| Valeur         | Effet                                                                       |
| -------------- | --------------------------------------------------------------------------- |
| `nowrap`       | les éléments restent sur une seule ligne                                    |
| `wrap`         | les éléments peuvent passer sur plusieurs lignes                            |
| `wrap-reverse` | les éléments peuvent passer sur plusieurs lignes, mais dans le sens inverse |

Par défaut, la valeur est `nowrap`.

---

# 13. `flex-wrap: nowrap`

La valeur `nowrap` signifie que les éléments doivent rester sur une seule ligne.

```css
.container {
  display: flex;
  flex-wrap: nowrap;
}
```

C’est le comportement par défaut.

Si les éléments sont trop larges pour entrer dans le conteneur, Flexbox va d’abord essayer de réduire leur taille.

Si les éléments ne peuvent plus être réduits, ils peuvent finir par dépasser du conteneur.

Exemple :

```css
.container {
  display: flex;
  flex-wrap: nowrap;
  width: 400px;
}

.item {
  width: 200px;
}
```

Si on a trois éléments de `200px`, cela donne :

```text
200px + 200px + 200px = 600px
```

Mais le conteneur fait seulement `400px`.

Flexbox va donc essayer de réduire les éléments pour les faire tenir sur une seule ligne.

---

# 14. `flex-wrap: wrap`

La valeur `wrap` permet aux éléments de passer à la ligne lorsqu’il n’y a plus assez d’espace.

```css
.container {
  display: flex;
  flex-wrap: wrap;
}
```

Exemple :

```css
.container {
  display: flex;
  flex-wrap: wrap;
  width: 400px;
}

.item {
  width: 200px;
}
```

Si trois éléments de `200px` sont dans un conteneur de `400px`, les deux premiers peuvent être sur la première ligne, et le troisième peut passer à la ligne suivante.

Résultat logique :

```text
Élément 1  Élément 2
Élément 3
```

Cette propriété est très utile pour créer des interfaces responsives.

---

# 15. `flex-wrap: wrap-reverse`

La valeur `wrap-reverse` permet aussi aux éléments de passer sur plusieurs lignes, mais les lignes sont placées dans le sens inverse.

```css
.container {
  display: flex;
  flex-wrap: wrap-reverse;
}
```

Cette valeur est moins utilisée que `wrap`, mais elle peut être utile dans certains cas précis.

---

# 16. Le raccourci `flex-flow`

La propriété `flex-flow` permet de définir `flex-direction` et `flex-wrap` en une seule ligne.

Elle combine deux propriétés :

```css
flex-direction
flex-wrap
```

La première valeur correspond à `flex-direction`.

La deuxième valeur correspond à `flex-wrap`.

Exemple :

```css
.container {
  display: flex;
  flex-flow: row wrap;
}
```

Ce code signifie :

```css
.container {
  display: flex;
  flex-direction: row;
  flex-wrap: wrap;
}
```

Les deux écritures sont équivalentes.

---

# 17. Exemple complet

HTML :

```html
<section class="cards">
  <article class="card">Carte 1</article>
  <article class="card">Carte 2</article>
  <article class="card">Carte 3</article>
  <article class="card">Carte 4</article>
</section>
```

CSS :

```css
.cards {
  display: flex;
  flex-direction: row;
  flex-wrap: wrap;
  gap: 1rem;
}

.card {
  width: 200px;
  padding: 1rem;
  border: 1px solid black;
}
```

Dans cet exemple :

- `.cards` est le conteneur flexible ;
- `.card` représente les éléments flexibles ;
- `display: flex` active Flexbox ;
- `flex-direction: row` place les cartes horizontalement ;
- `flex-wrap: wrap` permet aux cartes de passer à la ligne ;
- `gap: 1rem` ajoute un espace entre les cartes.

---

# 18. Même exemple avec `flex-flow`

On peut écrire le même CSS plus simplement avec `flex-flow`.

```css
.cards {
  display: flex;
  flex-flow: row wrap;
  gap: 1rem;
}

.card {
  width: 200px;
  padding: 1rem;
  border: 1px solid black;
}
```

Cette version produit le même résultat.

```css
flex-flow: row wrap;
```

signifie :

```css
flex-direction: row;
flex-wrap: wrap;
```

---

# 19. Points d’attention

## 19.1 `display: flex` se met sur le parent

Une erreur fréquente est de mettre `display: flex` sur les mauvais éléments.

Il faut l’appliquer au conteneur, pas nécessairement aux enfants.

Exemple correct :

```css
.container {
  display: flex;
}
```

HTML :

```html
<div class="container">
  <div>Élément 1</div>
  <div>Élément 2</div>
</div>
```

Ici, `.container` contrôle la disposition de ses enfants directs.

---

## 19.2 Flexbox agit sur les enfants directs

Flexbox organise seulement les enfants directs du conteneur.

Exemple :

```html
<div class="container">
  <div class="item">
    <p>Texte</p>
  </div>
  <div class="item">
    <p>Texte</p>
  </div>
</div>
```

Si `.container` est en `display: flex`, les éléments `.item` seront organisés par Flexbox.

Les balises `<p>` à l’intérieur ne sont pas directement contrôlées par ce conteneur flex.

---

## 19.3 `row` et `column` changent le sens de lecture de l’interface

Avec `row`, les éléments sont placés horizontalement.

Avec `column`, ils sont placés verticalement.

Il faut donc choisir la direction selon la structure visuelle souhaitée.

---

## 19.4 `nowrap` peut créer des dépassements

Par défaut, `flex-wrap` vaut `nowrap`.

Cela peut parfois créer un débordement horizontal si les éléments ne peuvent pas être suffisamment réduits.

Pour une rangée de cartes responsive, on utilise souvent :

```css
flex-wrap: wrap;
```

ou :

```css
flex-flow: row wrap;
```

---

# 20. Mini-bilan

Flexbox est un outil CSS qui permet de gérer plus facilement la disposition des éléments dans une page Web.

Pour utiliser Flexbox, on commence par transformer un conteneur en conteneur flexible :

```css
.container {
  display: flex;
}
```

Les enfants directs deviennent alors des éléments flexibles.

La propriété `flex-direction` permet de choisir l’axe principal :

```css
flex-direction: row;
flex-direction: row-reverse;
flex-direction: column;
flex-direction: column-reverse;
```

La propriété `flex-wrap` permet de choisir si les éléments peuvent passer à la ligne :

```css
flex-wrap: nowrap;
flex-wrap: wrap;
flex-wrap: wrap-reverse;
```

La propriété `flex-flow` permet de combiner les deux :

```css
flex-flow: row wrap;
```

Flexbox est particulièrement utile pour créer des interfaces modernes, adaptables et plus faciles à organiser.

# Cheatsheet — Flexbox, première partie

| Propriété                        | Où l’utiliser ?  | Rôle                                             |
| -------------------------------- | ---------------- | ------------------------------------------------ |
| `display: flex`                  | sur le conteneur | active Flexbox                                   |
| `flex-direction: row`            | sur le conteneur | place les éléments de gauche à droite            |
| `flex-direction: row-reverse`    | sur le conteneur | place les éléments de droite à gauche            |
| `flex-direction: column`         | sur le conteneur | place les éléments de haut en bas                |
| `flex-direction: column-reverse` | sur le conteneur | place les éléments de bas en haut                |
| `flex-wrap: nowrap`              | sur le conteneur | force les éléments à rester sur une seule ligne  |
| `flex-wrap: wrap`                | sur le conteneur | permet aux éléments de passer à la ligne         |
| `flex-wrap: wrap-reverse`        | sur le conteneur | permet le retour à la ligne dans le sens inverse |
| `flex-flow: row wrap`            | sur le conteneur | raccourci pour `flex-direction` et `flex-wrap`   |

---

## À retenir

La base de Flexbox repose sur trois idées :

```css
.container {
  display: flex;
}
```

Le conteneur contrôle la disposition de ses enfants directs.

```css
.container {
  flex-direction: row;
}
```

La direction définit l’axe principal.

```css
.container {
  flex-wrap: wrap;
}
```

Le retour à la ligne permet de mieux gérer les petits écrans.
