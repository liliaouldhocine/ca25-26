# Flexbox : exemple complet de mise en page

## Objectifs du cours

À la fin de ce cours, vous serez capables de :

- analyser une mise en page complète construite avec Flexbox ;
- reconnaître les conteneurs flexibles imbriqués ;
- comprendre comment `flex-direction`, `flex-flow`, `flex`, `flex-basis`, `align-items` et `align-content` travaillent ensemble ;
- distinguer les propriétés appliquées aux conteneurs et celles appliquées aux éléments flexibles ;
- construire une structure de page avec un `header`, une `sidebar` et une zone de contenu.

---

# 1. Mise en contexte

Dans les cours précédents, nous avons vu plusieurs propriétés Flexbox séparément.

Nous avons vu des propriétés appliquées au conteneur :

```css
display: flex;
flex-direction: row;
flex-wrap: wrap;
justify-content: center;
align-items: center;
align-content: flex-start;
```

Nous avons aussi vu des propriétés appliquées aux éléments flexibles :

```css
flex-grow: 1;
flex-shrink: 1;
flex-basis: 200px;
flex: 1 1 auto;
align-self: center;
```

Dans ce cours, nous allons analyser un exemple complet de mise en page.

L’objectif est de comprendre comment plusieurs conteneurs flexibles peuvent être imbriqués les uns dans les autres pour créer une interface plus réaliste.

---

# 2. Structure générale de la page

La page est composée de trois grandes zones :

```text
+--------------------------------+
| Header                         |
+----------+---------------------+
| Sidebar  | Content             |
|          |                     |
|          |                     |
+----------+---------------------+
```

Nous avons donc :

- un conteneur principal `.container` ;
- une section `.header` en haut ;
- une section `.body` en dessous ;
- dans `.body`, une `.sidebar` à gauche ;
- dans `.body`, une section `.content` à droite.

La structure HTML contient donc des conteneurs imbriqués.

# 5. Analyse de la structure HTML

Le HTML est organisé comme ceci :

```html
<div class="container">
  <header class="header"></header>

  <main class="body">
    <aside class="sidebar"></aside>
    <section class="content"></section>
  </main>
</div>
```

La balise `.container` contient deux enfants directs :

```text
.header
.body
```

La section `.body` contient à son tour deux enfants directs :

```text
.sidebar
.content
```

La section `.sidebar` contient plusieurs `span`.

La section `.content` contient plusieurs `div`.

Il y a donc plusieurs niveaux de Flexbox dans la même page.

---

# 6. Analyse du conteneur principal `.container`

```css
.container {
  display: flex;
  flex-direction: column;
  height: 100vh;
}
```

Ici, on transforme `.container` en conteneur flexible avec :

```css
display: flex;
```

Ses enfants directs deviennent donc des éléments flexibles.

Les enfants directs sont :

```text
.header
.body
```

Ensuite, on utilise :

```css
flex-direction: column;
```

Cela signifie que les enfants directs seront placés verticalement.

Résultat :

```text
.header
.body
```

Le `header` sera donc au-dessus du `body`.

Finalement :

```css
height: 100vh;
```

signifie que le conteneur prend toute la hauteur visible du navigateur.

Pour rappel :

```text
100vh = 100% de la hauteur du viewport
```

---

# 7. Analyse de `.header`

```css
.header {
  min-height: 50px;
  background: red;
}
```

Le `header` reçoit une hauteur minimale de `50px`.

On utilise :

```css
min-height: 50px;
```

plutôt que seulement :

```css
height: 50px;
```

Cela signifie que le `header` ne pourra pas être plus petit que `50px`, mais il pourra grandir si son contenu demande plus d’espace.

La propriété :

```css
background: red;
```

sert seulement à visualiser la zone du `header`.

---

# 8. Analyse de `.body`

```css
.body {
  background: green;
  flex: 1;
  display: flex;
  flex-flow: row nowrap;
  align-items: stretch;
}
```

La section `.body` est particulière, car elle joue deux rôles.

Elle est d’abord un élément flexible dans `.container`.

Mais elle devient aussi un conteneur flexible pour ses propres enfants.

---

## 8.1 `.body` comme élément flexible

La propriété suivante s’applique à `.body` en tant qu’élément flexible de `.container` :

```css
flex: 1;
```

Cette écriture signifie que `.body` peut occuper l’espace disponible.

Dans cette mise en page, `.container` a une hauteur de `100vh`.

Le `header` prend au minimum `50px`.

La section `.body` prend donc l’espace restant.

On obtient donc :

```text
Hauteur de .body = hauteur disponible restante
```

En pratique :

```text
.body ≈ 100vh - hauteur du header
```

---

## 8.2 Comprendre `flex: 1`

L’écriture :

```css
flex: 1;
```

est un raccourci très utilisé.

Elle signifie essentiellement que l’élément peut grandir pour prendre l’espace disponible.

Dans la plupart des cas, on peut la comprendre comme une façon de dire :

```text
Prends l’espace restant disponible.
```

Dans notre exemple, cela permet à `.body` de remplir tout l’espace sous le `header`.

---

## 8.3 `.body` comme conteneur flexible

Ensuite, `.body` devient elle-même un conteneur flexible :

```css
display: flex;
```

Ses enfants directs deviennent donc des éléments flexibles.

Les enfants directs de `.body` sont :

```text
.sidebar
.content
```

---

## 8.4 Direction de `.body`

```css
flex-flow: row nowrap;
```

La propriété `flex-flow` est un raccourci pour :

```css
flex-direction: row;
flex-wrap: nowrap;
```

Cela signifie que `.sidebar` et `.content` seront placés sur une ligne horizontale.

Résultat :

```text
.sidebar | .content
```

On utilise `row`, car on veut que la sidebar soit à gauche et que le contenu soit à droite.

On utilise `nowrap`, car on ne veut pas que la sidebar et le contenu passent sur deux lignes différentes.

---

## 8.5 Alignement vertical dans `.body`

```css
align-items: stretch;
```

Comme `.body` utilise :

```css
flex-direction: row;
```

son axe principal est horizontal.

Son axe secondaire est donc vertical.

La propriété `align-items` agit sur l’axe secondaire.

Ici :

```css
align-items: stretch;
```

signifie que les éléments `.sidebar` et `.content` vont s’étirer verticalement pour prendre toute la hauteur disponible.

Il faut noter que `stretch` est la valeur par défaut de `align-items`.

On aurait donc pu ne pas l’écrire, mais elle est utile ici pour rendre le comportement explicite.

---

# 9. Analyse de `.sidebar`

```css
.sidebar {
  flex: 0 0 200px;
  background: orange;
  display: flex;
  flex-direction: column;
}
```

La `.sidebar` est aussi particulière.

Elle est un élément flexible dans `.body`.

Mais elle devient aussi un conteneur flexible pour ses propres enfants.

---

## 9.1 `.sidebar` comme élément flexible

La propriété suivante contrôle la taille de `.sidebar` dans `.body` :

```css
flex: 0 0 200px;
```

Cette écriture est un raccourci pour :

```css
flex-grow: 0;
flex-shrink: 0;
flex-basis: 200px;
```

Cela signifie :

| Propriété     |  Valeur | Effet                              |
| ------------- | ------: | ---------------------------------- |
| `flex-grow`   |     `0` | la sidebar ne s’agrandit pas       |
| `flex-shrink` |     `0` | la sidebar ne rétrécit pas         |
| `flex-basis`  | `200px` | la taille de départ est de `200px` |

Comme le parent `.body` est en direction `row`, l’axe principal est horizontal.

Donc, ici :

```css
flex-basis: 200px;
```

correspond à une largeur de départ de `200px`.

La sidebar aura donc une largeur fixe de `200px`.

---

## 9.2 Pourquoi fixer `flex-basis: 200px` ?

Si on ne donnait pas de taille initiale à la sidebar, elle pourrait prendre seulement la largeur de son contenu.

Par exemple, si elle contient seulement les nombres :

```text
1
2
3
```

sa largeur pourrait devenir très petite.

En écrivant :

```css
flex: 0 0 200px;
```

on obtient une sidebar stable, lisible et prévisible.

---

## 9.3 `.sidebar` comme conteneur flexible

Ensuite, on transforme aussi `.sidebar` en conteneur flexible :

```css
display: flex;
```

Ses enfants directs sont les éléments `span`.

On utilise :

```css
flex-direction: column;
```

Cela signifie que les `span` seront empilés verticalement.

Résultat :

```text
1
2
3
```

---

# 10. Analyse de `.sidebar span`

```css
.sidebar span {
  margin: 0.5rem;
  background: teal;
  flex-basis: 50px;
}
```

Les éléments `span` sont les enfants directs de `.sidebar`.

Comme `.sidebar` est un conteneur flexible, les `span` deviennent des éléments flexibles.

---

## 10.1 Espacement avec `margin`

```css
margin: 0.5rem;
```

Cette règle ajoute une marge de `0.5rem` autour de chaque `span`.

Si la taille de base du navigateur est `16px`, alors :

```text
0.5rem = 8px
```

Cette marge permet de créer de l’espace :

- entre les éléments ;
- entre les éléments et les bords de la sidebar.

---

## 10.2 Taille des éléments avec `flex-basis`

```css
flex-basis: 50px;
```

Comme `.sidebar` utilise :

```css
flex-direction: column;
```

son axe principal est vertical.

Donc, pour ses enfants, `flex-basis` correspond à une hauteur de départ.

Les `span` auront donc une hauteur de base de `50px`.

---

# 11. Analyse de `.content`

```css
.content {
  background: yellow;
  display: flex;
  flex: auto;
  flex-flow: row wrap;
  padding: 0.2rem;
  align-content: flex-start;
}
```

La section `.content` est l’espace principal de la page.

Elle est un élément flexible dans `.body`.

Elle devient aussi un conteneur flexible pour ses propres enfants, c’est-à-dire les blocs de contenu.

---

## 11.1 `.content` comme élément flexible

```css
flex: auto;
```

Le raccourci :

```css
flex: auto;
```

équivaut à :

```css
flex: 1 1 auto;
```

Cela signifie :

| Propriété     | Valeur | Effet                               |
| ------------- | -----: | ----------------------------------- |
| `flex-grow`   |    `1` | le contenu peut s’agrandir          |
| `flex-shrink` |    `1` | le contenu peut rétrécir            |
| `flex-basis`  | `auto` | la taille de départ est automatique |

Dans notre mise en page, `.body` est en direction `row`.

La `.sidebar` prend `200px`.

La section `.content` prend donc l’espace horizontal restant.

---

## 11.2 `.content` comme conteneur flexible

```css
display: flex;
```

Cette règle transforme `.content` en conteneur flexible.

Ses enfants directs sont :

```text
.content div
```

Ils deviennent donc des éléments flexibles.

---

## 11.3 Direction et retour à la ligne

```css
flex-flow: row wrap;
```

Cette règle est un raccourci pour :

```css
flex-direction: row;
flex-wrap: wrap;
```

Cela signifie que les blocs à l’intérieur de `.content` sont placés horizontalement.

S’il n’y a pas assez de place, ils peuvent passer à la ligne.

Résultat possible :

```text
Bloc 1  Bloc 2  Bloc 3
Bloc 4  Bloc 5  Bloc 6
```

Cette configuration est très utile pour créer une zone de cartes ou une galerie responsive.

---

## 11.4 Espacement intérieur avec `padding`

```css
padding: 0.2rem;
```

Cette règle ajoute un petit espace entre les blocs et les bords de la zone `.content`.

Cela évite que les blocs soient collés directement contre les limites du conteneur.

---

## 11.5 Espacement entre les lignes avec `align-content`

```css
align-content: flex-start;
```

La propriété `align-content` agit lorsqu’il y a plusieurs lignes d’éléments flexibles.

Ici, comme `.content` utilise :

```css
flex-wrap: wrap;
```

les blocs peuvent passer sur plusieurs lignes.

La propriété `align-content` permet alors de contrôler l’espacement vertical entre ces lignes.

Par défaut, `align-content` vaut :

```css
align-content: stretch;
```

Cela peut étirer les lignes pour utiliser tout l’espace disponible.

Dans cet exemple, on utilise plutôt :

```css
align-content: flex-start;
```

Cela signifie que les lignes seront regroupées au début de l’axe secondaire.

Comme `.content` utilise `flex-direction: row`, l’axe secondaire est vertical.

Les lignes seront donc regroupées vers le haut.

---

# 12. Analyse de `.content div`

```css
.content div {
  background: purple;
  flex-basis: 200px;
  height: 200px;
  margin: 1rem;
}
```

Les `div` à l’intérieur de `.content` sont les blocs de contenu.

Comme `.content` est un conteneur flexible, ces `div` sont des éléments flexibles.

---

## 12.1 Largeur initiale avec `flex-basis`

```css
flex-basis: 200px;
```

Comme `.content` utilise :

```css
flex-direction: row;
```

son axe principal est horizontal.

Donc, ici, `flex-basis` correspond à une largeur initiale.

Chaque bloc commence donc avec une largeur de `200px`.

---

## 12.2 Hauteur fixe

```css
height: 200px;
```

Cette règle fixe la hauteur de chaque bloc à `200px`.

Chaque bloc aura donc une forme carrée approximative :

```text
200px de large
200px de haut
```

---

## 12.3 Marges autour des blocs

```css
margin: 1rem;
```

Cette règle ajoute une marge autour de chaque bloc.

Si la taille de base est `16px`, alors :

```text
1rem = 16px
```

Cela crée de l’espace entre les blocs.

Aujourd’hui, on pourrait aussi utiliser `gap` sur le conteneur `.content`, mais `margin` reste une solution fréquente dans beaucoup de codes existants.

---

# 13. Résumé des conteneurs flexibles

Dans cet exemple, il n’y a pas un seul conteneur flexible.

Il y en a plusieurs.

| Élément      | Devient un conteneur flexible ? | Enfants contrôlés      |
| ------------ | ------------------------------- | ---------------------- |
| `.container` | oui                             | `.header`, `.body`     |
| `.body`      | oui                             | `.sidebar`, `.content` |
| `.sidebar`   | oui                             | les `span`             |
| `.content`   | oui                             | les `div`              |

Chaque conteneur flexible contrôle seulement ses enfants directs.

C’est une règle importante de Flexbox.

---

# 14. Résumé des éléments flexibles

Un même élément peut être à la fois :

- un élément flexible dans son parent ;
- un conteneur flexible pour ses enfants.

Par exemple, `.body` est :

```text
un élément flexible de .container
un conteneur flexible pour .sidebar et .content
```

`.sidebar` est :

```text
un élément flexible de .body
un conteneur flexible pour ses span
```

`.content` est :

```text
un élément flexible de .body
un conteneur flexible pour ses div
```

Cette idée est essentielle pour comprendre les mises en page plus complexes.

# 17. Points d’attention

## 17.1 `flex-basis` dépend de l’axe principal

Il faut toujours regarder la direction du conteneur parent.

Avec :

```css
flex-direction: row;
```

`flex-basis` agit comme une largeur de départ.

Avec :

```css
flex-direction: column;
```

`flex-basis` agit comme une hauteur de départ.

---

## 17.2 `align-content` nécessite plusieurs lignes

La propriété :

```css
align-content: flex-start;
```

n’a un effet visible que si les éléments flexibles peuvent se répartir sur plusieurs lignes.

Il faut donc généralement avoir :

```css
flex-wrap: wrap;
```

Sans plusieurs lignes, `align-content` n’aura pas d’effet important.

---

## 17.3 `align-items: stretch` est la valeur par défaut

Dans `.body`, on écrit :

```css
align-items: stretch;
```

Cette règle est utile pédagogiquement, mais elle n’est pas obligatoire, car `stretch` est la valeur par défaut de `align-items`.

---

## 17.4 `flex: auto` et `flex: 1` ne veulent pas dire exactement la même chose

Ces deux écritures se ressemblent, mais elles n’ont pas exactement le même comportement.

```css
flex: auto;
```

équivaut à :

```css
flex: 1 1 auto;
```

La taille de départ est automatique.

```css
flex: 1;
```

signifie essentiellement que l’élément peut prendre l’espace disponible avec une base flexible.

Dans la pratique, `flex: 1` est souvent utilisé pour dire :

```text
Prends une part de l’espace disponible.
```

---

## 17.5 Un élément peut avoir deux rôles

Dans cet exemple, `.body`, `.sidebar` et `.content` ont deux rôles.

Ils sont des éléments flexibles dans leur parent.

Mais ils deviennent aussi des conteneurs flexibles pour leurs enfants.

C’est très courant dans les vraies interfaces Web.

---

# 18. Version améliorée avec `gap`

Dans le code original, on utilise des marges sur les éléments.

Exemple :

```css
.content div {
  margin: 1rem;
}
```

Une autre approche moderne consiste à utiliser `gap` sur le conteneur.

Exemple :

```css
.content {
  display: flex;
  flex-flow: row wrap;
  gap: 1rem;
  padding: 1rem;
  align-content: flex-start;
}

.content div {
  flex-basis: 200px;
  height: 200px;
}
```

L’avantage de `gap` est qu’il gère l’espace entre les éléments sans ajouter de marges individuelles.

Le code devient souvent plus clair.

---

# 20. Mini-bilan

Cet exemple montre comment construire une mise en page complète avec Flexbox.

Le conteneur principal `.container` organise la page en colonne :

```text
header
body
```

La section `.body` organise ses enfants en ligne :

```text
sidebar | content
```

La `.sidebar` organise ses éléments verticalement.

La section `.content` organise ses blocs en lignes avec retour possible à la ligne.

Pour analyser une mise en page Flexbox, il faut toujours se demander :

```text
1. Quel est le conteneur flexible ?
2. Quels sont ses enfants directs ?
3. Quelle est la direction de l’axe principal ?
4. Est-ce que les éléments peuvent passer à la ligne ?
5. Est-ce que la propriété utilisée agit sur le conteneur ou sur un élément ?
```

# Cheatsheet — Exemple de layout Flexbox

## Structure générale

```text
.container
├── .header
└── .body
    ├── .sidebar
    └── .content
```

---

## Conteneur principal

```css
.container {
  display: flex;
  flex-direction: column;
  height: 100vh;
}
```

Rôle :

```text
Organiser le header et le body verticalement.
```

---

## Header

```css
.header {
  min-height: 50px;
}
```

Rôle :

```text
Garder une hauteur minimale stable.
```

---

## Body

```css
.body {
  flex: 1;
  display: flex;
  flex-flow: row nowrap;
  align-items: stretch;
}
```

Rôle :

```text
Prendre l’espace restant et placer sidebar + content côte à côte.
```

---

## Sidebar

```css
.sidebar {
  flex: 0 0 200px;
  display: flex;
  flex-direction: column;
}
```

Rôle :

```text
Garder une largeur de 200px et empiler ses éléments.
```

---

## Éléments de la sidebar

```css
.sidebar span {
  flex-basis: 50px;
}
```

Rôle :

```text
Donner une hauteur de départ aux éléments, car la sidebar est en column.
```

---

## Content

```css
.content {
  flex: auto;
  display: flex;
  flex-flow: row wrap;
  align-content: flex-start;
}
```

Rôle :

```text
Prendre l’espace restant et organiser les blocs avec retour à la ligne.
```

---

## Blocs du contenu

```css

  height: 200px;
}
```

Rôle :

```text
Créer des blocs de 200px de large et 200px de haut.
```

---

## À retenir

Dans Flexbox, un élément peut être à la fois :

```text
un enfant flexible de son parent
et
un conteneur flexible pour ses propres enfants
```

C’est ce qui permet de construire des interfaces complètes et structurées.
